This manual describes the features and customization options for the Emacs Lisp file =bbdb-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=bbdb-extras.el= extends BBDB (the Insidious Big Brother Database),
Emacs' contact management system.  The stock BBDB interface requires
several prompts to create a record and asks for confirmation before
routine actions such as opening a URL or deleting a field.  This
package streamlines those interactions and adds the ability to export
contacts to the system address book.

The extensions fall into three areas:

- *Quick record creation*: create a new contact by providing only a
  name, skipping the prompts for email, phone, and other fields
  ([[#h:quick-record-creation][Quick record creation]]).
- *Export*: send the current BBDB record to macOS Contacts as a vCard
  ([[#h:export-vcard][Exporting to vCard]]).
- *Patched behaviors*: open URLs without confirmation and display the
  BBDB buffer without splitting windows ([[#h:patched-functions][Patched functions]]).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Quick record creation
:PROPERTIES:
:CUSTOM_ID: h:quick-record-creation
:END:

#+findex: bbdb-extras-create-quick
When you just need to jot down someone's name without filling in every
field, use ~bbdb-extras-create-quick~.  It prompts only for the
contact's first and last name, creates a minimal BBDB record, and
displays it.  This replaces the standard ~bbdb-create-internal~
workflow, which asks for email, phone, address, and notes.

Internally the command delegates to ~bbdb-extras-read-record-quick~
([[#h:read-record-quick][Reading a quick record]]), which handles the name prompt and
validation.

** Exporting to vCard
:PROPERTIES:
:CUSTOM_ID: h:export-vcard
:END:

#+findex: bbdb-extras-export-vcard
The command ~bbdb-extras-export-vcard~ exports the BBDB record at
point to a temporary =.vcf= file and opens it with the system's
default handler (=open= on macOS), which is typically the Contacts
application.  This provides a quick path from BBDB to the system
address book without manual copy-pasting.  It relies on the
=bbdb-vcard= package for the actual vCard serialization.

** Deleting without confirmation
:PROPERTIES:
:CUSTOM_ID: h:delete-no-confirm
:END:

#+findex: bbdb-extras-delete-field-or-record-no-confirm
By default, ~bbdb-delete-field-or-record~ asks for confirmation before
every deletion.  If you find this disruptive---especially during bulk
cleanup---the command
~bbdb-extras-delete-field-or-record-no-confirm~ performs the same
deletion but bypasses the confirmation prompt.  Use with care, as the
action is not undoable.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Reading a quick record
:PROPERTIES:
:CUSTOM_ID: h:read-record-quick
:END:

#+findex: bbdb-extras-read-record-quick
The function ~bbdb-extras-read-record-quick~ creates an empty BBDB
record and populates only its first and last name fields via
~bbdb-read-name~.  The optional FIRST-AND-LAST argument controls the
reading mode (see the ~bbdb-read-name~ docstring for details).  This
function is used by ~bbdb-extras-create-quick~
([[#h:quick-record-creation][Quick record creation]]).

* Patched functions
:PROPERTIES:
:CUSTOM_ID: h:patched-functions
:END:

=bbdb-extras.el= overrides two standard BBDB functions via ~advice-add~
with =:override=.

** Browsing URLs directly
:PROPERTIES:
:CUSTOM_ID: h:browse-url
:END:

#+findex: bbdb-extras-browse-url
The function ~bbdb-extras-browse-url~ replaces ~bbdb-browse-url~.  The
stock version asks the user to confirm before opening each URL; the
patched version opens the URL in the =url= field directly with
~browse-url~.  A prefix argument WHICH (starting from 0) selects which
URL to open when the field contains multiple entries.

** Displaying the BBDB buffer
:PROPERTIES:
:CUSTOM_ID: h:pop-up-window
:END:

#+findex: bbdb-extras-pop-up-window
The function ~bbdb-extras-pop-up-window~ replaces ~bbdb-pop-up-window~.
The stock version splits the largest window to show the =*BBDB*=
buffer, which can be disruptive in tiled layouts.  The patched version
reuses an existing window instead of splitting, providing a less
intrusive display.  If the =*BBDB*= buffer is already visible, the
function simply selects its window when SELECT is non-nil.
