This manual describes the features and customization options for the Emacs Lisp file =register-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=register-extras.el= extends the built-in Emacs register system (=register.el=) with additional storage operations and a unified transient menu. Emacs registers are single-character slots that can hold text, numbers, positions, window configurations, and more, but the standard commands are scattered and easy to forget. This package adds:

- A command to store text directly from the kill ring into a register ([[#h:kill-ring-to-register][Kill ring to register]]).
- A lightweight buffer-register system for quick switching between buffers by single-character keys ([[#h:buffer-registers][Buffer registers]]).
- A transient dispatch menu that groups all register operations in one place ([[#h:transient-menu][Transient menu]]).

The package depends on =register= and =transient=.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Kill ring to register
:PROPERTIES:
:CUSTOM_ID: h:kill-ring-to-register
:END:

#+findex: register-extras-kill-to-register
The built-in ~copy-to-register~ stores the current region, but sometimes you want to store a previously killed piece of text without first yanking and re-selecting it. The command ~register-extras-kill-to-register~ prompts you to choose an item from the kill ring (via ~read-from-kill-ring~) and then a target register (via ~register-read-with-preview~), and stores the chosen text in that register with ~set-register~. This saves the round-trip of yanking, selecting, and copying to a register.

** Buffer registers
:PROPERTIES:
:CUSTOM_ID: h:buffer-registers
:END:

#+findex: register-extras-buffer-to-register
#+findex: register-extras-jump-to-buffer
While Emacs registers can store buffer positions, there is no built-in way to bind a single character to a buffer for instant switching. The command ~register-extras-buffer-to-register~ prompts for a character and associates it with the current buffer in an internal alist (~register-extras-keys-alist~). Any previous association for that character is replaced.

The companion command ~register-extras-jump-to-buffer~ prompts for a character and switches to the buffer associated with it. If the buffer has been killed since the association was made, a message is displayed instead.

These two commands provide a lightweight alternative to bookmarks or tab-bar tabs when you want to toggle rapidly between a small set of buffers during a focused work session.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:

#+findex: register-extras-dispatch
The command ~register-extras-dispatch~ presents a transient menu that organizes all register-related operations by content type. The menu groups include:

- *region*: append, copy, and prepend to register.
- *number*: store and increment.
- *kill ring*: store via ~register-extras-kill-to-register~ ([[#h:kill-ring-to-register][Kill ring to register]]).
- *kmacro*: store and execute.
- *rectangle*: copy to register.
- *window/frame config*: store frame, window, or all-windows configuration.
- *navigation*: jump to register and store point.
- *buffer*: jump and store via the buffer-register commands ([[#h:buffer-registers][Buffer registers]]).
- *display*: consult (via =consult-register=), list, and view registers.
- *insert*: insert register contents.

This menu provides a single entry point for the full range of register operations, removing the need to remember individual key bindings for each command.
