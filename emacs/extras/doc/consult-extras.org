This manual describes the features and customization options for the Emacs Lisp file =consult-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=consult-extras.el= extends =consult=, the package that provides enhanced search and navigation commands built on Emacs' completing-read interface. While =consult= ships with powerful commands like ~consult-ripgrep~ and ~consult-locate~, it does not include directory-scoped variants or a way to toggle multiline matching on the fly.

This package fills those gaps with two groups of commands:

- Directory-scoped search commands that restrict =consult-ripgrep= and =consult-locate= to the current directory, avoiding the need to specify a search root each time ([[#h:directory-scoped-search][Directory-scoped search]]).
- An enhanced Org heading navigation command that widens the buffer and unfolds all headings before jumping, ensuring consistent results regardless of the buffer's visibility state ([[#h:org-heading-navigation][Org heading navigation]]).

The package also provides a toggle for multiline ripgrep search ([[#h:multiline-toggle][Multiline toggle]]).

It depends on the =paths= package for directory path definitions.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Multiline toggle
:PROPERTIES:
:CUSTOM_ID: h:multiline-toggle
:END:

#+findex: consult-extras-toggle-multiline
By default, ~consult-ripgrep~ searches line by line. When you need to find patterns that span multiple lines---such as a function signature followed by a specific body expression---the command ~consult-extras-toggle-multiline~ appends the =--multiline --multiline-dotall= flags to ~consult-ripgrep-args~. Calling it again removes the flags, restoring single-line behavior.

This is a toggle command: it modifies the global value of ~consult-ripgrep-args~, so the change persists until toggled back or until Emacs is restarted.

** Directory-scoped search
:PROPERTIES:
:CUSTOM_ID: h:directory-scoped-search
:END:

#+findex: consult-extras-locate-current
#+findex: consult-extras-locate-file-current
#+findex: consult-extras-ripgrep-current
Three commands restrict standard =consult= search commands to the current directory (~default-directory~):

- ~consult-extras-locate-current~ runs ~consult-locate~ scoped to the current directory. On macOS, it uses =mdfind-wrapper= with the =-onlyin= flag to restrict Spotlight's search scope. This is useful when you want to find a file somewhere within the current project tree without searching the entire filesystem.

- ~consult-extras-locate-file-current~ works like ~consult-extras-locate-current~ but adds the =-name= flag, so it matches only against file names rather than file contents or metadata. Use this when you know part of the filename but not its exact location within the directory tree.

- ~consult-extras-ripgrep-current~ runs ~consult-ripgrep~ in the current directory. This is a convenience wrapper that passes ~default-directory~ as the search root, saving you from having to specify it interactively.

** Org heading navigation
:PROPERTIES:
:CUSTOM_ID: h:org-heading-navigation
:END:

#+findex: consult-extras-org-heading
When working in a large Org file with many folded subtrees and narrowed regions, jumping to a heading with the standard ~consult-org-heading~ can produce unexpected results: narrowed-away headings are invisible, and folded content may not be reachable.

The command ~consult-extras-org-heading~ addresses this by widening the buffer and unfolding all headings (via ~org-extras-fold-show-all-headings~ from =org-extras.el=) before calling ~consult-org-heading~. After the jump, it recenters the display so the target heading appears near the top of the window.

It accepts the same optional MATCH and SCOPE arguments as ~consult-org-heading~, so it can be used as a drop-in replacement.
