This manual describes the features and customization options for the Emacs Lisp file =copilot-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=copilot-extras.el= extends the =copilot.el= package, which
integrates GitHub Copilot into Emacs.  The stock =copilot-mode= is a
simple on/off toggle with no built-in awareness of buffer context.
This extension adds two pieces of missing functionality:

- *Conditional activation*: enable Copilot automatically in
  programming buffers while skipping read-only buffers and
  user-specified modes ([[#h:conditional-activation][Conditional activation]]).
- *Restart workaround*: quickly cycle =copilot-mode= off and on when
  the process stops responding ([[#h:restart][Restarting Copilot]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~copilot-extras-excluded-modes~
:PROPERTIES:
:CUSTOM_ID: h:copilot-extras-excluded-modes
:END:

#+vindex: copilot-extras-excluded-modes
The user option ~copilot-extras-excluded-modes~ is a list of major
mode symbols in which =copilot-mode= should /not/ be activated by
~copilot-extras-enable-conditionally~
([[#h:conditional-activation][Conditional activation]]).  You might add modes where inline
completions are distracting or conflict with other tools---for
example, terminal emulators, shell prompts, or documentation browsers.

The type is a list of symbols.  The default value is an empty list
~()~, meaning no modes are excluded beyond the read-only check.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Conditional activation
:PROPERTIES:
:CUSTOM_ID: h:conditional-activation
:END:

#+findex: copilot-extras-enable-conditionally
The function ~copilot-extras-enable-conditionally~ is designed to be
added to a hook such as ~prog-mode-hook~ or ~text-mode-hook~.  It
activates =copilot-mode= in the current buffer /unless/ either of the
following is true:

1. The buffer is read-only (~buffer-read-only~ is non-nil).
2. The buffer's major mode appears in
   ~copilot-extras-excluded-modes~ ([[#h:copilot-extras-excluded-modes][~copilot-extras-excluded-modes~]]).

This provides a convenient "enable everywhere except..." policy without
requiring per-mode hook configuration.

** Restarting Copilot
:PROPERTIES:
:CUSTOM_ID: h:restart
:END:

#+findex: copilot-extras-restart-copilot
The function ~copilot-extras-restart-copilot~ quickly disables and
re-enables =copilot-mode= in the current buffer, with a brief pause
between the two operations.  This is a workaround for situations where
the Copilot process stops providing suggestions.  The function only
acts when =copilot-mode= is already active, so it is safe to call
unconditionally---for instance, from a periodic timer.
