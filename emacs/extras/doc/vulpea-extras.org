This manual describes the features and customization options for the Emacs Lisp file =vulpea-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=vulpea-extras.el= extends the =vulpea= package, a helper library for note-taking with Org Roam. Org Roam excels at linking and retrieving notes, but it does not automatically track which notes contain actionable items. This package bridges that gap by:

- Detecting whether an Org Roam note qualifies as a "project" based on its content---TODO items, scheduled headings, deadlines, or anniversaries ([[#h:project-detection][Project detection]]).
- Automatically tagging files with a ="project"= tag when they meet the criteria, and removing the tag when they no longer do ([[#h:updating-the-project-tag][Updating the project tag]]).
- Dynamically updating ~org-agenda-files~ before each agenda invocation to include project files and recently modified notes, ensuring your agenda always reflects the current state of your knowledge base ([[#h:updating-agenda-files][Updating agenda files]]).

The package depends on =paths= and =vulpea=, and integrates with =org-roam= and =org-roam-extras.el=.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Excluding directories
:PROPERTIES:
:CUSTOM_ID: h:excluding-directories
:END:

#+vindex: vulpea-extras-excluded-directories
The user option ~vulpea-extras-excluded-directories~ specifies a list of directory paths whose contents should be excluded from project detection. Any Org file located within one of these directories or their subdirectories will be ignored by ~vulpea-extras-project-p~, meaning it will never receive the ="project"= tag automatically. The default value contains two directories defined via the =paths= package. You might want to add directories that contain reference material or archival notes that should not appear in your agenda.

The type is a list of directory paths (~(repeat directory)~).

** Excluding files
:PROPERTIES:
:CUSTOM_ID: h:excluding-files
:END:

#+vindex: vulpea-extras-excluded-files
The user option ~vulpea-extras-excluded-files~ specifies individual file paths to exclude from project detection. This is useful when a specific file contains TODO items or deadlines but should not be treated as a project for agenda purposes. The default value is an empty list.

The type is a list of file paths (~(repeat file)~).

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Project detection
:PROPERTIES:
:CUSTOM_ID: h:project-detection
:END:

#+findex: vulpea-extras-project-p
The function ~vulpea-extras-project-p~ determines whether the current buffer qualifies as a project. It returns non-~nil~ if the buffer is in ~org-mode~, is not in an excluded directory or file ([[#h:excluding-directories][Excluding directories]], [[#h:excluding-files][Excluding files]]), and contains at least one headline that satisfies any of the following conditions:

- It is a TODO item (its ~:todo-type~ is ~todo~) and it is not archived.
- It is not done and has a ~SCHEDULED~ or ~DEADLINE~ property, and it is not archived.

The function respects tag inheritance when checking for the =ARCHIVE= tag. Completed tasks are deliberately excluded so that files containing only finished work are not flagged as active projects.

#+findex: vulpea-extras-anniversary-p
The function ~vulpea-extras-anniversary-p~ returns non-~nil~ if the current Org buffer contains an anniversary expression (=%%(org-anniversary ...)=). Files with anniversaries are treated as projects for tagging purposes, ensuring that recurring personal dates appear in the agenda.

#+findex: vulpea-extras-buffer-p
The function ~vulpea-extras-buffer-p~ returns non-~nil~ if the current buffer's file is located within ~org-roam-directory~. This is used as a guard to ensure that project-detection logic runs only on Org Roam files.

** Updating the project tag
:PROPERTIES:
:CUSTOM_ID: h:updating-the-project-tag
:END:

#+findex: vulpea-extras-project-update-tag
The function ~vulpea-extras-project-update-tag~ inspects the current buffer and adds the ="project"= tag if the buffer qualifies as a project (via ~vulpea-extras-project-p~ or ~vulpea-extras-anniversary-p~), or removes it if the buffer no longer qualifies. It uses ~vulpea-buffer-tags-set~ to modify the tags and only writes changes when the tag set has actually changed, avoiding unnecessary file modifications.

This function is designed to be added to a hook such as ~before-save-hook~ for Org Roam files. It includes guards against running in the minibuffer or when the buffer is in an inconsistent state (checked via ~track-changes-inconsistent-state-p~ when available).

** Updating agenda files
:PROPERTIES:
:CUSTOM_ID: h:updating-agenda-files
:END:

#+findex: vulpea-extras-agenda-files-update
The function ~vulpea-extras-agenda-files-update~ dynamically rebuilds ~org-agenda-files~ by combining three sources:

1. The current value of ~org-agenda-files~.
2. All files tagged with ="project"= in the Org Roam database (queried via ~vulpea-extras-project-files~).
3. Recently modified Org Roam files (retrieved via ~org-roam-extras-recent~ from =org-roam-extras.el=, limited to the last 2 days and 500 files).

After merging these lists and removing duplicates, it subtracts any files listed in ~org-extras-agenda-files-excluded~. This function is installed as =:before= advice on ~org-agenda~, so the agenda file list is always fresh when you open the agenda.

#+findex: vulpea-extras-project-files
The function ~vulpea-extras-project-files~ queries the Org Roam database for all files that contain a node tagged with ="project"=. It returns a deduplicated list of file paths. This is the function that connects the automatic tagging system ([[#h:updating-the-project-tag][Updating the project tag]]) to the agenda file list.

#+findex: vulpea-extras-file-in-excluded-directory-p
The function ~vulpea-extras-file-in-excluded-directory-p~ checks whether a given file resides in one of the directories listed in ~vulpea-extras-excluded-directories~ ([[#h:excluding-directories][Excluding directories]]). It is used internally by ~vulpea-extras-project-p~ to filter out files that should not be considered for project status.
