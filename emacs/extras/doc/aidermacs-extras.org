This manual describes the features and customization options for the Emacs Lisp file =aidermacs-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=aidermacs-extras.el= extends the =aidermacs= package, which
integrates the Aider AI coding assistant into Emacs.  While
=aidermacs= itself handles running Aider sessions and sending prompts,
this extension focuses on the surrounding workflow: extracting useful
information from the chat history, managing the Aider process
lifecycle, and launching Aider sessions with sensible defaults.

The main capabilities are:

- Copying recent or selected portions of the Aider chat history to the
  kill ring for reference or sharing ([[#h:chat-history][Chat history commands]]).
- Preventing accidental termination of active Aider sessions
  ([[#h:process-management][Process management]]).
- Saving the repository map to a file for inspection
  ([[#h:repository-map][Repository map]]).
- Launching Aider in the current directory with the current file
  pre-loaded ([[#h:launching-aider][Launching Aider]]).

** Chat history
:PROPERTIES:
:CUSTOM_ID: h:chat-history
:END:

#+findex: aidermacs-extras-copy-recent-history-to-kill-ring
After a long Aider session you may want to review or share the most
recent exchanges.  The command
~aidermacs-extras-copy-recent-history-to-kill-ring~ copies the last N
lines of the =.aider.chat.history.md= file (located in the project
root) to the kill ring.  By default N is 1000; with a prefix argument
(e.g., =C-u 500=) you can specify a different count.

The command uses the external =tail= utility rather than loading the
file into an Emacs buffer, which keeps it fast even when the history
file is very large.

#+findex: aidermacs-extras-copy-prompt-region
When you need a specific stretch of the conversation rather than the
tail end, use ~aidermacs-extras-copy-prompt-region~.  This command
scans the current buffer---expected to be the
=.aider.chat.history.md= file---for Aider prompt blocks (lines
beginning with =#### =) and prompts you to select a start block and
an end block via minibuffer completion.

The copied region spans from the beginning of the start block to the
beginning of the block /after/ the end block (or to the end of the
buffer if the end block is the last one).  Pressing =RET= without
selecting an end block copies everything from the start block to the
end of the buffer.  The text is placed in the kill ring.

** Repository map
:PROPERTIES:
:CUSTOM_ID: h:repository-map
:END:

#+findex: aidermacs-extras-save-repo-map
The command ~aidermacs-extras-save-repo-map~ runs
=aider --show-repo-map= and writes the output to =repo-map.md= in the
project root.  This gives you a static snapshot of the repository
structure that Aider sees, which can be useful for debugging context
issues.  When the optional ASYNC argument is non-nil, the command runs
asynchronously.

** Launching Aider
:PROPERTIES:
:CUSTOM_ID: h:launching-aider
:END:

#+findex: aidermacs-extras-run-in-current-dir
The command ~aidermacs-extras-run-in-current-dir~ starts an Aider
session in the current ~default-directory~.  If invoked from a buffer
that visits a file, the command automatically adds that file to the
Aider session with the =--file= flag.  It also looks for an
associated help file (via ~files-extras-get-help-file~ from
=files-extras.el=) and, if one exists, adds it as well.  This saves
you from having to manually specify files after launching Aider.

** Process management
:PROPERTIES:
:CUSTOM_ID: h:process-management
:END:

#+findex: aidermacs-extras-confirm-kill-buffer
The function ~aidermacs-extras-confirm-kill-buffer~ is designed to be
added to ~kill-buffer-query-functions~.  When you attempt to kill a
buffer, it checks whether the buffer is a =comint-mode= buffer whose
name starts with =*aidermacs= and which has a running process.  If all
three conditions hold, the function prompts for confirmation before
allowing the kill.  This prevents accidental termination of an active
Aider session---a mistake that would lose the current conversation
state.

If the buffer is not an Aidermacs buffer or has no running process,
the function returns ~t~ immediately and the kill proceeds without
interruption.
