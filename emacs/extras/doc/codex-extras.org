This manual describes the features and customization options for the Emacs Lisp file =codex-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=codex-extras.el= extends the =codex= package with additional functionality for working with the OpenAI Codex CLI from within Emacs.  It is the Codex counterpart to =claude-code-extras.el=, providing a similar set of conveniences for session management, theme synchronization, skill execution, project auditing, and debugging.

The package builds on =ai-extras.el= ([[info:ai-extras#Top][ai-extras]]) for shared functionality such as the unified session switcher, home-row key assignments, buffer protection, notifications, snippet expansion, and terminal fixes.  Codex-specific features include theme synchronization with Codex's TOML configuration, a skill runner that discovers skills from Codex's =.agents/skills/= directories, and notification handling for Codex CLI hook events.

At load time, =codex-extras.el= registers the Codex backend with =ai-extras.el= and installs hooks on ~codex-start-hook~, ~codex-event-hook~, and standard Emacs hooks to integrate all features automatically.  The backend icon is the official Codex terminal-prompt logo rendered as an inline SVG via ~ai-extras-svg-icon~ ([[info:ai-extras#h:svg-icons][SVG icons]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Theme synchronization
:PROPERTIES:
:CUSTOM_ID: h:theme-sync-options
:END:

#+vindex: codex-extras-sync-theme
The user option ~codex-extras-sync-theme~ controls whether Emacs theme changes are propagated to Codex.  When non-~nil~, switching between light and dark Emacs themes updates the =tui.theme= key in =~/.codex/config.toml= and sends the =/theme= command to all running Codex sessions.  The default is ~nil~.

** Session handoff
:PROPERTIES:
:CUSTOM_ID: h:handoff-options
:END:

#+vindex: codex-extras-handoff-file
The user option ~codex-extras-handoff-file~ specifies the path to the handoff file used by ~codex-extras-handoff~ ([[#h:handoff-command][Session handoff]]).  The default is =/tmp/codex-handoff.md=.  The handoff skill writes session transition instructions to this file before the command reads them.

** Skill runner
:PROPERTIES:
:CUSTOM_ID: h:skill-runner-options
:END:

#+vindex: codex-extras-skill-directories
The user option ~codex-extras-skill-directories~ specifies additional directories to scan for Codex skills, beyond the standard locations (=~/.codex/skills/= and the project's =.agents/skills/=).

#+vindex: codex-extras-run-skill-model
The user option ~codex-extras-run-skill-model~ overrides the model used when running skills via ~codex-extras-run-skill~.  The default is ~nil~, which uses the CLI's default model.

** Project audit
:PROPERTIES:
:CUSTOM_ID: h:audit-options
:END:

#+vindex: codex-extras-audit-skills
The user option ~codex-extras-audit-skills~ lists the skills invoked by ~codex-extras-audit-project~ ([[#h:audit-command][Project audit]]).  The default includes =/code-audit=, =/design-audit=, and =/interpretability-audit=.  Each skill is sent with the =--accept= flag for non-interactive execution.

#+vindex: codex-extras-audit-project-directories
The user option ~codex-extras-audit-project-directories~ provides completion candidates for the project audit directory prompt.  New directories entered by the user are automatically persisted to this list.

** Debug backtrace
:PROPERTIES:
:CUSTOM_ID: h:debug-options
:END:

#+vindex: codex-extras-debug-backtrace-model
The user option ~codex-extras-debug-backtrace-model~ specifies the =gptel= model used to analyze Emacs backtraces and identify the culprit package.  The default is ~gemini-flash-lite-latest~, a fast model suitable for this classification task.

#+vindex: codex-extras-debug-backtrace-backend
The user option ~codex-extras-debug-backtrace-backend~ specifies the =gptel= backend name for backtrace analysis.  The default is ="Gemini"=.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Session management
:PROPERTIES:
:CUSTOM_ID: h:session-commands
:END:

#+findex: codex-extras-start-or-switch
The command ~codex-extras-start-or-switch~ starts a new Codex session or opens the unified session switcher.  When no Codex sessions are active, it calls ~codex~ to start a new one.  When sessions exist, it delegates to the =ai-extras= session switcher ([[info:ai-extras#h:session-management][Session management]]), which displays all AI sessions (including Claude Code sessions, if any) with home-row key bindings.

#+findex: codex-extras-handoff
The command ~codex-extras-handoff~ closes the current Codex session and starts a new one with the handoff prompt.  The handoff skill must have been run first to write the handoff file at ~codex-extras-handoff-file~ ([[#h:handoff-options][Session handoff]]).  The new session starts in the same project directory as the closed one, with the handoff contents passed as a CLI argument.

** Tool execution
:PROPERTIES:
:CUSTOM_ID: h:tool-commands
:END:

#+findex: codex-extras-run-skill
The command ~codex-extras-run-skill~ discovers available Codex skills, prompts the user to select one with completion, optionally prompts for arguments, and sends the skill invocation to the active Codex session.  Skills are discovered from =~/.codex/skills/=, the project's =.agents/skills/= directory, and any directories in ~codex-extras-skill-directories~ ([[#h:skill-runner-options][Skill runner]]).

#+findex: codex-extras-audit-project
The command ~codex-extras-audit-project~ runs a comprehensive audit of a project directory.  It prompts for a directory (with completion from ~codex-extras-audit-project-directories~), then sequentially sends each skill in ~codex-extras-audit-skills~ to a Codex session in that directory.  Each skill is invoked with =--accept= for non-interactive execution.  If no existing Codex session is found, the command creates one; if session creation fails, it signals an error.

** Debugging
:PROPERTIES:
:CUSTOM_ID: h:debug-command
:END:

#+findex: codex-extras-debug-backtrace
The command ~codex-extras-debug-backtrace~ automates the workflow of debugging an Emacs error.  When invoked from the =*Backtrace*= buffer, it saves the backtrace, sends it to a lightweight LLM (via =gptel=) to identify candidate packages, presents the candidates via ~completing-read~, then starts a Codex session in the selected package's source directory with instructions to fix the bug.

The LLM backend and model are configured via ~codex-extras-debug-backtrace-backend~ and ~codex-extras-debug-backtrace-model~ ([[#h:debug-options][Debug backtrace]]).

** Buffer management
:PROPERTIES:
:CUSTOM_ID: h:buffer-commands
:END:

#+findex: codex-extras-setup-kill-on-exit
The command ~codex-extras-setup-kill-on-exit~ wraps the process sentinel so that the buffer is automatically killed when the Codex process exits.  This is called automatically on session startup via ~codex-start-hook~.

* Transient menus
:PROPERTIES:
:CUSTOM_ID: h:transient-menus
:END:

#+findex: ai-extras-menu
All =codex-extras= commands are accessible through the unified ~ai-extras-menu~ ([[info:ai-extras#h:transient-menu][ai-extras menu]]).  Codex-specific entries (such as the theme sync toggle) are appended to the shared menu at load time.  The menu is organized into columns:

- *Sessions*: start or switch, handoff.
- *Tools*: run skill, audit project, debug backtrace.
- *Alerts*: toggle alert notifications.
- *Buffer*: setup kill on exit, fix rendering, disable scrollback.
- *Options*: toggle boolean settings (alert on ready, protect buffers, sync theme).
