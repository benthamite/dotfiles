This manual describes the features and customization options for the Emacs Lisp file =faces-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=faces-extras.el= extends Emacs' built-in =faces= library with a system for programmatically managing face attributes. Emacs faces control the visual appearance of text---fonts, colors, heights, and styles---but configuring them in bulk requires repetitive calls to ~set-face-attribute~. This package provides a higher-level interface that can apply face settings individually or in batches, store them for later reapplication (for example, after a theme change), and evaluate dynamic values at apply-time.

The package also defines user options for common font and height settings used across fixed-pitch, variable-pitch, and Org mode elements, making it easy to adjust typographic parameters from a single location.

The main capabilities are:

- *Batch face attribute management.* Functions to set, store, and reapply collections of face attributes ([[#h:setting-face-attributes][Setting face attributes]]).
- *Font and height configuration.* User options for fixed-pitch and variable-pitch fonts, plus Org-specific height adjustments ([[#h:font-options][Font definitions]] and [[#h:height-options][Height definitions]]).
- *Enhanced face inspection.* A describe-face variant that disables ~hl-line-mode~ to avoid interference ([[#h:describe-face][Describing faces]]).
- *Custom face definitions.* A face for the display-time string in the mode line or tab bar ([[#h:display-time-face][Display time face]]).

** Font definitions
:PROPERTIES:
:CUSTOM_ID: h:font-options
:END:

#+vindex: faces-extras-variable-pitch-font
The user option ~faces-extras-variable-pitch-font~ specifies the font family for variable-pitch text. You might change this if you prefer a different serif or sans-serif font for prose-heavy buffers such as Org mode or EWW.

The default value is ="Source Serif Pro"=. The type is a string.

#+vindex: faces-extras-fixed-pitch-font
The user option ~faces-extras-fixed-pitch-font~ specifies the font family for fixed-pitch text. This is the monospace font used in code buffers, Org source blocks, and similar contexts.

The default value is ="SauceCodePro Nerd Font"=. The type is a string.

** Height definitions
:PROPERTIES:
:CUSTOM_ID: h:height-options
:END:

These options define sizes and relative heights for various text elements. Heights are expressed as floating-point multipliers relative to the base font size (e.g., ~0.8~ means 80% of the default height).

#+vindex: faces-extras-fixed-pitch-size
The user option ~faces-extras-fixed-pitch-size~ specifies the absolute size (in tenths of a point) for fixed-pitch faces. A value of ~120~ corresponds to a 12-point font. The type is an integer. The default value is ~120~.

#+vindex: faces-extras-fixed-pitch-height
The user option ~faces-extras-fixed-pitch-height~ specifies the relative height for fixed-pitch faces. This is typically used when fixed-pitch text appears alongside variable-pitch text and needs to be smaller to maintain visual harmony. The type is a number. The default value is ~0.8~.

#+vindex: faces-extras-variable-pitch-height
The user option ~faces-extras-variable-pitch-height~ specifies the relative height for variable-pitch faces. A value greater than ~1.0~ makes variable-pitch text larger than the default, which can improve readability for prose. The type is a number. The default value is ~1.4~.

#+vindex: faces-extras-org-level-height
The user option ~faces-extras-org-level-height~ controls the relative height for ~org-level-*~ heading faces. The type is a number. The default value is ~0.9~.

#+vindex: faces-extras-org-block-height
The user option ~faces-extras-org-block-height~ controls the relative height for the ~org-block~ face, which applies to source blocks and other block elements in Org. The type is a number. The default value is ~0.8~.

#+vindex: faces-extras-org-code-height
The user option ~faces-extras-org-code-height~ controls the relative height for the ~org-code~ face. The type is a number. The default value is ~1.2~.

#+vindex: faces-extras-org-date-height
The user option ~faces-extras-org-date-height~ controls the relative height for the ~org-date~ face. The type is a number. The default value is ~0.9~.

#+vindex: faces-extras-org-tag-height
The user option ~faces-extras-org-tag-height~ controls the relative height for the ~org-tag~ face. The type is a number. The default value is ~0.9~.

#+vindex: faces-extras-org-property-value-height
The user option ~faces-extras-org-property-value-height~ controls the relative height for the ~org-property-value~ face. The type is a number. The default value is ~0.8~.

** Setting face attributes
:PROPERTIES:
:CUSTOM_ID: h:setting-face-attributes
:END:

#+findex: faces-extras-set-face-attribute
#+findex: faces-extras-set-face-attributes
#+findex: faces-extras-set-and-store-face-attributes
#+findex: faces-extras-set-custom-face-attributes
The package provides a layered system for applying face attributes. At the lowest level, ~faces-extras-set-face-attribute~ takes a single attribute specification---a list whose ~car~ is the face symbol and whose ~cdr~ is a property list of attribute-value pairs---and applies it via ~set-face-attribute~. Crucially, it evaluates any symbols or functions in the value positions before applying them, so you can use variable names or lambdas as attribute values. If any evaluated value is or contains ~unspecified~ (which can happen before the theme has loaded), the call is skipped; the caller is expected to retry later via a theme-load hook:

#+begin_src emacs-lisp
(faces-extras-set-face-attribute '(default :height faces-extras-fixed-pitch-size))
#+end_src

The function ~faces-extras-set-face-attributes~ takes a list of such specifications and applies them all.

For persistent management, ~faces-extras-set-and-store-face-attributes~ both applies the attributes immediately and appends them to the variable ~faces-extras-custom-face-attributes~. List-valued attributes are automatically wrapped in lambdas for deferred evaluation. This means you can accumulate face settings from different parts of your configuration and reapply them all at once.

The command ~faces-extras-set-custom-face-attributes~ reapplies all stored attributes from ~faces-extras-custom-face-attributes~. This is useful after a theme change or frame creation, when faces may have been reset. You can add it to a theme-loading hook:

#+begin_src emacs-lisp
(add-hook 'modus-themes-after-load-theme-hook
          #'faces-extras-set-custom-face-attributes)
#+end_src

** Describing faces
:PROPERTIES:
:CUSTOM_ID: h:describe-face
:END:

#+findex: faces-extras-describe-face
The built-in ~describe-face~ command can be misleading when ~hl-line-mode~ is active, because the highlight overlay adds attributes that obscure the face you are trying to inspect. The command ~faces-extras-describe-face~ temporarily disables both buffer-local ~hl-line-mode~ and ~global-hl-line-mode~, calls ~describe-face~ with the face at point, and then restores the original mode state. This gives you an accurate reading of the face without manual toggling.

** Display time face
:PROPERTIES:
:CUSTOM_ID: h:display-time-face
:END:

The face ~faces-extras-display-time~ inherits from ~bold~ and is intended for the ~display-time-string~ when shown in the mode line or tab bar. It is used by the time element in =tab-bar-extras.el= ([[#h:display-time-face][Display time face]]) to give the clock a visually distinct appearance.
