This manual describes the features and customization options for the Emacs Lisp file =org2blog-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=org2blog-extras.el= extends =org2blog=, the package that publishes Org mode content to WordPress blogs. The upstream package provides commands for creating, editing, and publishing posts and pages, but its tag handling and confirmation prompts may not suit every workflow.

This package makes two targeted improvements:

- *Tag migration.* A command to convert standard Org heading tags into the =POST_TAGS= property that =org2blog= uses for WordPress tag assignment ([[#h:tag-migration][Tag migration]]).
- *Smarter confirmation.* Advice that makes the publishing confirmation prompt appear only for new posts (those without a =post-id=), skipping it for updates to already-published content ([[#h:conditional-confirmation][Conditional publishing confirmation]]).

The package depends on =org2blog=.

** Tag migration
:PROPERTIES:
:CUSTOM_ID: h:tag-migration
:END:

#+findex: org2blog-extras-move-tags-to-drawer
WordPress assigns tags to posts through the =POST_TAGS= property in the Org heading's property drawer, but you may already have tags defined in the standard Org tag format (=:tag1:tag2:= at the end of the heading line). The command ~org2blog-extras-move-tags-to-drawer~ scans the current buffer for headings with Org tags, converts them to a comma-separated string, and stores the result in the =POST_TAGS= property.

For example, a heading like =* My Post :emacs:org:= would gain a property =POST_TAGS= with the value ="emacs, org"=. The command processes all headings in the buffer from the current position forward, so you should typically invoke it with point at the beginning of the buffer.

Note that this command does not remove the original Org tags from the heading line. You may want to remove them manually after running the command if they are no longer needed.

** Conditional publishing confirmation
:PROPERTIES:
:CUSTOM_ID: h:conditional-confirmation
:END:

#+findex: org2blog-extras-subtree-publish-save
The =org2blog= variable ~org2blog/wp-confirm-post~, when set to ~t~, causes every publishing operation to prompt for confirmation. This is a useful safety net for new posts, but it becomes tedious when you are iterating on an already-published post or page.

The function ~org2blog-extras-subtree-publish-save~ is installed as =:around= advice on both ~org2blog-subtree-post-save~ and ~org2blog-subtree-page-save~. It checks whether the current post or page has a =post-id= property (indicating it has been published before):

- If a =post-id= exists, the original publishing function is called directly without any confirmation prompt. This makes rapid updates frictionless.
- If no =post-id= exists, the advice prompts with "Post has no ID. Proceed anyway?" before allowing publication. This prevents accidental creation of new posts.

This advice is applied automatically when =org2blog-extras.el= is loaded. It replaces the blanket confirmation with a context-sensitive check that protects against unintended first publications while streamlining updates.
