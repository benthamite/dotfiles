This manual describes the features and customization options for the Emacs Lisp file =slack-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=slack-extras.el= extends the =emacs-slack= package with personal workflow integrations that depend on external packages not bundled with =emacs-slack=. The package provides:

- Extended org link storage that adds ~slack-activity-feed-buffer-mode~ support to =ol-emacs-slack= ([[#h:link-storage][Link storage]]).
- Commands to capture the message at point to personal or work inboxes ([[#h:capture][Capture]]).
- A tab-bar notification element showing Slack unreads and mentions ([[#h:tab-bar-notifications][Tab-bar notifications]]).

The package depends on =slack= and =ol=.

* Link storage
:PROPERTIES:
:CUSTOM_ID: h:link-storage
:END:

#+findex: slack-extras--store-link
The function ~slack-extras--store-link~ replaces ~ol/slack-store-link~ as the =emacs-slack= link `:store` handler via ~org-link-set-parameters~. It delegates to the original ~ol/slack-store-link~ in ~slack-message-buffer-mode~ and ~slack-thread-message-buffer-mode~, and adds support for ~slack-activity-feed-buffer-mode~ by reading the ~ts~ and ~room-id~ text properties at point. This makes ~org-store-link~ and capture templates that use ~%a~ work in the activity feed.

** Capture
:PROPERTIES:
:CUSTOM_ID: h:capture
:END:

#+findex: slack-extras-personal-capture
The command ~slack-extras-personal-capture~ captures the Slack message at point to the personal inbox via ~org-capture~ template =s=. It is bound to =o= in ~slack-activity-feed-buffer-mode-map~, ~slack-thread-message-buffer-mode-map~ and ~slack-message-buffer-mode-map~.

#+findex: slack-extras-work-capture
The command ~slack-extras-work-capture~ captures the Slack message at point to the work inbox via ~org-capture~ template =S=. It is bound to =E= in ~slack-activity-feed-buffer-mode-map~, ~slack-thread-message-buffer-mode-map~ and ~slack-message-buffer-mode-map~.

* Tab-bar notifications
:PROPERTIES:
:CUSTOM_ID: h:tab-bar-notifications
:END:

#+vindex: tab-bar-extras-slack-notifications-enabled
The variable ~tab-bar-extras-slack-notifications-enabled~ controls whether the Slack notification element is displayed in the tab bar. It defaults to ~t~.

#+findex: slack-extras-actionable-unreads-p
The function ~slack-extras-actionable-unreads-p~ determines whether there are Slack notifications worth the user's attention. It triggers a debounced refresh of the Activity feed unread summary and returns the value of ~slack-has-unreads~.

The tab-bar element ~tab-bar-extras-slack-element~ displays a Slack icon (from =nerd-icons=) and a badge when ~slack-extras-actionable-unreads-p~ returns non-nil. The badge shows the total mention count, or a bullet (~•~) when there are actionable unreads but no @mentions (such as thread replies or DMs). Registration appends the element to ~tab-bar-extras-global-mode-string~ and immediately syncs ~global-mode-string~ via ~tab-bar-extras-set-global-mode-string~, so the badge appears even when =slack-extras= loads after the initial tab-bar sync.

#+findex: tab-bar-extras-toggle-slack-notifications
The command ~tab-bar-extras-toggle-slack-notifications~ toggles Slack notifications in the tab bar. It also participates in the master toggle: when ~tab-bar-extras-toggle-notifications~ is called, Slack notifications are toggled along with the other notification sources.
