This manual describes the features and customization options for the Emacs Lisp file =org-msg-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=org-msg-extras.el= extends =org-msg=, the package that enables composing rich HTML emails using Org mode syntax. While =org-msg= handles the core composition and export workflow, =org-msg-extras.el= adds convenience features around the edges: managing multiple email signatures for different contexts, copying message bodies for external grammar checking, and improving the initial cursor placement when composing.

The package provides three categories of functionality:

- *Signature management.* User options for defining separate plain text and HTML signatures for personal and work email accounts ([[#h:user-options][User options]]).
- *Message body operations.* Commands to copy the email body to the kill ring or to an external grammar checker ([[#h:message-body-operations][Message body operations]]).
- *Composition helpers.* Commands that position the cursor and adjust buffer state for a smoother composing experience ([[#h:composition-helpers][Composition helpers]]).

The package depends on =org-msg= and =org-extras=.

** Email signatures
:PROPERTIES:
:CUSTOM_ID: h:email-signatures
:END:

#+vindex: org-msg-extras-personal-plain-text-signature
#+vindex: org-msg-extras-personal-html-signature
#+vindex: org-msg-extras-epoch-plain-text-signature
#+vindex: org-msg-extras-epoch-html-signature
#+vindex: org-msg-extras-tlon-plain-text-signature
#+vindex: org-msg-extras-tlon-html-signature
=org-msg-extras.el= provides six user options for defining email signatures, covering three contexts (personal, Epoch AI, and Tlön) and two formats (plain text and HTML/Org).

The user option ~org-msg-extras-personal-plain-text-signature~ holds the signature for personal emails sent in plain text format. Its default value is ="\n--\nPablo\n"=, a minimal signature preceded by the conventional =--= separator. The type is a string.

The user option ~org-msg-extras-personal-html-signature~ holds the signature for personal emails composed in Org mode (which =org-msg= exports to HTML). Its default value is ="\n#+begin_signature\n--\n*Pablo*\n#+end_signature"=, using the Org signature block syntax that =org-msg= recognizes. The type is a string.

The user option ~org-msg-extras-epoch-plain-text-signature~ holds the signature for Epoch AI emails in plain text, including name, title, and website URL. The type is a string.

The user option ~org-msg-extras-epoch-html-signature~ holds the signature for Epoch AI emails in Org/HTML format, including the Epoch logo and links to the website, Twitter, and newsletter. The type is a string.

The user option ~org-msg-extras-tlon-plain-text-signature~ holds the signature for Tlön emails in plain text, including name, title, and company name. The type is a string.

The user option ~org-msg-extras-tlon-html-signature~ holds the signature for Tlön emails in Org/HTML format, with styled name, title, and company name. The type is a string.

These variables are not automatically inserted into messages by =org-msg-extras.el= itself. You should configure your mail user agent or =org-msg= setup to reference the appropriate variable based on the sending account. For example, you might set ~org-msg-signature~ dynamically in a hook based on which email account is active.

** Message body operations
:PROPERTIES:
:CUSTOM_ID: h:message-body-operations
:END:

#+findex: org-msg-extras-kill-message
#+findex: org-msg-extras-open-in-grammarly
When composing an email in =org-msg-edit-mode=, you may want to extract the message body for use elsewhere. The command ~org-msg-extras-kill-message~ copies the body of the current email to the Emacs kill ring, excluding the header section (everything up to and including the =:END:= line of the properties drawer) and the signature block (starting at =#+begin_signature=). If the current buffer is not in ~org-msg-edit-mode~, the command signals a user error.

The command ~org-msg-extras-open-in-grammarly~ builds on ~org-msg-extras-kill-message~ by first copying the message body to the kill ring and then opening the Grammarly web editor in the default browser. You can then paste the message body into Grammarly for grammar and style checking. This provides a workaround for cases where the Grammarly Emacs integration is not working reliably.

** Composition helpers
:PROPERTIES:
:CUSTOM_ID: h:composition-helpers
:END:

#+findex: org-msg-extras-begin-compose
#+findex: org-msg-extras-end-of-meta-data
When you begin composing an email in =org-msg=, the cursor may be positioned in the header area or after the signature, requiring manual navigation to the body. The command ~org-msg-extras-begin-compose~ moves point to the optimal position for starting to type the message body. If ~org-msg-signature~ is set, the command places point just before the signature. Otherwise, it calls ~org-msg-extras-end-of-meta-data~ to skip past the properties drawer and inserts a newline. It also calls ~simple-extras-visible-mode-enhanced~ to adjust buffer visibility for composing.

The command ~org-msg-extras-end-of-meta-data~ is a helper that advances point past consecutive Org property lines (lines matching ~org-property-re~). It is used internally by ~org-msg-extras-begin-compose~ to skip the meta-data section at the top of the =org-msg= buffer ([[#h:composition-helpers][Composition helpers]]).
