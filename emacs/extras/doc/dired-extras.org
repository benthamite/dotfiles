This manual describes the features and customization options for the Emacs Lisp file =dired-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=dired-extras.el= extends Dired, Emacs' built-in directory editor, with a collection of file operations, navigation improvements, and integrations with other packages. While Dired is already powerful, certain common tasks---copying filenames in various formats, quickly navigating to frequently used directories, managing screenshots, duplicating files---require custom commands.

This package provides:

- Commands for copying filenames to the kill ring in several formats: sans extension, absolute path, or context-sensitive based on the current detail level ([[#h:copying-filenames][Copying filenames]]).
- File marking, deletion, and duplication commands ([[#h:file-operations][File operations]]).
- Enhanced navigation and sorting ([[#h:navigation-and-sorting][Navigation and sorting]]).
- Image handling commands via Image Dired ([[#h:image-dired-integration][Image Dired integration]]).
- An enhanced detail-hiding toggle ([[#h:hide-details][Hide details]]).
- A transient dispatch menu for quick access to frequently used directories ([[#h:transient-dispatch][Transient dispatch]]).
- A patch for ~gnus-dired-mail-buffers~ to support modern mail composition modes ([[#h:gnus-dired-patch][Gnus Dired patch]]).

The package depends on =el-patch=, =paths=, =shut-up=, and =transient=.

** Copying filenames
:PROPERTIES:
:CUSTOM_ID: h:copying-filenames
:END:

#+findex: dired-extras-copy-filename-as-kill-sans-extension
#+findex: dired-extras-copy-filename-as-kill-absolute
#+findex: dired-extras-copy-filename-as-kill-dwim
#+findex: dired-extras-image-copy-filename-as-kill-absolute
Dired's built-in ~dired-copy-filename-as-kill~ copies the filename at point, but the format is not always what you need. Four commands cover common variations:

- ~dired-extras-copy-filename-as-kill-sans-extension~ copies the filename at point without its file extension. This is handy when you need the base name for renaming, referencing, or constructing a related filename.

- ~dired-extras-copy-filename-as-kill-absolute~ copies the full absolute path of the marked files (or the file at point). Internally, it passes the prefix argument =(0)= to ~dired-copy-filename-as-kill~.

- ~dired-extras-copy-filename-as-kill-dwim~ adapts its behavior to the current display state. When ~dired-hide-details-mode~ is active, it copies base filenames (without directory components); otherwise, it copies full paths. The filenames are joined with newlines, making them easy to paste into a shell or document.

- ~dired-extras-image-copy-filename-as-kill-absolute~ works in Image Dired buffers, copying the absolute path of the marked image file (or the one at point) to the kill ring.

** File operations
:PROPERTIES:
:CUSTOM_ID: h:file-operations
:END:

#+findex: dired-extras-mark-screenshots
#+findex: dired-extras-do-delete-fast
#+findex: dired-extras-duplicate-this-file
#+findex: dired-extras-copy-to-remote-docs-directory
Several commands streamline common file operations:

- ~dired-extras-mark-screenshots~ marks all files matching the macOS screenshot naming pattern (=Screenshot YYYY-MM-DD at HH.MM.SS.png=). This is useful for quickly selecting screenshots for bulk operations such as moving, renaming, or deleting.

- ~dired-extras-do-delete-fast~ deletes marked files (or the file at point) without moving them to the system trash. It temporarily overrides ~system-move-file-to-trash~ before calling ~dired-do-delete~. Use this when deleting large numbers of files where the overhead of the trash utility is unwanted.

- ~dired-extras-duplicate-this-file~ creates a copy of the file at point in the same directory, appending =[N]= before the extension where N is the first available integer (e.g., =report.pdf= becomes =report[1].pdf=). The Dired buffer is reverted afterward to show the new file.

- ~dired-extras-copy-to-remote-docs-directory~ copies marked files to a remote server (=stafforini.com/docs=) via =scp= and copies the public URL of the last file to the kill ring. This command contains hardcoded server details and requires appropriate SSH configuration.

** Navigation and sorting
:PROPERTIES:
:CUSTOM_ID: h:navigation-and-sorting
:END:

#+findex: dired-extras-up-directory-reuse
#+findex: dired-extras-sort-toggle-dwim
Two commands improve Dired navigation:

- ~dired-extras-up-directory-reuse~ navigates to the parent directory by calling ~find-alternate-file~ with =".."=, reusing the current Dired buffer instead of opening a new one. This keeps the buffer list clean when browsing directory trees.

- ~dired-extras-sort-toggle-dwim~ toggles the sorting method (typically between name and date) and refreshes the buffer. If point is on the first file when sorting is toggled, it remains on the first file afterward; otherwise, it follows the file that was at point. With a prefix argument, it calls ~dired-sort-toggle-or-edit~ to let you edit the listing switches directly.

** Image Dired integration
:PROPERTIES:
:CUSTOM_ID: h:image-dired-integration
:END:

#+findex: dired-extras-image-dired-current-directory
#+findex: dired-extras-copy-image
Two commands integrate Dired with image viewing:

- ~dired-extras-image-dired-current-directory~ opens all images in the current Dired directory in Image Dired, calling ~image-dired-show-all-from-dir~.

- ~dired-extras-copy-image~ copies the image in the current =image-mode= buffer to the system clipboard using macOS AppleScript. It sets the clipboard to the POSIX file path of the current buffer. This is macOS-specific.

** Hide details
:PROPERTIES:
:CUSTOM_ID: h:hide-details
:END:

#+findex: dired-extras-hide-details-mode-enhanced
The command ~dired-extras-hide-details-mode-enhanced~ toggles both ~dired-hide-details-mode~ and ~dired-omit-mode~ simultaneously. This provides a consistent "clean" and "full" view: in clean mode, file details and omitted files (such as dotfiles) are both hidden; in full mode, everything is visible.

It accepts an optional ARG following the standard toggle convention: when called interactively it toggles, a positive number enables the clean view, and a negative number disables it.

* Transient menus
:PROPERTIES:
:CUSTOM_ID: h:transient-dispatch
:END:

#+findex: dired-extras-dispatch
The command ~dired-extras-dispatch~ opens a transient menu providing quick access to frequently used directories. The menu is organized into columns:

- *Dired folders*: personal directories such as bibliography, dotfiles, Emacs config, notes, downloads, repos, and various library locations.
- *Music*: subdirectories for classical, popular, tango, and unsorted music.
- *Special*: context-sensitive entries like the file at point, root, home, and the current buffer's directory (via ~dired-jump~).
- *Projects*: personal project repositories (consensus-trader, stafforini.com, tangodb), constructed from ~paths-dir-personal-repos~.
- *Tlon: Google Drive* and *Tlon: Dropbox*: project-specific directories for various Tlon publications and team members.

Directory paths are drawn from variables defined in the =paths= package.

** Quiet deletion cleanup
:PROPERTIES:
:CUSTOM_ID: h:quiet-deletion
:END:

#+findex: dired-extras-clean-up-after-deletion-quietly
#+findex: dired-extras-y-or-n-p-just-yes
When Dired deletes a file, it may prompt to kill the associated buffer. The function ~dired-extras-clean-up-after-deletion-quietly~ is installed as =:around= advice on ~dired-clean-up-after-deletion~ to suppress these prompts. It temporarily wraps ~y-or-n-p~ with ~dired-extras-y-or-n-p-just-yes~, which returns ~t~ for any prompt starting with "Kill Dired buffer" or "Kill buffer of", and delegates to the original function otherwise.

** Gnus Dired patch
:PROPERTIES:
:CUSTOM_ID: h:gnus-dired-patch
:END:

#+findex: gnus-dired-mail-buffers
The function ~gnus-dired-mail-buffers~ (from =gnus-dired=) is patched via =el-patch= to use ~derived-mode-p~ instead of checking for an exact major mode match. The original implementation only recognizes buffers in ~mail-mode~ or those returned by ~message-buffers~, which means ~mu4e-compose-mode~ and other derived mail modes are not detected. The patch makes ~gnus-dired-attach~ work correctly from any mail composition buffer that derives from ~mail-mode~.
