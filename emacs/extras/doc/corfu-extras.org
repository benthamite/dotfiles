This manual describes the features and customization options for the Emacs Lisp file =corfu-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=corfu-extras.el= extends =corfu=, the lightweight completion-overlay package for Emacs. Corfu displays completion candidates in a popup at point, but two common workflow needs are not covered out of the box:

- Enabling Corfu in the minibuffer without conflicting with =vertico=, which also operates there ([[#h:minibuffer-activation][Minibuffer activation]]).
- Transferring a Corfu completion session to the minibuffer for richer interaction via =consult= ([[#h:transfer-to-minibuffer][Transferring completion to the minibuffer]]).

This package provides one function and one command addressing these needs. The transfer command requires the =consult= package.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Minibuffer activation
:PROPERTIES:
:CUSTOM_ID: h:minibuffer-activation
:END:

#+findex: corfu-extras-enable-always-in-minibuffer
When both Corfu and Vertico are installed, enabling Corfu unconditionally in the minibuffer would cause conflicts: both packages would attempt to display candidates simultaneously. The function ~corfu-extras-enable-always-in-minibuffer~ solves this by checking whether Vertico is active (via the internal variable ~vertico--input~) before enabling ~corfu-mode~.

This function is designed to be added to ~minibuffer-setup-hook~. With it in place, Corfu provides completions in prompts like =M-:= (~eval-expression~) and =M-!= (~shell-command~), while Vertico remains in charge during =M-x=, ~find-file~, and other standard completing-read prompts.

This approach is adapted from Protesilaos Stavrou's configuration.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Transferring completion to the minibuffer
:PROPERTIES:
:CUSTOM_ID: h:transfer-to-minibuffer
:END:

#+findex: corfu-extras-move-to-minibuffer
Sometimes the Corfu popup is too constrained for the task at hand: you may want to filter candidates further, preview them, or use Consult's narrowing features. The command ~corfu-extras-move-to-minibuffer~ transfers the active Corfu completion session to the minibuffer by calling ~consult-completion-in-region~. The completion boundaries, table, predicate, and extra properties are all preserved.

This command is automatically added to ~corfu-continue-commands~, which means invoking it does not dismiss the Corfu popup prematurely. You can bind it to a key in ~corfu-map~ for convenient access while the popup is visible.
