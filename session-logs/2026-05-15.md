## Key findings

- Temporary package PR pins need an explicit lifecycle marker in `config.org`:

  ```elisp
  ; awaiting PR merge: https://github.com/OWNER/REPO/pull/NUMBER
  ```

- `release-dotfiles` should own cleanup and release safety: report open pins, restore merged pins, and stop on closed-unmerged pins.
- The PR creation workflow should own pin creation: after the PR URL exists, update the package recipe to the PR head repo and branch, tangle `config.org`, and commit the dotfiles pin separately.
