* Overview
:PROPERTIES:
:ID:       FC7EF2FA-CDB3-43C0-9EFC-545DEA807191
:END:

This directory contains my [[https://docs.anthropic.com/en/docs/claude-code][Claude Code]] configuration: settings, hooks, and custom skills. Claude Code is a CLI-based coding agent; I use it exclusively from the terminal. Indeed, these days I hardly do anything other than using Claude Code.

The =~/.claude/= directory symlinks its core files here, so changes are version-controlled:

- =~/.claude/settings.json= → =claude/settings.json=
- =~/.claude/settings.local.json= → =claude/settings.local.json=
- =~/.claude/CLAUDE.md= → =claude/CLAUDE.md=
- =~/.claude/skills/= → =claude/skills/=

** CLAUDE.md
:PROPERTIES:
:ID:       9B404E8C-F109-4188-9F3E-1F1A9A64A37D
:END:

Project-level instructions referencing the =dotfiles-context= skill. The global =~/.claude/CLAUDE.md= (symlinked here) defines conventions that apply across all projects:

- Fix root causes, not symptoms; no silent fallbacks.
- Verify fixes before declaring them done.
- Commit all changes immediately; never mix unrelated changes.
- Use subagents and teams liberally; prefer the most powerful model.
- Use =trash= instead of =rm -rf=; never kill Emacs without confirmation.

** Development skills
:PROPERTIES:
:ID:       71770F84-ED6B-4F0B-B121-1F23CBCEB769

*** code-audit
:PROPERTIES:
:ID:       D3121B68-8480-41B2-835B-97E85BB7DE29
:END:

Audits code for bugs, security issues, error handling gaps, and correctness problems. Categorizes findings as critical, bugs, fragile, or minor, with specific line numbers. Pass =--accept= to auto-fix all findings.

*** design-audit

Audits code for architectural quality: unnecessary complexity, missing or wrong abstractions, duplication, and refactoring opportunities. Pass =--accept= to auto-refactor high-impact findings.

*** interpretability-audit
:PROPERTIES:
:ID:       800B3CAA-A993-4929-BB4A-7467A56FA70C
:END:

Audits code for intent transparency: magic values, misleading names, missing rationale comments, undocumented conventions. Pass =--accept= to auto-fix high-impact clarity issues.

*** build
:PROPERTIES:
:END:

Spec-based build workflow inspired by [[https://x.com/trq212/status/2005315275026260309][Thariq Shaukat's method]]. Reads =spec.md= in the current project and interviews the user in depth using =AskUserQuestion= to flesh out every aspect: technical implementation, UI/UX, tradeoffs, edge cases, security, and more. The questions are deliberately non-obvious, probing hidden complexity and unstated assumptions. Once the interview is complete, writes the detailed spec back to the file. Optionally executes the spec in the same session or defers to a fresh one.

*** test-suite
:PROPERTIES:
:ID:       057FB141-D07F-4BD4-B880-912B67275263
:END:

Creates comprehensive test suites. Performs reconnaissance on the tech stack, then generates tests prioritized by criticality: data integrity and input boundaries first, business logic and integrations second, regressions and configuration third.

*** pr-audit
:PROPERTIES:
:ID:       03187E6C-B3E2-47EB-821B-68121F4F883F
:END:

Six-step PR verification: byte-compile/type-check/lint, run tests, review commits, check upstream alignment, verify file hygiene, and check CI results. I run it before submitting or merging any PR.

*** add-to-emacs-packages

Checks both the packages list and the GitHub profile README, skipping whichever is already done.

** Release and operations skills
:PROPERTIES:
:ID:       F49B76EA-C29B-45D6-AD8D-D3E47DB4073E
:END:

*** release
:PROPERTIES:
:ID:       10DF729B-32D5-48E0-A150-9F5D18F799AF
:END:

Cuts releases for my Emacs packages (read dynamically from =my-emacs-packages.org=). Validates the repo, classifies commits since the last tag, suggests a semver bump, and drafts release notes. Also has an audit mode that scans all packages for version mismatches or unreleased commits. Pass =--accept= to skip the confirmation gate.

*** release-dotfiles

Cuts releases for the dotfiles repo itself. Verifies that every personal package repo has a clean working tree in its local elpaca clone, checks the dotfiles working tree, classifies commits since the last tag, suggests a semver bump, drafts release notes, and creates a GitHub release. Pass =--accept= to skip the confirmation gate.

*** triage-issues

Scans, prioritizes, and batch-resolves open GitHub issues for =tlon-team/tlon.el=. Fetches issues, ranks by complexity, presents batches for review, fixes them with parallel subagents, closes on GitHub as =tlon-ai=, and archives the corresponding org headings.

*** emacs-freeze
:PROPERTIES:
:ID:       5ED7EB34-774B-489C-95C5-27C8131CE048
:END:

Diagnoses a frozen Emacs process: samples the stack, decodes native-comp symbols, identifies the hot path, and suggests recovery options (SIGUSR2, SIGTERM, or SIGKILL with appropriate warnings).

** Multi-model and research skills
:PROPERTIES:
:ID:       8BF704E6-F252-4E27-846A-0B2F9C52B62C
:END:

*** best-of-n
:PROPERTIES:
:ID:       76E6E656-8DCE-4433-B9AA-1F6D0ACDB3B0
:END:

Queries multiple AI models with temperature-varied sampling. For each model, generates N responses and selects the best one (using Gemini 3 Flash as judge). Then synthesizes across models using Claude Opus 4.6 with extended thinking. Has a brainstorm mode that merges unique ideas instead of picking one winner. Presets: Quick (4 models x 3), Intense (7 x 5), Ultra (7 x 6).

*** proofread
:PROPERTIES:
:ID:       D9EC1B04-5470-4EF9-8C4C-FEDFB2333128
:END:

Proofreads documents using either aspell (fast, spelling-only) or an LLM (Gemini Flash, with configurable depth: mechanical, light style, or comprehensive). Auto-applies safe corrections and flags style suggestions for interactive review. Uses British English conventions.

** Meta skills

*** migrate-profile

Consolidates Claude Code project data (sessions, memory) across elpaca profile changes. Scans =~/.claude/projects/= for all directories matching the elpaca pattern, groups them by package name (extracted by splitting on =-elpaca-repos-= or =-elpaca-sources-=), and merges all old-profile entries into the current profile's directory. No explicit package list is needed. Handles known package renames (e.g. =infovore= → =elfeed-ai=). Non-destructive: never overwrites existing data.

* Hooks
:PROPERTIES:
:ID:       4D862953-7E8A-4154-BC16-919ED6B092A5
:END:

Three hooks enforce workflow invariants.

** require-elisp-test-before-commit.sh (PreToolUse)
:PROPERTIES:
:ID:       4E0CDC5D-C436-4701-AC99-E29F950318B4
:END:

Blocks =git commit= when Elisp files (=*.el= or =emacs/config.org=) are staged but no test has been run in the current session. Forces the agent to run =emacs --batch= before committing.

** track-elisp-test.sh (PostToolUse)
:PROPERTIES:
:ID:       85C6B926-9F3F-431A-9650-F504CEE12B4A
:END:

Creates a marker file (=/tmp/claude-elisp-tested-{SESSION_ID}=) whenever =emacs --batch= runs. This marker is what the pre-commit hook checks for. The marker is consumed (deleted) after a successful commit, so each commit requires a fresh test.

** symlink-project-memory.sh (PostToolUse)
:PROPERTIES:
:ID:       086B58F4-949F-4C88-B855-687EAD5F3B63
:END:

On the first Edit or Write in a project, moves Claude Code's memory directory from its default location (=~/.claude/projects/<encoded-path>/memory/=) into the project's =.claude/memory/= directory, replacing the original with a symlink. This keeps memory local and version-controllable.
