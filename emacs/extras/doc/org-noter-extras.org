This manual describes the features and customization options for the Emacs Lisp file =org-noter-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=org-noter-extras.el= extends =org-noter=, the package that facilitates taking interleaved notes on PDF documents within Org mode. When =org-noter= creates skeleton annotations from PDF highlights, the resulting headings are generic ("Highlight on page N") and the extracted text often contains hyphenation artifacts from the PDF layout engine. =org-noter-extras.el= provides a suite of commands to clean up, reformat, and navigate these annotations more efficiently.

The package provides the following capabilities:

- *Annotation cleanup.* A command to replace generic highlight headings with custom titles and wrap the extracted text in quote blocks ([[#h:annotation-cleanup][Annotation cleanup]]).
- *Annotation merging.* A command to merge the contents of two adjacent annotations into one, useful when a single passage spans multiple highlights ([[#h:merging-annotations][Merging annotations]]).
- *Dehyphenation.* Functions to remove hyphenation artifacts from text extracted from PDFs, where words broken across lines retain their hyphens ([[#h:dehyphenation][Dehyphenation]]).
- *Page offset correction.* A command to adjust page numbers in annotation headings when the PDF page numbers differ from the printed page numbers ([[#h:page-offset][Page offset correction]]).
- *Navigation.* Commands to move between notes while keeping focus in the Org buffer rather than switching to the PDF viewer ([[#h:navigation][Navigation]]).

The package depends on =org-noter=.

** Dehyphenation characters
:PROPERTIES:
:CUSTOM_ID: h:dehyphenation-characters
:END:

#+vindex: org-noter-extras-dehyphenate-hyphens
The constant ~org-noter-extras-dehyphenate-hyphens~ holds a list of strings representing hyphen characters that should be treated as word-break markers by the dehyphenation functions. Its default value is =("-" "¬")=, covering the standard ASCII hyphen and the not sign (=U+00AC=), which some PDF extraction tools produce in place of soft hyphens. You can change the active hyphen character interactively with ~org-noter-extras-set-hyphen~ ([[#h:dehyphenation][Dehyphenation]]).

** Heading and quote patterns
:PROPERTIES:
:CUSTOM_ID: h:heading-and-quote-patterns
:END:

#+vindex: org-noter-highlight-heading-regexp
#+vindex: org-noter-quote-begin
#+vindex: org-noter-quote-end
Several constants define the patterns that =org-noter-extras.el= uses to identify and manipulate annotation structures:

The constant ~org-noter-highlight-heading-regexp~ matches the default heading format that =org-noter= creates for highlight annotations: ="Highlight on page[s]? \\(.*\\)"=. The first capture group extracts the page number or page range. This pattern is used by the cleanup, merge, and offset commands.

The constant ~org-noter-quote-begin~ holds the string ="\n#+begin_quote\n"=, used when wrapping annotation content in an Org quote block during cleanup ([[#h:annotation-cleanup][Annotation cleanup]]).

The constant ~org-noter-quote-end~ holds the string ="\n#+end_quote\n"=, matching the end of the quote block.

* Commands
:PROPERTIES:

** Annotation cleanup
:PROPERTIES:
:CUSTOM_ID: h:annotation-cleanup
:END:

#+findex: org-noter-extras-cleanup-annotation
After running ~org-noter-create-skeleton~ to generate annotations from PDF highlights, each annotation gets a generic heading like "Highlight on page 42" with the extracted text in a child "Contents" heading. The command ~org-noter-extras-cleanup-annotation~ transforms this into a more useful format:

1. It synchronizes the current note position with the PDF viewer.
2. It switches to the Org notes buffer.
3. It prompts you for a descriptive title for the annotation.
4. It extracts the text content from the "Contents" subheading, running dehyphenation and filling the text to remove PDF line breaks.
5. It replaces the generic heading with the supplied title (without any page-number suffix).
6. It wraps the extracted content in an Org quote block (=#+begin_quote= ... =#+end_quote=).
7. It appends a citation of the form =[cite:@KEY, p. N]= after the quote block, where =KEY= is the bib key obtained from the inherited =Custom_ID= property and =N= is the page number extracted from the original heading.

This command is intended to be called with point on a "Highlight on page N" heading or its "Contents" child heading. It signals an error if no =Custom_ID= property can be found by inheritance.

** Merging annotations
:PROPERTIES:
:CUSTOM_ID: h:merging-annotations
:END:

#+findex: org-noter-extras-merge-contents-with-next
When a highlighted passage spans two separate PDF annotations (for example, because the text crosses a page boundary or you highlighted in two steps), you end up with two adjacent "Highlight on page" headings. The command ~org-noter-extras-merge-contents-with-next~ combines them into a single annotation:

1. With point on a "Highlight on page N" heading, it extracts the contents from both the current and the next annotation at the same outline level.
2. It renames the current heading to "Highlight on pages N--M" (using an en dash), where M is the page number from the next annotation.
3. It replaces the body of the current "Contents" subheading with the concatenation of both annotations' text.
4. It deletes the next annotation subtree entirely.

If point is not on a highlight heading, or if there is no next annotation at the same level, the command signals a user error.

** Dehyphenation
:PROPERTIES:
:CUSTOM_ID: h:dehyphenation
:END:

#+findex: org-noter-extras-dehyphenate
#+findex: org-noter-extras-set-hyphen
Text extracted from PDFs often contains hyphens at the positions where the original layout engine broke words across lines. The command ~org-noter-extras-dehyphenate~ removes these artifacts by searching for the pattern "letter-hyphen space letter" and joining the two letter sequences. It operates on the current paragraph by default, or on the active region if one is set. The hyphen characters it searches for are defined by ~org-noter-extras-dehyphenate-hyphens~ ([[#h:dehyphenation-characters][Dehyphenation characters]]).

The command ~org-noter-extras-set-hyphen~ lets you interactively change the hyphen character used for dehyphenation. It presents a choice between =-= (standard ASCII hyphen) and =­= (soft hyphen) and updates ~org-noter-extras-dehyphenate-hyphens~ accordingly. This is useful when you encounter a PDF that uses an unusual character for line-break hyphens.

** Page offset correction
:PROPERTIES:
:CUSTOM_ID: h:page-offset
:END:

#+findex: org-noter-extras-highlight-offset
PDF page numbers and printed page numbers often differ because the PDF includes front matter (title page, table of contents, etc.) that is not counted in the book's own pagination. The command ~org-noter-extras-highlight-offset~ corrects all "Highlight on page N" headings in the current buffer by subtracting a user-specified offset from each page number.

The offset is defined as the PDF page number minus the book page number. For example, if the PDF viewer shows page 220 but the printed page says 210, the offset is 10. The command prompts for this value interactively. The offset is typically positive (PDF pages tend to be higher than book pages) but can be negative if needed.

** Navigation
:PROPERTIES:
:CUSTOM_ID: h:navigation
:END:

#+findex: org-noter-extras-sync-next-note
#+findex: org-noter-extras-sync-prev-note
The standard =org-noter= navigation commands ~org-noter-sync-next-note~ and ~org-noter-sync-prev-note~ move to the next or previous annotation and shift focus to the PDF viewer window. This can be disruptive when you want to review annotations in sequence while keeping focus on the Org notes buffer.

The command ~org-noter-extras-sync-next-note~ calls the standard ~org-noter-sync-next-note~ and then immediately switches focus back to the Org notes window using ~other-window~. The command ~org-noter-extras-sync-prev-note~ does the same for backward navigation with ~org-noter-sync-prev-note~. The PDF buffer still scrolls to the correct location; only the keyboard focus differs.

** Extracting annotation contents
:PROPERTIES:
:CUSTOM_ID: h:extracting-annotation-contents
:END:

#+findex: org-noter-get-annotation-contents
The function ~org-noter-get-annotation-contents~ extracts and cleans the text from an =org-noter= annotation subtree. It determines whether point is on a "Highlight on page" heading or a "Contents" heading, narrows to the content region, fills the text with a very large ~fill-column~ (effectively unwrapping all lines), runs ~org-noter-extras-dehyphenate~ to remove hyphenation artifacts ([[#h:dehyphenation][Dehyphenation]]), and returns the cleaned text as a string.

This function is used internally by ~org-noter-extras-cleanup-annotation~ ([[#h:annotation-cleanup][Annotation cleanup]]) and ~org-noter-extras-merge-contents-with-next~ ([[#h:merging-annotations][Merging annotations]]).
