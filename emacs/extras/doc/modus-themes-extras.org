This manual describes the features and customization options for the Emacs Lisp file =modus-themes-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=modus-themes-extras.el= extends the =modus-themes= package (Modus Operandi and Modus Vivendi) with automatic theme switching based on the operating system's appearance setting. On macOS, both the Emacs Mac port and Emacs Plus provide mechanisms to detect whether the system is in light or dark mode; this package uses those mechanisms to load the appropriate Modus theme without manual intervention.

The package also provides theme-aware face customizations, particularly for the =highlight-parentheses= package, ensuring that parenthesis highlighting colors stay consistent with the active Modus theme.

The main capabilities are:

- *Automatic theme switching.* Detecting the system appearance and loading Modus Operandi (light) or Modus Vivendi (dark) accordingly, with support for both Emacs Mac and Emacs Plus distributions ([[#h:conditional-theme-loading][Conditional theme loading]]).
- *Theme-aware face customization.* Setting parenthesis highlighting colors from the Modus palette ([[#h:theme-customizations][Theme customizations]]).

* Conditional theme loading
:PROPERTIES:
:CUSTOM_ID: h:conditional-theme-loading
:END:

#+findex: modus-themes-extras-load-theme-conditionally
The function ~modus-themes-extras-load-theme-conditionally~ is the main entry point for automatic theme switching. It queries the Emacs distribution using ~simple-extras-get-emacs-distro~ (from =simple-extras.el=) and sets up the appropriate mechanism:

- For the *Emacs Mac port* (~'emacs-mac~), it immediately calls ~modus-themes-extras-load-theme-emacs-mac~ to read the current system appearance and load the matching theme.
- For *Emacs Plus* (~'emacs-plus~), it adds ~modus-themes-extras-load-theme-emacs-plus~ to the ~ns-system-appearance-change-functions~ hook, which Emacs Plus calls whenever the system appearance changes, and also loads the theme immediately based on the current value of ~ns-system-appearance~. The immediate load ensures the theme is applied even when initialization runs after the hook's initial startup invocation (e.g. on a fresh profile build where elpaca takes time to compile packages).

You would typically call this function once during initialization:

#+begin_src emacs-lisp
(modus-themes-extras-load-theme-conditionally)
#+end_src

** Emacs Mac port
:PROPERTIES:
:CUSTOM_ID: h:emacs-mac
:END:

#+findex: modus-themes-extras-load-theme-emacs-mac
The command ~modus-themes-extras-load-theme-emacs-mac~ reads the current appearance from the =mac-application-state= function. When the appearance is ="NSAppearanceNameAqua"= (light mode), it loads ~modus-operandi~. When it is ="NSAppearanceNameDarkAqua"= (dark mode), it loads ~modus-vivendi~. This command can also be called interactively to force a theme refresh.

** Emacs Plus
:PROPERTIES:
:CUSTOM_ID: h:emacs-plus
:END:

#+findex: modus-themes-extras-load-theme-emacs-plus
The function ~modus-themes-extras-load-theme-emacs-plus~ is designed for the ~ns-system-appearance-change-functions~ hook in Emacs Plus. It receives the new appearance as a symbol (~'light~ or ~'dark~) and loads the corresponding Modus theme. Because this function is triggered by the hook, the theme switch happens automatically whenever the system appearance changes.

** Parenthesis highlighting
:PROPERTIES:
:CUSTOM_ID: h:parenthesis-highlighting
:END:

#+findex: modus-themes-extras-highlight-parentheses
The function ~modus-themes-extras-highlight-parentheses~ configures the =highlight-parentheses= package to use colors from the active Modus theme. It retrieves intense background colors (cyan, magenta, green, yellow) and their corresponding foreground colors from the Modus palette via ~modus-themes-get-color-value~ and sets ~highlight-parentheses-background-colors~ and ~highlight-parentheses-colors~ accordingly.

If ~global-highlight-parentheses-mode~ is available, the function also enables it. This function is suitable for use in the ~modus-themes-after-load-theme-hook~ or any theme-switching hook to keep parenthesis colors synchronized with the active theme.

** Extra face customization
:PROPERTIES:
:CUSTOM_ID: h:extra-faces
:END:

#+findex: modus-themes-extras-set-faces
The function ~modus-themes-extras-set-faces~ is provided as a hook point for additional theme-specific face customizations. Its body is currently empty, but its docstring shows how to add conditional face settings based on the active theme:

#+begin_src emacs-lisp
(if (eq 'modus-operandi (car custom-enabled-themes))
    (set-face-attribute 'hl-sentence nil :background "#bfefff")
  (set-face-attribute 'hl-sentence nil :background "#004065"))
#+end_src

You can add such expressions to this function and call it from a theme-loading hook to apply per-theme face adjustments.
