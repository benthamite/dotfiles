This manual describes the features and customization options for the Emacs Lisp file =frame-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=frame-extras.el= extends Emacs' built-in ~frame~ library with commands for resizing and positioning frames on macOS. While Emacs can maximize frames natively, it does not provide convenient commands for half-screen layouts---a common arrangement when you want Emacs on one side and another application on the other. This package fills that gap.

The package also includes utilities for restoring the window divider face and returning focus to Emacs from other macOS applications.

The main capabilities are:

- *Frame sizing.* Commands to maximize the frame or resize it to the left or right half of the screen ([[#h:frame-sizing][Frame sizing and positioning]]).
- *Focus and visual restoration.* Restoring focus to Emacs via AppleScript and resetting the window divider face ([[#h:restoration][Restoration utilities]]).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Frame sizing and positioning
:PROPERTIES:
:CUSTOM_ID: h:frame-sizing
:END:

#+findex: frame-extras-maximize-frame
#+findex: frame-extras-left-half
#+findex: frame-extras-right-half
The command ~frame-extras-maximize-frame~ maximizes the current frame to fill the screen. It works by clearing any fullscreen state and then setting the ~fullscreen~ frame parameter to ~'maximized~. After a brief delay (to allow the window manager to finish resizing), it records the frame width in ~frame-extras-maximized-frame-width~ for use by the half-screen commands.

The command ~frame-extras-left-half~ resizes the frame to occupy the left half of the screen. It first calls ~frame-extras-maximize-frame~ to determine the full screen width, then sets the frame width to half of ~frame-extras-maximized-frame-width~. The frame remains anchored at the left edge of the screen.

The command ~frame-extras-right-half~ resizes the frame to the right half. It calls ~frame-extras-left-half~ to set the correct width, then repositions the frame so that its left edge aligns with the screen midpoint. Together, ~frame-extras-left-half~ and ~frame-extras-right-half~ provide a quick way to set up a side-by-side layout.

** Restoration utilities
:PROPERTIES:
:CUSTOM_ID: h:restoration
:END:

#+findex: frame-extras-restore-window-divider
#+findex: frame-extras-restore-focus
The command ~frame-extras-restore-window-divider~ resets the ~window-divider~ face to its default attributes using ~custom-set-faces~. This is useful if the divider has been made invisible by a theme or customization and you want to restore its visibility.

The command ~frame-extras-restore-focus~ uses AppleScript (via ~do-applescript~) to bring the Emacs application to the foreground and give it input focus on macOS. This is helpful after an external process has stolen focus, for example after running a shell command that opens another application.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

#+vindex: frame-extras-maximized-frame-width
The variable ~frame-extras-maximized-frame-width~ stores the width (in columns) of the frame after maximization. It defaults to ~244~ but is updated dynamically each time ~frame-extras-maximize-frame~ is called ([[#h:frame-sizing][Frame sizing and positioning]]). The half-screen commands use this value to calculate the correct width for a half-width frame.
