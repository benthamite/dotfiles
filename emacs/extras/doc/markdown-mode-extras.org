This manual describes the features and customization options for the Emacs Lisp file =markdown-mode-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=markdown-mode-extras.el= extends =markdown-mode= with commands for
clipboard conversion, section manipulation, and link editing.  Much of
the package revolves around interoperability between Markdown and Org
mode, using Pandoc as the conversion engine.  If you frequently copy
content between Org and Markdown files, or paste formatted text from
the web into a Markdown buffer, these commands save considerable
manual reformatting.

The main capabilities are:

- *Clipboard conversion*: paste clipboard content while converting
  between Org, Markdown, and HTML formats via Pandoc
  ([[#h:clipboard-conversion][Clipboard conversion]]).
- *Section operations*: extract or copy the body of the current
  Markdown section ([[#h:section-operations][Section operations]]).
- *Link editing*: strip the URL from a Markdown link, keeping only the
  description text ([[#h:link-editing][Link editing]]).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Clipboard conversion
:PROPERTIES:
:CUSTOM_ID: h:clipboard-conversion
:END:

#+findex: markdown-mode-extras-org-paste-dwim
The command ~markdown-mode-extras-org-paste-dwim~ converts clipboard
content between Org and Markdown based on the current major mode.  In
a =markdown-mode= buffer it assumes the clipboard holds Org syntax and
converts to Markdown; in an =org-mode= buffer it does the reverse.
The conversion is performed by piping the system clipboard (=pbv= on
macOS, =xclip= elsewhere) through Pandoc with =--wrap=none=.  A few
post-processing cleanups are applied to the Pandoc output before
yanking.

#+findex: markdown-mode-extras-paste-with-conversion
When you copy formatted text from a web page or other rich-text
source, ~markdown-mode-extras-paste-with-conversion~ converts the
clipboard to GitHub-Flavored Markdown (=gfm-raw_html=) and inserts the
result.  It uses ~simple-extras-pandoc-convert~ from =simple-extras.el=
to detect whether the clipboard contains HTML or Org and to choose the
appropriate Pandoc input format.  For the reverse process (copying
Markdown as formatted HTML), see ~ox-clip-formatted-copy~.

** Section operations
:PROPERTIES:
:CUSTOM_ID: h:section-operations
:END:

#+findex: markdown-mode-extras-copy-section
The command ~markdown-mode-extras-copy-section~ copies the body of the
Markdown section at point to the kill ring and echoes it.  A "section"
is the text between a heading (=^#+ ...=) and the next heading of the
same or higher level.  If called with a non-nil SECTION argument
(available only from Lisp), the command searches for the named section
instead of using the one at point.

Internally, the command calls ~markdown-mode-extras-get-section~
([[#h:get-section][Getting section content]]).

** Link editing
:PROPERTIES:
:CUSTOM_ID: h:link-editing
:END:

#+findex: markdown-mode-extras-remove-url-in-link
When editing a Markdown document you sometimes want to unlink text
while keeping the description.  The command
~markdown-mode-extras-remove-url-in-link~ does exactly that: with
point inside a Markdown link (=[description](url)=), it removes the
link markup and leaves only the description text.

With a prefix argument, the command instead prompts for a /new/ URL
and re-inserts the link with the original description and the
replacement URL.  This is handy for updating broken or outdated links.

The command uses ~markdown-mode-extras-delete-link~
([[#h:delete-link][Deleting a link]]) to remove the original link structure.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Getting section content
:PROPERTIES:
:CUSTOM_ID: h:get-section
:END:

#+findex: markdown-mode-extras-get-section
The function ~markdown-mode-extras-get-section~ returns the body of a
Markdown section as a string.  If the optional SECTION argument (a
heading title) is provided, the function searches the buffer for that
heading; otherwise it uses the section containing point.  Section
boundaries are determined by ~markdown-outline-previous~ and
~markdown-outline-next~.

** Deleting a link
:PROPERTIES:
:CUSTOM_ID: h:delete-link
:END:

#+findex: markdown-mode-extras-delete-link
The function ~markdown-mode-extras-delete-link~ removes the entire
Markdown link at point (=[description](url)=) from the buffer and
returns the link's description text.  It uses ~markdown-link-at-pos~
to locate the link boundaries.  This function is used by
~markdown-mode-extras-remove-url-in-link~ ([[#h:link-editing][Link editing]]).
