This manual describes the features and customization options for the Emacs Lisp file =use-package-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=use-package-extras.el= provides helper macros and a utility function that simplify Emacs configuration when using the =use-package= macro. In configurations where package installation is handled by an external manager such as =elpaca=, you often want =use-package= declarations to configure packages without attempting to install them. This package provides a dedicated macro for that pattern, along with a shorthand for interactive lambdas and a startup-time reporter.

The package depends on =use-package=.

* Macros
:PROPERTIES:
:CUSTOM_ID: h:macros

** Declaring built-in or externally managed packages
:PROPERTIES:
:CUSTOM_ID: h:use-feature
:END:

#+findex: use-feature
When your Emacs configuration uses an external package manager (such as =elpaca=) that handles installation separately, every =use-package= declaration for an already-installed package needs =:ensure nil= to prevent =use-package= from trying to install it again. The macro ~use-feature~ wraps ~use-package~ and automatically injects =:ensure nil=, so you can write:

#+begin_src emacs-lisp
(use-feature my-package
  :init
  (setq my-package-variable t)
  :config
  (my-package-mode 1))
#+end_src

This is equivalent to:

#+begin_src emacs-lisp
(use-package my-package
  :ensure nil
  :init
  (setq my-package-variable t)
  :config
  (my-package-mode 1))
#+end_src

The macro accepts the same NAME and ARGS as ~use-package~. It is also useful for configuring built-in Emacs features that do not need installation at all.

** Creating interactive lambdas
:PROPERTIES:
:CUSTOM_ID: h:lambda-bang
:END:

#+findex: lambda!
The macro ~lambda!~ provides a concise way to create anonymous interactive functions. It wraps the provided BODY forms in ~(lambda () (interactive) ...)~, saving you from writing the boilerplate each time. For example:

#+begin_src emacs-lisp
(lambda! (message "Hello!"))
#+end_src

expands to:

#+begin_src emacs-lisp
(lambda () (interactive) (message "Hello!"))
#+end_src

This is particularly convenient in key-binding definitions and hook setups where a full ~defun~ would be excessive.

** Displaying startup time
:PROPERTIES:
:CUSTOM_ID: h:displaying-startup-time
:END:

#+findex: use-package-extras-display-startup-time
The function ~use-package-extras-display-startup-time~ calculates and displays how long Emacs took to initialize and how many garbage collections occurred during startup. It computes the elapsed time from ~before-init-time~ to ~after-init-time~ and reads the ~gcs-done~ variable, then formats the result into a message like "Emacs loaded in 0.52 seconds with 5 garbage collections."

This function is typically added to ~emacs-startup-hook~ so the information appears in the echo area once initialization is complete. It provides a quick way to monitor whether configuration changes are affecting startup performance.
