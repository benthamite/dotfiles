This manual describes the features and customization options for the Emacs Lisp file =ledger-mode-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=ledger-mode-extras.el= extends =ledger-mode=, the Emacs major mode
for editing plain-text accounting files compatible with the Ledger
command-line tool.  The stock mode covers basic editing and reporting,
but leaves several common workflows---importing transactions from
brokerages, adjusting transaction dates, sorting in reverse
chronological order, and fetching commodity prices---to the user.
This package fills those gaps.

The extensions are organized around six workflows:

- *Importing transactions* from CSV exports (Polymarket, Interactive
  Brokers) and from the system clipboard (Wise Airbnb payouts)
  ([[#h:import][Importing transactions]]).
- *Navigation and editing*: creating entries, aligning amounts,
  copying or killing transactions, and narrowing to a single
  transaction ([[#h:navigation-editing][Navigation and editing]]).
- *Date manipulation*: incrementing or decrementing the date of the
  transaction at point ([[#h:date-manipulation][Date manipulation]]).
- *Reports*: running common Ledger reports with a single command
  ([[#h:reporting][Reporting]]).
- *Price updates*: fetching commodity and cryptocurrency prices via
  external Python scripts ([[#h:price-updates][Price updates]]).
- *Sorting*: sorting transactions chronologically or reverse-
  chronologically, for a region or the whole buffer
  ([[#h:sorting][Sorting]]).

The package depends on =ledger-mode= and =paths= (for file path
variables such as ~paths-file-ledger~ and ~paths-dir-ledger~).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~ledger-mode-extras-currencies~
:PROPERTIES:
:CUSTOM_ID: h:ledger-mode-extras-currencies
:END:

#+vindex: ledger-mode-extras-currencies
The user option ~ledger-mode-extras-currencies~ is a list of currency
symbols (strings) for which price information should be fetched.  This
list is consumed by the external Python scripts invoked by the price
update commands ([[#h:price-updates][Price updates]]).  The type is a list of strings.  The
default value is an empty list.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Importing transactions
:PROPERTIES:
:CUSTOM_ID: h:import
:END:

#+findex: ledger-extras-import-polymarket
The command ~ledger-extras-import-polymarket~ reads a CSV file exported
from Polymarket's transaction history and inserts the corresponding
Ledger transactions at the end of the ledger file.  It prompts for the
CSV file path (defaulting to the downloads directory) and, for each
new token encountered, asks for a short symbol to use in the ledger.
Only "Buy" and "Sell" rows are processed; other transaction types
signal an error.

*Note:* as of early 2025 the Polymarket export functionality appears
broken.  The docstring recommends recording balance changes as capital
gains instead.

#+findex: ledger-extras-import-interactive-brokers
The command ~ledger-extras-import-interactive-brokers~ reads a CSV file
exported from Interactive Brokers' Flex Queries (trade history) and
inserts the corresponding Ledger transactions.  IBKR dates are in
=YYYYMMDD= format and are converted to =YYYY-MM-DD=.  The CSV may
contain summary lines that aggregate multiple trades into one row;
these must be removed manually before import.

#+findex: ledger-mode-extras-import-wise-clipboard-airbnb
The command ~ledger-mode-extras-import-wise-clipboard-airbnb~ imports
Airbnb payout transactions from the system clipboard.  The expected
clipboard format is the text you get by selecting and copying
transactions on the Wise website.  The command parses the clipboard
for amount lines (=+ NNN.NN USD=) and their nearest preceding full
date lines (=DD Month YYYY=), then inserts a Ledger transaction for
each positive amount.  Negative amounts are silently skipped.

A typical clipboard block looks like:

#+begin_example
16 January
Completed
16 January 2026
AIRBNB 4977
+ 200.79 USD
#+end_example

The command reports how many transactions were inserted and how many
negative amounts were skipped.

** Navigation and editing
:PROPERTIES:
:CUSTOM_ID: h:navigation-editing
:END:

#+findex: ledger-mode-extras-new-entry-below
The command ~ledger-mode-extras-new-entry-below~ positions point below
the current transaction, ready for typing a new entry.  It aligns the
current line, moves to the next transaction, and opens a line above
it (via ~crux-smart-open-line-above~).

#+findex: ledger-mode-extras-align-and-next
The command ~ledger-mode-extras-align-and-next~ aligns the amounts in
the current transaction using ~ledger-post-align-xact~ and then
advances point to the beginning of the next transaction or directive.
This is useful as a "save and move on" action during data entry.

#+findex: ledger-mode-extras-kill-transaction-at-point
#+findex: ledger-mode-extras-copy-transaction-at-point
The commands ~ledger-mode-extras-kill-transaction-at-point~ and
~ledger-mode-extras-copy-transaction-at-point~ kill (cut) or copy the
entire transaction at point to the kill ring, respectively.  Both
delegate to ~ledger-mode-extras-copy-or-kill-transaction-at-point~.

#+findex: ledger-mode-extras-narrow-to-xact
The command ~ledger-mode-extras-narrow-to-xact~ narrows the buffer to
show only the current transaction.  Use =C-x n w= (~widen~) to
restore the full view.  This is helpful when you want to focus on a
single transaction without visual clutter.

** Date manipulation
:PROPERTIES:
:CUSTOM_ID: h:date-manipulation
:END:

#+findex: ledger-mode-extras-increase-date-by-one-day
#+findex: ledger-mode-extras-decrease-date-by-one-day
When a transaction was recorded on the wrong date, the commands
~ledger-mode-extras-increase-date-by-one-day~ and
~ledger-mode-extras-decrease-date-by-one-day~ adjust the date of the
transaction at point by plus or minus one day.  They temporarily
narrow to the transaction, parse the existing date, compute the new
date, and replace it in place.  Both commands delegate to
~ledger-mode-extras-increase-date~, which accepts an arbitrary number
of days.

** Reporting
:PROPERTIES:
:CUSTOM_ID: h:reporting
:END:

#+findex: ledger-mode-extras-report-account
#+findex: ledger-mode-extras-report-net-worth
#+findex: ledger-mode-extras-report-net-worth-USD
These commands run named reports from ~ledger-reports~ without
requiring you to navigate the report menu:

- ~ledger-mode-extras-report-account~ runs the ="account"= report.
- ~ledger-mode-extras-report-net-worth~ runs the ="net worth"= report.
- ~ledger-mode-extras-report-net-worth-USD~ runs the ="net worth (USD)"=
  report.

Each calls ~ledger-report~ with the appropriate report name.

** Price updates
:PROPERTIES:
:CUSTOM_ID: h:price-updates
:END:

#+findex: ledger-mode-extras-update-commodities
The command ~ledger-mode-extras-update-commodities~ runs the external
Python script =commodities.py= (located in ~paths-dir-ledger~)
asynchronously.  This script fetches current prices for the
commodities tracked in your ledger.

#+findex: ledger-mode-extras-update-coin-prices
The command ~ledger-mode-extras-update-coin-prices~ runs the external
Python script =coinprices.py= (located in
=paths-dir-ledger/coinprices/=) asynchronously and appends its output
to the price database file (~paths-file-ledger-db~).

** Sorting
:PROPERTIES:
:CUSTOM_ID: h:sorting
:END:

#+findex: ledger-mode-extras-sort-region-or-buffer
#+findex: ledger-mode-extras-sort-region-or-buffer-reversed
The commands ~ledger-mode-extras-sort-region-or-buffer~ and
~ledger-mode-extras-sort-region-or-buffer-reversed~ are the primary
sorting entry points.  If a region is active, they sort the
transactions within it; otherwise they sort the entire buffer.  The
first sorts in chronological order (oldest first, delegating to
~ledger-sort-region~ or ~ledger-sort-buffer~); the second sorts in
reverse chronological order (newest first).

#+findex: ledger-mode-extras-sort-region-reversed
#+findex: ledger-mode-extras-sort-buffer-reversed
The lower-level commands ~ledger-mode-extras-sort-region-reversed~ and
~ledger-mode-extras-sort-buffer-reversed~ perform reverse-chronological
sorting on an explicit region or the whole buffer, respectively.  They
are called by the "or-buffer" variants above and can also be used
directly.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Transaction insertion
:PROPERTIES:
:CUSTOM_ID: h:transaction-insertion
:END:

#+findex: ledger-mode-extras-insert-transaction
The function ~ledger-mode-extras-insert-transaction~ takes a list of
FIELDS (payee, date, account, quantity, token-symbol, price, and
optional fees) and appends a formatted Ledger transaction to FILE
(defaulting to ~paths-file-ledger~).  After inserting the transaction
lines it calls ~ledger-mode-extras-align-and-next~
([[#h:navigation-editing][Navigation and editing]]) to align the amounts.  This function is
used by the import commands ([[#h:import][Importing transactions]]).

** Date conversion
:PROPERTIES:
:CUSTOM_ID: h:date-conversion
:END:

#+findex: ledger-extras-convert-interactive-brokers-date
The function ~ledger-extras-convert-interactive-brokers-date~ converts
an Interactive Brokers date string in =YYYYMMDD= format to the
standard =YYYY-MM-DD= format used in Ledger files.

** Core helpers
:PROPERTIES:
:CUSTOM_ID: h:core-helpers
:END:

#+findex: ledger-mode-extras-copy-or-kill-transaction-at-point
#+findex: ledger-mode-extras-increase-date
The function ~ledger-mode-extras-copy-or-kill-transaction-at-point~
is the shared implementation behind the kill and copy commands
([[#h:navigation-editing][Navigation and editing]]).  It accepts an ACTION argument (~copy~ or
~kill~) and operates on the transaction at point.

The function ~ledger-mode-extras-increase-date~ adjusts the date of
the transaction at point by a given number of DAYS (positive or
negative).  It is used by the one-day increment and decrement commands
([[#h:date-manipulation][Date manipulation]]).
