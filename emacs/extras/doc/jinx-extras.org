This manual describes the features and customization options for the Emacs Lisp file =jinx-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=jinx-extras.el= extends the =jinx= package, a fast spell-checking
framework that uses external checkers such as Hunspell or Aspell.
While Jinx handles detection and correction of misspellings, this
extension adds two conveniences that the base package does not provide:

- *Abbreviation learning*: every time you correct a misspelling
  through Jinx, the typo/correction pair is saved as a
  ~text-mode-abbrev-table~ entry so that Emacs will auto-correct the
  same typo in the future ([[#h:abbreviation-learning][Abbreviation learning]]).
- *Language toggling*: a single command to switch Jinx between English
  and Spanish, useful for bilingual writers
  ([[#h:toggle-languages][Toggling languages]]).

** Toggling languages
:PROPERTIES:
:CUSTOM_ID: h:toggle-languages
:END:

#+findex: jinx-extras-toggle-languages
If you regularly write in two languages, switching Jinx's dictionary
each time can be tedious.  The command ~jinx-extras-toggle-languages~
checks the current value of ~jinx-languages~: if it is ="en"=, the
command switches to ="es"= (Spanish), and vice versa.  It calls the
standard ~jinx-languages~ function to apply the change so that the
buffer is re-checked immediately.

This command assumes a bilingual English/Spanish workflow.  If you work
with different languages, you would need to adjust the toggle targets
in the source.

** Abbreviation learning
:PROPERTIES:
:CUSTOM_ID: h:abbreviation-learning
:END:

When you correct a misspelled word through Jinx's correction
interface, =jinx-extras.el= automatically defines an abbreviation in
~text-mode-abbrev-table~ mapping the misspelled word to its
correction.  From that point on, typing the same misspelling in any
~text-mode~-derived buffer triggers automatic expansion to the correct
word (provided ~abbrev-mode~ is active).

This is implemented via =:before= advice on the internal Jinx
function ~jinx--correct-replace~.  The advice extracts the original
misspelled text from the Jinx overlay, pairs it with the selected
correction, and calls ~define-abbrev~.  A confirmation message is
displayed in the echo area for each new abbreviation.

Over time this builds a personalized typo-correction dictionary that
speeds up writing without any manual configuration.
