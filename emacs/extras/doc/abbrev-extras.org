This manual describes the features and customization options for the Emacs Lisp file =abbrev-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=abbrev-extras.el= extends Emacs' built-in abbreviation facility (=abbrev.el=). The standard =add-mode-abbrev= command requires you to be in the target major mode and to have the expansion text in the buffer. This can be inconvenient when you want to define abbreviations for modes you are not currently using or when the expansion is not present in any buffer.

This package provides an interactive command that lets you select any major mode from a completing-read prompt and type both the abbreviation and its expansion as strings ([[#h:adding-abbreviations][Adding abbreviations]]). It also handles creating the abbrev table if it does not yet exist, and optionally persists the definitions to disk automatically ([[#h:abbrev-extras-write-file-p][~abbrev-extras-write-file-p~]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~abbrev-extras-write-file-p~
:PROPERTIES:
:CUSTOM_ID: h:abbrev-extras-write-file-p
:END:

#+vindex: abbrev-extras-write-file-p
The user option ~abbrev-extras-write-file-p~ controls whether abbreviation definitions are automatically saved to ~abbrev-file-name~ after adding a new mode-specific abbreviation with ~abbrev-extras-add-mode-abbrev~ ([[#h:adding-abbreviations][Adding abbreviations]]).

The default value is ~t~, meaning abbreviations are saved immediately. Set this to ~nil~ if you prefer to accumulate changes in memory and save manually with ~abbrev-extras-write-abbrevs~ ([[#h:writing-abbreviations][Writing abbreviations]]).

The type is a boolean.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Adding abbreviations
:PROPERTIES:
:CUSTOM_ID: h:adding-abbreviations
:END:

#+findex: abbrev-extras-add-mode-abbrev
The command ~abbrev-extras-add-mode-abbrev~ lets you define a mode-specific abbreviation in three steps. It prompts for:

1. A major mode, with completion over all functions ending in =-mode=.
2. The abbreviation string (the short form you will type).
3. The expansion string (the text it should expand to).

The command determines the correct abbrev table name by appending =-abbrev-table= to the selected mode name (for example, ~org-mode-abbrev-table~ for ~org-mode~). If that table does not yet exist, it is created automatically. The abbreviation is then defined using ~define-abbrev~.

When ~abbrev-extras-write-file-p~ is non-nil ([[#h:abbrev-extras-write-file-p][~abbrev-extras-write-file-p~]]), the full set of abbreviation definitions is written to ~abbrev-file-name~ immediately after adding the new entry.

** Writing abbreviations
:PROPERTIES:
:CUSTOM_ID: h:writing-abbreviations
:END:

#+findex: abbrev-extras-write-abbrevs
The command ~abbrev-extras-write-abbrevs~ saves all abbreviation definitions (both global and mode-specific) to the file specified by ~abbrev-file-name~. This is a thin wrapper around ~write-abbrev-file~ that also displays a confirmation message.

Use this command when ~abbrev-extras-write-file-p~ is set to ~nil~ and you want to persist your accumulated changes, or at any time to ensure the abbrev file is up to date.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Abbreviation table management
:PROPERTIES:
:CUSTOM_ID: h:table-management
:END:

#+findex: abbrev-extras-get-sorted-modes
#+findex: abbrev-extras-add-abbrev-to-table
#+findex: abbrev-extras-maybe-create-abbrev-table
Three helper functions support the main command:

- ~abbrev-extras-get-sorted-modes~ scans the obarray for all functions whose names end in =-mode= and returns a sorted list of their names. This provides the completion candidates for ~abbrev-extras-add-mode-abbrev~ ([[#h:adding-abbreviations][Adding abbreviations]]).

- ~abbrev-extras-add-abbrev-to-table~ defines an abbreviation in a given table, creating the table first if needed via ~abbrev-extras-maybe-create-abbrev-table~.

- ~abbrev-extras-maybe-create-abbrev-table~ checks whether a table symbol is bound and, if not, creates an empty abbrev table for it using ~define-abbrev-table~.
