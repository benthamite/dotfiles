This manual describes the features and customization options for the Emacs Lisp file =org-pomodoro-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=org-pomodoro-extras.el= extends =org-pomodoro=, the Emacs implementation of the Pomodoro Technique integrated with Org mode task clocking. The standard =org-pomodoro= package provides timers for work intervals (pomodoros) and breaks, but its modeline display and notification messages are not always ideal.

This package addresses two specific annoyances:

- The default modeline format does not show how many pomodoros have been completed in the current session. The function ~org-extras-pomodoro-format-timer~ enriches the display with a pomodoro count ([[#h:timer-formatting][Timer formatting]]).
- When a pomodoro is killed prematurely, the original notification includes a humorous but distracting message. A patch silences this in favor of a clean, empty notification body ([[#h:killed-notification][Silencing the killed notification]]).

The package depends on =el-patch= for modifying built-in behavior.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Timer formatting
:PROPERTIES:
:CUSTOM_ID: h:timer-formatting
:END:

#+findex: org-extras-pomodoro-format-timer
The function ~org-extras-pomodoro-format-timer~ updates the variable ~org-pomodoro-format~ to include both the remaining time and the current pomodoro count. The resulting modeline display looks like =🍅 24:15|3=, where =24:15= is the time remaining and =3= is the number of pomodoros completed so far.

This function is intended to be called as advice on ~org-pomodoro~ itself, so that the count is refreshed each time a new pomodoro starts. Without this, the count shown in the modeline would be stale.

** Silencing the killed notification
:PROPERTIES:
:CUSTOM_ID: h:killed-notification
:END:

#+findex: org-pomodoro-killed
The function ~org-pomodoro-killed~ is patched via =el-patch= to replace the original notification body ("One does not simply kill a pomodoro!") with an empty string. The notification title ("Pomodoro killed.") is preserved. All other behavior of the original function---resetting the timer, playing the killed sound, handling Org clock state, and running ~org-pomodoro-killed-hook~---remains unchanged.
