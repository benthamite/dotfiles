This manual describes the features and customization options for the Emacs Lisp file =elfeed-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=elfeed-extras.el= extends =elfeed=, the Emacs RSS/Atom feed reader, with convenience commands for reading, filtering, and managing feed entries. The package depends on =elfeed= and =elfeed-show=.

The extensions address several common workflows:

- *Reading in EWW.* Opening feed entries in the built-in EWW browser with automatic readability mode, providing a cleaner reading experience than many raw web pages ([[#h:reading-entries][Reading entries]]).
- *Entry management.* Marking all entries as read, copying entry URLs, and adding entries to a bibliography via =zotra= ([[#h:entry-management][Entry management]]).
- *Filtering.* Toggling between showing all entries and only unread ones, or filtering by specific tags like "wiki" ([[#h:filtering][Filtering]]).
- *Background updates.* Updating feeds even when Elfeed buffers are not open ([[#h:background-update][Background update]]).
- *Follow mode.* Automatically displaying entries as you move through the search buffer, similar to ~org-agenda-follow-mode~ ([[#h:follow-mode][Follow mode]]).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Reading entries
:PROPERTIES:
:CUSTOM_ID: h:reading-entries
:END:

#+findex: elfeed-extras-show-visit-in-eww
Some feed entries are best read directly on the original website rather than in Elfeed's show buffer, but external browsers take you out of Emacs. The command ~elfeed-extras-show-visit-in-eww~ opens the current entry's link in EWW and automatically enables ~eww-readable~ mode once the page finishes loading. This strips away navigation, ads, and other clutter.

The command works in both ~elfeed-show-mode~ (using the displayed entry) and ~elfeed-search-mode~ (using the selected entry). Note that readability mode depends on the website's HTML structure and may not produce good results on all sites. This function is borrowed from Protesilaos Stavrou.

#+findex: elfeed-extras-jump-to-next-link
When reading a long entry with multiple hyperlinks, ~elfeed-extras-jump-to-next-link~ moves point to the next link in the entry and recenters the view. This makes it easy to scan through links without scrolling manually.

** Entry management
:PROPERTIES:
:CUSTOM_ID: h:entry-management
:END:

#+findex: elfeed-extras-mark-all-as-read
When you want to start fresh, ~elfeed-extras-mark-all-as-read~ selects all entries in the search buffer and removes the =unread= tag from all of them. This is a bulk operation that affects every currently displayed entry.

#+findex: elfeed-extras-kill-link-url-of-entry
The command ~elfeed-extras-kill-link-url-of-entry~ copies the URL of the entry displayed in the show buffer to the kill ring. A confirmation message is printed in the echo area. This is useful when you want to share or save a link without opening it externally.

#+findex: elfeed-extras-add-entry
If you use a BibTeX workflow, ~elfeed-extras-add-entry~ passes the current entry's URL to ~zotra-extras-add-entry~ (from =zotra-extras.el=), which handles adding it to your bibliography and downloading associated files. This command is only active in ~elfeed-show-mode~.

** Filtering
:PROPERTIES:
:CUSTOM_ID: h:filtering
:END:

#+findex: elfeed-extras-toggle-read-entries
By default, Elfeed shows only unread entries. The command ~elfeed-extras-toggle-read-entries~ toggles between showing all entries (including read ones) and showing only unread entries. It maintains its state in the variable ~elfeed-extras-toggle-read-entries~.

#+findex: elfeed-extras-toggle-wiki-entries
For feeds that include wiki-tagged entries, ~elfeed-extras-toggle-wiki-entries~ switches between showing only unread wiki entries (=+unread +wiki=) and showing all unread entries except wiki entries (=+unread -wiki=). This is useful when you want to focus on a specific category of content. The state is tracked in the variable ~elfeed-extras-toggle-wiki-entries~.

** Background update
:PROPERTIES:
:CUSTOM_ID: h:background-update
:END:

#+findex: elfeed-extras-update
The standard ~elfeed-update~ command requires Elfeed to be open. The function ~elfeed-extras-update~ first ensures the database is loaded with ~elfeed-db-ensure~ and then triggers the update, allowing it to run from timers, hooks, or scripts without requiring an open Elfeed buffer. A message confirms that the update has started.

* Follow mode
:PROPERTIES:
:CUSTOM_ID: h:follow-mode
:END:

#+findex: elfeed-extras-follow-next
#+findex: elfeed-extras-follow-previous
Inspired by ~org-agenda-follow-mode~, this feature automatically displays entries as you move through the Elfeed search buffer. Rather than selecting an entry to read it, you simply move the cursor and the entry appears in the other window.

The command ~elfeed-extras-follow-next~ moves the cursor down one line and displays the corresponding entry. The command ~elfeed-extras-follow-previous~ does the same upward. After displaying the entry, focus returns to the search buffer so you can continue browsing.

These commands use the internal helper ~elfeed-extras-follow-entry~ for the core logic and ~elfeed-extras-display-buffer~ for displaying the entry buffer without stealing focus.

** Filtering and display helpers
:PROPERTIES:
:CUSTOM_ID: h:filtering-helpers
:END:

#+findex: elfeed-extras-filter-tags
#+findex: elfeed-extras-display-buffer
#+findex: elfeed-extras-follow-entry
#+findex: elfeed-extras-disable-undo
The function ~elfeed-extras-filter-tags~ applies a filter string to the search buffer via ~elfeed-search-set-filter~ and displays a message indicating the active filter.

The function ~elfeed-extras-display-buffer~ shows a buffer in the other window without selecting it, using ~pop-to-buffer~. It is used internally by the follow mode commands ([[#h:follow-mode][Follow mode]]).

The function ~elfeed-extras-follow-entry~ implements the core follow mode logic: it moves by LINES, recenters, displays the entry, and returns focus to the search buffer.

The function ~elfeed-extras-disable-undo~ sets ~buffer-undo-list~ to ~nil~ in the Elfeed search buffer, disabling undo tracking to save memory. It is intended for use in ~elfeed-search-mode-hook~.
