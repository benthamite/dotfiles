This manual describes the features and customization options for the Emacs Lisp file =calendar-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=calendar-extras.el= extends Emacs' built-in ~calendar~ library with
geolocation awareness, date utilities, and visual agenda display.  The
package exists because the stock calendar has no facility for
automatically determining the user's location, and common date
arithmetic tasks---such as finding the last day of the previous month
or enumerating every date in a range---require boilerplate that is
better factored into reusable helpers.

The geolocation subsystem ([[#h:geolocation][Geolocation]]) fetches latitude, longitude,
and city name from an IP-based service and stores them for use by
sunrise/sunset calculations and other location-dependent features.

The date utilities ([[#h:date-utilities][Date utilities]]) cover inserting formatted dates and
generating date ranges.  A separate command ([[#h:block-agenda][Block agenda]]) displays
the current day's Org agenda in the =calfw= visual time-block view.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Geolocation settings
:PROPERTIES:
:CUSTOM_ID: h:geolocation-settings
:END:

#+vindex: calendar-extras-location-name
The user option ~calendar-extras-location-name~ stores the name of the
current location as a string.  You can set it manually, or let
~calendar-extras-set-geolocation~ populate it automatically from an IP
lookup ([[#h:geolocation][Geolocation]]).  The default value is an empty string.

#+vindex: calendar-extras-personal-geolocation
The user option ~calendar-extras-personal-geolocation~ holds an alist
of geolocation data.  Each key is a city name (a string) and each
value is a plist with =:latitude= (float), =:longitude= (float), and
=:timezone= (string).  Like ~calendar-extras-location-name~, this
option is typically set by ~calendar-extras-set-geolocation~ rather
than by hand.  The default value is an empty list.

#+vindex: calendar-extras-use-geolocation
The user option ~calendar-extras-use-geolocation~ is a boolean that
controls whether geolocation features are active.  When non-nil,
~calendar-extras-set-geolocation~ can automatically determine and set
location-related variables.  The default value is ~nil~.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Geolocation
:PROPERTIES:
:CUSTOM_ID: h:geolocation
:END:

#+findex: calendar-extras-set-geolocation
When you move to a different city and want calendar computations
(sunrise/sunset, holidays) to reflect your new location, invoke
~calendar-extras-set-geolocation~.  The command calls the Geoapify
API---using an API key retrieved from =auth-source-pass= under the
entry =tlon/core/api.geoapify.com=---to look up your latitude,
longitude, and city name from your IP address.  It then writes the
result into ~calendar-extras-personal-geolocation~ and
~calendar-extras-location-name~ and confirms the update in the echo
area.

Internally, the command delegates the HTTP request to
~calendar-extras-get-geolocation~ ([[#h:get-geolocation][Fetching geolocation data]]).

** Date utilities
:PROPERTIES:
:CUSTOM_ID: h:date-utilities
:END:

#+findex: calendar-extras-time-last-day-of-last-month
Bookkeeping and reporting tasks often need the last day of the
previous month.  The command ~calendar-extras-time-last-day-of-last-month~
computes that date and inserts it at point in =YYYY-MM-DD= format.  It
handles the January boundary correctly, rolling back to December of
the previous year when necessary.

** Block agenda
:PROPERTIES:
:CUSTOM_ID: h:block-agenda
:END:

#+findex: calendar-extras-calfw-block-agenda
If you use the =calfw= (Calendar Framework) package and prefer a
visual, time-blocked view of your day, the command
~calendar-extras-calfw-block-agenda~ opens a =calfw= buffer in
=block-day= view, sourcing its contents from Org agenda items.  This
provides an at-a-glance alternative to the standard Org agenda
dispatcher.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Fetching geolocation data
:PROPERTIES:
:CUSTOM_ID: h:get-geolocation
:END:

#+findex: calendar-extras-get-geolocation
The function ~calendar-extras-get-geolocation~ performs the HTTP
request to the Geoapify API and parses the JSON response.  It returns
a plist with keys =:lat=, =:lon=, and =:city=.  This function
requires an API key configured via =auth-source-pass=.  It is used by
~calendar-extras-set-geolocation~ ([[#h:geolocation][Geolocation]]) and can also be called
programmatically when you need raw geolocation data without updating
the user options.

** Enumerating date ranges
:PROPERTIES:
:CUSTOM_ID: h:get-dates-in-range
:END:

#+findex: calendar-extras-get-dates-in-range
The function ~calendar-extras-get-dates-in-range~ takes two date
strings (START-DATE and END-DATE, in a format understood by
~date-to-time~, such as =YYYY-MM-DD=) and returns a list of all dates
between them, inclusive, each formatted as =YYYY-MM-DD=.  This is
useful for generating sequences of dates in scripts or batch
operations.
