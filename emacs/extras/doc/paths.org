This manual describes the features and customization options for the Emacs Lisp file =paths.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=paths.el= provides a centralized registry of filesystem paths used across the Emacs configuration. Rather than scattering hard-coded paths through dozens of packages, this file defines every important directory and file location as a ~defcustom~ variable. Other packages reference these variables (e.g. ~paths-dir-notes~, ~paths-file-config~) instead of constructing paths themselves.

This approach has two advantages. First, it makes the configuration portable: when the filesystem layout changes (a new machine, a different Dropbox location, or a different user account), only the variables in =paths.el= need updating. Second, it provides discoverability: all known directories and files are visible in one place, with their relationships made explicit through nesting (e.g. ~paths-dir-music-tango~ is defined relative to ~paths-dir-music~, which is relative to ~paths-dir-google-drive~).

The variables are organized into three groups: directories ([[#h:directory-paths][Directory paths]]), files ([[#h:file-paths][File paths]]), and other identifiers ([[#h:other-identifiers][Other identifiers]]).

** Directory paths
:PROPERTIES:
:CUSTOM_ID: h:directory-paths
:END:

The directory variables define the filesystem tree used by the Emacs configuration. They form a hierarchy where child directories are constructed from parent variables using ~file-name-concat~, so changing a parent automatically adjusts all children.

*** Core directories
:PROPERTIES:
:CUSTOM_ID: h:core-directories
:END:

#+vindex: paths-dir-emacs-profiles
The following variables define the top-level directories from which most other paths derive:

- ~paths-dir-root~ (default: ="/"=): the filesystem root.
- ~paths-dir-downloads~ (default: =~/Downloads/=): the system downloads directory.
- ~paths-dir-system-apps~ (default: =/Applications/=): the macOS applications directory.
- ~paths-dir-dropbox~ (default: =~/Library/CloudStorage/Dropbox/=): the Dropbox sync directory. Many personal data directories are children of this path.
- ~paths-dir-google-drive~ (default: =~/My Drive/=): the Google Drive sync directory. Library and media directories are children of this path.
- ~paths-dir-emacs~ (default: ~user-emacs-directory~): the Emacs configuration directory.
- ~paths-dir-emacs-profiles~ (default: =~/.config/emacs-profiles/=): the directory containing multiple Emacs profiles.

These directories are mostly children of ~paths-dir-dropbox~ and contain personal data. Two exceptions: ~paths-dir-dotfiles~ derives from ~paths-dir-google-drive~, and ~paths-dir-dotemacs~ derives from ~paths-dir-dotfiles~.

- ~paths-dir-dotfiles~: the dotfiles repository (default: =My Drive/dotfiles/=). Child of ~paths-dir-google-drive~.
- ~paths-dir-dotemacs~: the Emacs subdirectory within dotfiles. Its default is user-dependent, resolved at load time based on the =$HOME= environment variable.
- ~paths-dir-notes~, ~paths-dir-journal~, ~paths-dir-wiki~: directories for Org notes, journal entries, and wiki content.
- ~paths-dir-wiki-entries~ and ~paths-dir-wiki-references~: subdirectories of the wiki.
- ~paths-dir-blog~: the blog directory.
- ~paths-dir-anki~: the Anki-related Org files.
- ~paths-dir-archive~ and ~paths-dir-inactive~: directories for archived and inactive content.
- ~paths-dir-android~: files synced with an Android device.
- ~paths-dir-bbdb~, ~paths-dir-people~: contact databases.
- ~paths-dir-ledger~: plain-text accounting files.
- ~paths-dir-bibliographic-notes~: notes attached to bibliographic entries.
- ~paths-dir-private~: private dotfiles not committed to the public repo.
- ~paths-dir-websites~: website source files.
- ~paths-dir-personal-bibliography~: BibTeX bibliography files and related resources.
- ~paths-dir-PW~: the PW project directory.

- ~paths-dir-emacs-var~ and ~paths-dir-emacs-local~: runtime data directories (children of ~paths-dir-emacs~).
- ~paths-dir-init-default~: default target for tangled init files (default: the =develop/= profile under ~paths-dir-emacs-profiles~).

These directories organize Git repositories by provenance:

- ~paths-dir-external-repos~ (default: =~/source/=): third-party repositories.
- ~paths-dir-tlon-repos~ and ~paths-dir-personal-repos~ (both default to =My Drive/repos/=): organization and personal repositories.
- ~paths-dir-tlon-todos~: Org files for organization task tracking.
- ~paths-dir-tlon-docs~, ~paths-dir-babel-refs~, ~paths-dir-translation-server~: specific repositories.
- ~paths-dir-split-git~ (default: =~/git-dirs/=): bare =.git= directories for split-worktree repos.
- ~paths-dir-all-repos~: a list aggregating several personal data directories for use in search and agenda commands.

*** Org directories
:PROPERTIES:
:CUSTOM_ID: h:org-directories
:END:

#+vindex: paths-dir-org
#+vindex: paths-dir-org-roam
The variable ~paths-dir-org~ specifies the root directory for Org files, intended to be used as the value of ~org-directory~. The default is ~paths-dir-dropbox~.

The variable ~paths-dir-org-roam~ specifies the Org-roam directory, intended for ~org-roam-directory~. The default is the same as ~paths-dir-org~.

*** Bibliography files
:PROPERTIES:
:CUSTOM_ID: h:bibliography-files
:END:

#+vindex: paths-file-personal-bibliography-old
#+vindex: paths-file-personal-bibliography-new
#+vindex: paths-files-bibliography-personal

The bibliography variables define BibTeX database files:

- ~paths-file-personal-bibliography-new~ (=new.bib=) and ~paths-file-personal-bibliography-old~ (=old.bib=): the active and legacy bibliography files.
- ~paths-files-bibliography-personal~: a list containing both personal bibliography files, suitable for passing to =ebib= or =citar=.
- ~paths-files-bibliography-all~: a list of all bibliography files, including both personal and organization files. Defaults to an empty list; organization packages append their files at load time.

** Other identifiers
:PROPERTIES:
:CUSTOM_ID: h:other-identifiers
:END:

#+vindex: paths-tlon-todos-generic-id
#+vindex: paths-tlon-todos-jobs-id
The variable ~paths-tlon-todos-generic-id~ holds the Org ID of the heading where generic (non-translation-job) TODOs are stored. The variable ~paths-tlon-todos-jobs-id~ holds the Org ID of the heading where translation job TODOs are stored. Both are UUIDs used by ~org-id-goto~ and related functions to jump directly to the right location.
