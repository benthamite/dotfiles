This manual describes the features and customization options for the Emacs Lisp file =bibtex-completion-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=bibtex-completion-extras.el= provides a targeted extension for the
=bibtex-completion= package, which underlies bibliography completion
interfaces such as =ivy-bibtex= and =helm-bibtex=.  It depends on
=el-patch= for modifying the initialization function.

The sole purpose of this package is to disable the automatic file
watching that =bibtex-completion= sets up during initialization.  By
default, =bibtex-completion= uses ~file-notify-add-watch~ to monitor
bibliography files and automatically reload them when changes are
detected.  This can cause performance issues with large bibliography
files or on systems where file notification events are frequent.

By removing the file-watching code, this package trades automatic
reloading for manual control.  The bibliography data can still be
refreshed on demand by calling ~bibtex-completion-clear-cache~ followed
by any completion command, or simply by restarting Emacs.

* Patched functions

:END:

** Initialization without file watching
:PROPERTIES:
:CUSTOM_ID: h:init-patch
:END:

#+findex: bibtex-completion-init
The function ~bibtex-completion-init~ is patched using =el-patch= to
remove the block of code that sets up file watches on bibliography
files.  The original function iterates over all files returned by
~bibtex-completion-normalize-bibliography~, verifies they exist, and
adds a ~file-notify-add-watch~ for each one.  The patched version skips
this entire loop.

The rest of the initialization---removing old watch descriptors and
pre-calculating display format widths---is preserved unchanged.  This
means existing watches from a previous initialization are still cleaned
up properly, and the completion display continues to work as expected.

The patch is applied at load time using =el-patch-defun=.  If you later
need to re-enable file watching, you can call the original function via
=el-patch-unpatch= or simply unload this package.
