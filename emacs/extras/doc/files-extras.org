This manual describes the features and customization options for the Emacs Lisp file =files-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=files-extras.el= extends the built-in =files.el= library with a broad
set of commands and utilities for managing files and buffers in Emacs.
It depends on the =paths= package for directory and file path constants
and on =transient= for quick-access dispatch menus.

The package addresses several common pain points in daily Emacs usage.
On macOS, it overrides the default trash mechanism to use the external
=trash= utility, which preserves Finder's "Put Back" functionality.  It
adds safeguards around the scratch buffer, preventing accidental kills
by burying it instead.  It also provides an alert system for auto-save
data, ensuring that recovery notifications are not silently missed.

Beyond buffer management, the package integrates with external tools.
The ~files-extras-ocr-pdf~ command drives =ocrmypdf= to add a text
layer to scanned PDFs, supporting language selection and forced re-OCR.
A pair of commands automates downloading and installing the Bypass
Paywalls browser extensions for Chrome and Firefox.

The package also offers two transient dispatch menus: one for jumping to
frequently used files (inbox, config, agenda, ledger, etc.) and one for
opening personal Elpaca-managed packages.  These menus provide a
keyboard-driven launcher for the files you access most often.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~files-extras-new-empty-buffer-major-mode~
:PROPERTIES:
:CUSTOM_ID: h:files-extras-new-empty-buffer-major-mode
:END:

#+vindex: files-extras-new-empty-buffer-major-mode
The user option ~files-extras-new-empty-buffer-major-mode~ specifies the
major mode to activate when creating a new empty buffer with
~files-extras-new-empty-buffer~ ([[#h:creating-buffers][Creating buffers]]).

The default value is ~org-mode~.  It accepts any symbol naming a valid
major mode function.  If set to ~nil~, the new buffer will use
~fundamental-mode~.  You might change this to ~markdown-mode~ or
~text-mode~ if you prefer a different default for scratch-like buffers.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Saving and reverting
:PROPERTIES:
:CUSTOM_ID: h:saving-and-reverting
:END:

#+findex: files-extras-save-and-revert-buffer
#+findex: files-extras-save-all-buffers
#+findex: files-extras-save-buffer-no-hooks
#+findex: files-extras-revert-all-file-buffers
The command ~files-extras-save-and-revert-buffer~ saves the current
buffer and immediately reverts it from disk.  This is useful when an
external process has modified the file and you want to ensure the buffer
reflects both your unsaved changes and the file's current state on disk.

The command ~files-extras-save-all-buffers~ saves all file-visiting
buffers without prompting for confirmation.  It passes a prefix argument
of 4 to ~save-some-buffers~, which bypasses the per-buffer confirmation
that the default behavior would require.

The command ~files-extras-save-buffer-no-hooks~ saves the current buffer
while temporarily disabling both ~before-save-hook~ and
~after-save-hook~.  This is useful when hooks perform operations (such
as formatting or linting) that you want to skip for a particular save.

The command ~files-extras-revert-all-file-buffers~ refreshes every open
file buffer without confirmation.  Buffers with unsaved modifications
are left untouched, but buffers whose underlying files are no longer
readable are killed.  A summary message is displayed when the process
completes.

** Creating buffers
:PROPERTIES:
:CUSTOM_ID: h:creating-buffers
:END:

#+findex: files-extras-new-empty-buffer
#+findex: files-extras-new-buffer-in-current-mode
The command ~files-extras-new-empty-buffer~ creates a new buffer named
=untitled<n>= and activates the major mode specified by the user option
~files-extras-new-empty-buffer-major-mode~ ([[#h:files-extras-new-empty-buffer-major-mode][files-extras-new-empty-buffer-major-mode]]).
The buffer is marked so that ~save-some-buffers~ will offer to save it.

The command ~files-extras-new-buffer-in-current-mode~ creates a new
buffer with the same major mode as the current one.  For ~shell-mode~ and
~eshell-mode~, it calls the appropriate shell launcher instead of simply
setting the mode, ensuring a proper process is attached.

** Switching buffers
:PROPERTIES:
:CUSTOM_ID: h:switching-buffers
:END:

#+findex: files-extras-switch-to-alternate-buffer
The command ~files-extras-switch-to-alternate-buffer~ switches to the
most recently used buffer in the current window, functioning as a quick
toggle between two buffers.  It respects the frame's ~buffer-predicate~
parameter, so it will skip buffers that the frame is configured to
ignore.  Internally it uses ~files-extras-get-alternate-buffer~ to
determine the target ([[#h:buffer-information][Buffer information]]).

** Killing and burying buffers
:PROPERTIES:
:CUSTOM_ID: h:killing-and-burying
:END:

#+findex: files-extras-kill-this-buffer
#+findex: file-extras-kill-other-buffer
#+findex: files-extras-kill-this-buffer-switch-to-other-window
#+findex: files-extras-kill-all-file-visiting-buffers
#+findex: files-extras-bury-buffer-switch-to-other-window
The command ~files-extras-kill-this-buffer~ kills the current buffer.
This is a more reliable alternative to the built-in ~kill-this-buffer~,
which can sometimes stop working due to menu-bar assumptions.

The command ~file-extras-kill-other-buffer~ kills the buffer displayed in
the other window without changing the current window's focus.

The command ~files-extras-kill-this-buffer-switch-to-other-window~ kills
the current buffer and then switches to the last used window via
~window-extras-switch-to-last-window~ (from =window-extras.el=).  This
is useful when you finish reviewing a buffer and want to return to your
previous editing context in a single action.

The command ~files-extras-kill-all-file-visiting-buffers~ kills every
buffer that is visiting a file.  It accepts an optional EXCLUDED-FILES
list (for programmatic use) to preserve specific buffers.

The command ~files-extras-bury-buffer-switch-to-other-window~ buries the
current buffer and switches to the last used window.  This is a softer
alternative to killing: the buffer remains available but is moved to the
bottom of the buffer list.

** Evaluating code
:PROPERTIES:
:CUSTOM_ID: h:evaluating-code
:END:

#+findex: files-extras-eval-region-or-buffer
The command ~files-extras-eval-region-or-buffer~ evaluates the active
region if one is selected, otherwise evaluates the entire buffer.  This
provides a single keybinding for both common evaluation workflows.

** Copying paths and contents
:PROPERTIES:
:CUSTOM_ID: h:copying-paths-and-contents
:END:

#+findex: files-extras-show-buffer-name
#+findex: files-extras-copy-current-path
#+findex: file-extras-copy-contents
#+findex: file-extras-copy-as-kill-dwim
The command ~files-extras-show-buffer-name~ displays the buffer name in
the echo area and copies it to the kill ring.

The command ~files-extras-copy-current-path~ copies the full path of the
current buffer's file (or ~default-directory~ if the buffer is not
visiting a file) to the kill ring.

The command ~file-extras-copy-contents~ copies the entire contents of a
file to the kill ring.  When called interactively, it uses the file at
point in Dired, the current buffer's file, or prompts for a file.

The command ~file-extras-copy-as-kill-dwim~ copies a context-appropriate
string to the kill ring.  Currently, in a ~helpful-mode~ buffer, it
extracts the symbol name from the buffer title and kills the buffer.

** OCR for PDF files
:PROPERTIES:
:CUSTOM_ID: h:ocr-pdf
:END:

#+findex: files-extras-ocr-pdf
The command ~files-extras-ocr-pdf~ applies optical character recognition
to a PDF file using the external =ocrmypdf= tool (which must be
installed separately, e.g., via =brew install ocrmypdf=).

The command determines the file to OCR from context: the file at point
in Dired, the file visited in ~pdf-view-mode~, or the file associated
with the current Ebib or BibTeX entry.  It prompts for the language of
the document, using either Ebib's language field or a general language
selector.

With a prefix argument, the command forces OCR even if the PDF already
contains a text layer.  Without the prefix argument, =ocrmypdf= will
skip files that have already been processed and report accordingly.

The process runs asynchronously.  Status messages are displayed as the
OCR proceeds, with duplicate messages suppressed.  The process filter
~files-extras-ocr-pdf-process-filter~ translates raw =ocrmypdf= output
into user-friendly messages ([[#h:process-support][Process support]]).

** File recovery and diffing
:PROPERTIES:
:CUSTOM_ID: h:file-recovery
:END:

#+findex: files-extras-recover-all-files
#+findex: files-extras-diff-buffer-with-file
The command ~files-extras-recover-all-files~ recovers all files that have
auto-save data in the Emacs auto-save directory.  This is useful after a
crash when ~recover-session~ fails because no session file was created.
After each recovery attempt, it runs ~files-extras-diff-buffer-with-file~
to show the differences.

The command ~files-extras-diff-buffer-with-file~ runs a unified diff
(=-u=) comparing the current buffer against the file on disk.  This
command is also automatically called after ~recover-this-file~ via
=:after= advice, so you always see what changed.

** Image conversion
:PROPERTIES:
:CUSTOM_ID: h:image-conversion
:END:

#+findex: files-extras-convert-image-to-pdf
The command ~files-extras-convert-image-to-pdf~ converts the image file
visited by the current buffer to PDF using the =convert= tool
(ImageMagick).  The resulting PDF is placed in the same directory with
the same base name.

** Removing blank lines
:PROPERTIES:
:CUSTOM_ID: h:removing-blank-lines
:END:

#+findex: file-extras-remove-extra-blank-lines
The command ~file-extras-remove-extra-blank-lines~ collapses consecutive
runs of blank lines into a single blank line throughout the current
buffer.

** Bypass paywalls extensions
:PROPERTIES:
:CUSTOM_ID: h:bypass-paywalls
:END:

#+findex: files-extras-download-bypass-paywalls-chrome
#+findex: files-extras-download-bypass-paywalls-firefox
The command ~files-extras-download-bypass-paywalls-chrome~ downloads the
Bypass Paywalls Chrome Clean extension as a zip file, extracts it, opens
the Chrome extensions management page (via a Keyboard Maestro macro),
and opens the extracted folder in Finder.  The user then drags the
folder onto the extensions page to complete installation.

The command ~files-extras-download-bypass-paywalls-firefox~ performs the
equivalent operation for Firefox, downloading an XPI file and opening
the Firefox extensions page for manual installation.

Both commands verify that the download URL is reachable before
proceeding and will signal an error if it is not.

** Opening external editors
:PROPERTIES:
:CUSTOM_ID: h:external-editors
:END:

#+findex: file-extras-grammarly-open-in-external-editor
The command ~file-extras-grammarly-open-in-external-editor~ opens
Grammarly's web-based external editor in the default browser.

** Buffer-local key bindings
:PROPERTIES:
:CUSTOM_ID: h:buffer-local-keys
:END:

#+findex: files-extras-buffer-local-set-key
The command ~files-extras-buffer-local-set-key~ interactively binds a
key sequence to a command in the current buffer only.  It creates a new
sparse keymap with the current local map as its parent, adds the
binding, and installs the new map.  This is useful for one-off bindings
in specific buffers without affecting the global or mode-level keymap.

** Transient dispatch menus
:PROPERTIES:
:CUSTOM_ID: h:transient-dispatch
:END:

#+findex: files-extras-dispatch
#+findex: files-extras-packages-dispatch
The command ~files-extras-dispatch~ presents a transient menu for
jumping to frequently used files: inbox (mobile and desktop), agenda,
config, feeds, notes, scratch, calendar, ledger, and others.  Each
entry opens the corresponding file or buffer with a single keystroke.

The command ~files-extras-packages-dispatch~ presents a transient menu
for opening personal Elpaca-managed package files, organized into
"Tlon" and "Personal" groups.  It is accessible from the main dispatch
menu via the =p= key.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Buffer information
:PROPERTIES:
:CUSTOM_ID: h:buffer-information
:END:

#+findex: files-extras-get-alternate-buffer
#+findex: files-extras-get-stem-of-current-buffer
#+findex: files-extras-buffer-file-name
#+findex: files-extras-open-buffer-files
#+findex: file-extras-bollp
The function ~files-extras-get-alternate-buffer~ returns the most
recently used buffer in the current window, respecting the frame's
~buffer-predicate~.  It is used internally by
~files-extras-switch-to-alternate-buffer~ ([[#h:switching-buffers][Switching buffers]]).

The function ~files-extras-get-stem-of-current-buffer~ returns the base
name (without extension or directory) of the current buffer's file.

The function ~files-extras-buffer-file-name~ returns the file name of
the current buffer, adjusting paths that go through the =git-dirs=
directory structure to point to their canonical locations.

The function ~files-extras-open-buffer-files~ returns a list of all
=.org= files currently open in Emacs buffers.

The function ~file-extras-bollp~ returns ~t~ if point is at the
beginning of the last line in the buffer.

** File operations
:PROPERTIES:
:CUSTOM_ID: h:file-operations
:END:

#+findex: files-extras-read-file
#+findex: files-extras-newest-file
#+findex: files-extras-lines-to-list
#+findex: files-extras-list-to-lines
#+findex: files-extras-get-help-file
#+findex: files-extras-get-nth-directory
#+findex: files-extras-get-current-dir-lowercased
The function ~files-extras-read-file~ prompts for a file path with a
context-aware default: the file at point in Dired, the current buffer's
file, or an explicit FILE argument.

The function ~files-extras-newest-file~ returns the path of the most
recently modified file in a directory, excluding =.DS_Store= and
=.localized= files (common macOS clutter).

The function ~files-extras-lines-to-list~ reads a file and returns its
lines as a list of strings.  The function ~files-extras-list-to-lines~
performs the inverse operation, writing a list to a file with one element
per line.

The function ~files-extras-get-help-file~ searches for a documentation
file (=.org= or =.md=) corresponding to the given source file, looking
in =doc/= and =docs/= subdirectories.

The function ~files-extras-get-nth-directory~ extracts the Nth path
component from a directory path.

The function ~files-extras-get-current-dir-lowercased~ returns the base
name of the current directory, lowercased, with hyphens replaced by
underscores.

** Package management
:PROPERTIES:
:CUSTOM_ID: h:package-management
:END:

#+findex: files-extras-open-elpaca-package
#+findex: files-extras-open-extras-package
The function ~files-extras-open-elpaca-package~ opens the main Elisp
file for a named Elpaca-managed package by constructing the path within
the Elpaca repos directory.

The function ~files-extras-open-extras-package~ prompts (with
completion) for a file in the =emacs/extras/= directory and opens it.

** Process support
:PROPERTIES:
:CUSTOM_ID: h:process-support
:END:

#+findex: files-extras-ocr-pdf-process-filter
The function ~files-extras-ocr-pdf-process-filter~ is a process filter
for =ocrmypdf= output.  It translates raw process output into
human-readable status messages and suppresses duplicates.  For example,
it informs the user when OCR has already been performed and a prefix
argument is needed to force re-processing.

** Trash and auto-save support
:PROPERTIES:
:CUSTOM_ID: h:trash-and-autosave
:END:

#+findex: files-extras-system-move-file-to-trash
#+findex: files-extras-bury-scratch-buffer
#+findex: files-extras-auto-save-alert
#+findex: files-extras-make-hashed-auto-save-file-name-a
#+findex: files-extras-make-hashed-backup-file-name-a
The function ~files-extras-system-move-file-to-trash~ overrides
~system-move-file-to-trash~ (via =:override= advice) to use the
external =trash= utility on macOS.  This preserves Finder's "Put Back"
functionality, which the default Emacs trash implementation does not
support.

The function ~files-extras-bury-scratch-buffer~ is added to
~kill-buffer-query-functions~.  It intercepts attempts to kill the
=*scratch*= buffer and buries it instead, returning ~nil~ to prevent the
kill.

The function ~files-extras-auto-save-alert~ is added to
~find-file-hook~.  When auto-save data is newer than the visited file,
it triggers a high-severity alert (via the =alert= package) to ensure
the notification is not missed.

The function ~files-extras-make-hashed-auto-save-file-name-a~ advises
~make-auto-save-file-name~ (=:around=) to hash the buffer file name,
preventing excessively long auto-save file paths.  Similarly,
~files-extras-make-hashed-backup-file-name-a~ advises
~make-backup-file-name-1~ to hash backup file names in directories
specified by ~backup-directory-alist~.
