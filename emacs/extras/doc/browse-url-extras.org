This manual describes the features and customization options for the Emacs Lisp file =browse-url-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=browse-url-extras.el= extends Emacs' built-in =browse-url= library with fine-grained control over which browser opens which URLs. By default, Emacs sends all URLs to the same browser function, but many users prefer to open certain domains in their system browser or in Firefox while keeping everything else in EWW. This package makes that easy by reading domain lists from plain text files and configuring ~browse-url-handlers~ accordingly.

The package depends on =browse-url=, =eww=, =paths=, =f=, and =s=.

The main capabilities are:

- *Per-domain browser routing.* Maintaining lists of domains that should open in the default system browser or in Firefox, loaded from configuration files ([[#h:browser-routing][Browser routing]]).
- *Interactive domain management.* Adding the current page's domain to an external browser list with a single command ([[#h:add-domain][Adding domains]]).
- *External browsing.* Opening the current EWW or xwidget-webkit URL in an external browser, or opening URLs in the background ([[#h:external-browsing][External browsing]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~browse-url-extras-browse-url-default-file~
:PROPERTIES:
:CUSTOM_ID: h:browse-url-extras-browse-url-default-file
:END:

#+vindex: browse-url-extras-browse-url-default-file
The user option ~browse-url-extras-browse-url-default-file~ specifies the path to a plain text file listing domains (one per line) that should be opened with the system's default browser via ~browse-url-default-browser~. When you encounter a domain that EWW handles poorly, you can add it to this file.

The default value points to =etc/browse-url-default.txt= in the Emacs configuration directory (via ~paths-dir-dotemacs~). The type is a file path.

** ~browse-url-extras-browse-url-firefox-file~
:PROPERTIES:
:CUSTOM_ID: h:browse-url-extras-browse-url-firefox-file
:END:

#+vindex: browse-url-extras-browse-url-firefox-file
The user option ~browse-url-extras-browse-url-firefox-file~ specifies the path to a plain text file listing domains that should be opened specifically in Firefox via ~browse-url-firefox~. This is useful for domains that require Firefox-specific features or extensions.

The default value points to =etc/browse-url-firefox.txt= in the Emacs configuration directory. The type is a file path.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Browser routing
:PROPERTIES:
:CUSTOM_ID: h:browser-routing
:END:

#+findex: browse-url-extras-set-domains-to-open-externally
The command ~browse-url-extras-set-domains-to-open-externally~ reads the domain lists from ~browse-url-extras-browse-url-default-file~ and ~browse-url-extras-browse-url-firefox-file~ ([[#h:user-options][User options]]) and configures ~browse-url-handlers~ so that matching URLs are routed to the appropriate external browser. It clears any existing handlers before rebuilding the list.

This command is called automatically when =browse-url-extras.el= is loaded and after adding a new domain with ~browse-url-extras-add-domain-to-open-externally~ ([[#h:add-domain][Adding domains]]).

** Adding domains
:PROPERTIES:
:CUSTOM_ID: h:add-domain
:END:

#+findex: browse-url-extras-add-domain-to-open-externally
The command ~browse-url-extras-add-domain-to-open-externally~ provides a convenient way to add a domain to one of the external browser lists without leaving Emacs. It determines a candidate domain by checking the current EWW URL or the first item in the kill ring, then prompts for confirmation or editing.

By default, the domain is added to the default browser list (~browse-url-extras-browse-url-default-file~). With a prefix argument, it is added to the Firefox list (~browse-url-extras-browse-url-firefox-file~) instead. After writing the domain to the file, the command refreshes ~browse-url-handlers~ via ~browse-url-extras-set-domains-to-open-externally~ ([[#h:browser-routing][Browser routing]]) and opens the original URL in the newly configured browser.

** External browsing
:PROPERTIES:
:CUSTOM_ID: h:external-browsing
:END:

#+findex: browse-url-extras-of-dired-file-externally
#+findex: browse-extras-browse-url-externally
#+findex: browse-url-open-externally-in-background
The command ~browse-url-extras-of-dired-file-externally~ opens the file at point in a Dired buffer using the system's default external browser. It temporarily overrides ~browse-url-browser-function~ with ~browse-url-default-browser~ and delegates to the standard ~browse-url-of-dired-file~. This is useful for HTML files that you want to view in a real browser rather than EWW.

The command ~browse-extras-browse-url-externally~ opens the URL of the current EWW page or xwidget-webkit session in an external browser, using the function specified by ~browse-url-secondary-browser-function~. You can also pass a URL argument directly. This is handy when you are reading a page in EWW and realize it requires a full browser for JavaScript-heavy content or forms.

The command ~browse-url-open-externally-in-background~ opens a URL in Firefox on macOS without bringing it to the foreground, using the =open -a Firefox --background= shell command. When called interactively, it prompts for a URL. This is useful for queuing pages to read later without disrupting your current workflow.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Configuration helpers
:PROPERTIES:
:CUSTOM_ID: h:configuration-helpers
:END:

#+findex: browse-url-extras-set-handler
#+findex: browse-url-extras-write-url-to-file
The function ~browse-url-extras-set-handler~ reads a URLS-FILE (one domain per line) and adds entries to ~browse-url-handlers~ mapping each domain (as a quoted regexp) to the specified HANDLER function. It is called by ~browse-url-extras-set-domains-to-open-externally~ ([[#h:browser-routing][Browser routing]]) for each browser configuration file.

The function ~browse-url-extras-write-url-to-file~ appends a URL string to a FILE, then cleans up the file by removing blank lines, eliminating duplicates, and sorting alphabetically. It is used by ~browse-url-extras-add-domain-to-open-externally~ ([[#h:add-domain][Adding domains]]) to maintain tidy domain lists.
