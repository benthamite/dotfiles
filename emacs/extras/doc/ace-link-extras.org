This manual describes the features and customization options for the Emacs Lisp file =ace-link-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=ace-link-extras.el= extends =ace-link=, the package that uses =avy= to let you jump to and open visible links with a few keystrokes. While =ace-link= provides mode-specific commands for common contexts (Org, EWW, Info, etc.), it does not handle all the ways links may appear---for example, HTML emails rendered by =w3m=, or the backlink sidebar of =org-roam=.

This package adds commands for four contexts:

- Opening links in =mu4e= message buffers, with special handling for the =w3m= HTML renderer ([[#h:mu4e-link-opening][mu4e link opening]]).
- Opening EWW links in an external browser or a new buffer ([[#h:eww-link-actions][EWW link actions]]).
- Following links in =org-roam-mode= backlink buffers ([[#h:org-roam-link-following][Org-roam link following]]).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** mu4e link opening
:PROPERTIES:
:CUSTOM_ID: h:mu4e-link-opening
:END:

#+findex: ace-link-extras-mu4e
When reading email in =mu4e-view-mode=, the standard ~ace-link-mu4e~ command works well for most HTML renderers. However, if ~mm-text-html-renderer~ is set to ~w3m~, the link collection mechanism differs: links are stored as =w3m= text properties rather than =shr= overlays.

The command ~ace-link-extras-mu4e~ detects which renderer is active and adapts accordingly. When =w3m= is the renderer, it collects links using ~ace-link--w3m-collect~, lets you select one via =avy=, and then opens the selected URL with ~browse-url~. This means the link is opened according to your ~browse-url-handlers~ configuration (which may route it to =eww=, Chrome, Firefox, or another browser) rather than being opened inside =w3m= itself.

When any other renderer is active, the command falls back to the standard ~ace-link-mu4e~ behavior.

** EWW link actions
:PROPERTIES:
:CUSTOM_ID: h:eww-link-actions
:END:

#+findex: ace-link-extras-eww-externally
#+findex: ace-link-extras-eww-new-buffer
The standard ~ace-link-eww~ command opens the selected link in the current =eww= buffer. Two additional commands provide alternative targets:

- ~ace-link-extras-eww-externally~ opens the selected link using the browser specified by ~browse-url-secondary-browser-function~. Internally, it passes a prefix argument of =(4)= to ~ace-link-eww~. Use this when you want to view a page in a full graphical browser without leaving your =eww= session.

- ~ace-link-extras-eww-new-buffer~ opens the selected link in a new =eww= buffer, preserving the current page. Internally, it passes a prefix argument of =(16)= to ~ace-link-eww~. This is useful when you want to keep multiple pages open for comparison.

** Org-roam link following
:PROPERTIES:
:CUSTOM_ID: h:org-roam-link-following
:END:

#+findex: ace-link-extras-org-roam
The =org-roam= backlink sidebar (=org-roam-mode= buffers) displays Org links that are not in a standard Org buffer, so ~ace-link-org~ alone does not handle the navigation correctly. The command ~ace-link-extras-org-roam~ uses ~ace-link--org-collect~ to gather link positions and =avy= to let you select one, then calls ~org-roam-preview-visit~ to navigate to the linked node.

* Functions

** Link extraction
:PROPERTIES:
:CUSTOM_ID: h:link-extraction
:END:

#+findex: ace-link-extras-w3m-get-url
The function ~ace-link-extras-w3m-get-url~ extracts a URL from a =w3m=-rendered buffer. It uses =avy= to let you select a link from those collected by ~ace-link--w3m-collect~, then returns the URL string from the =w3m-href-anchor= text property at the selected position. This function is called internally by ~ace-link-extras-mu4e~ ([[#h:mu4e-link-opening][mu4e link opening]]).
