This manual describes the features and customization options for the Emacs Lisp file =pdf-tools-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=pdf-tools-extras.el= extends the =pdf-tools= package with commands and functions that improve everyday PDF workflows in Emacs. The built-in =pdf-tools= is excellent for viewing and annotating PDFs, but certain operations---opening a PDF in an external viewer at the current page, extracting all text, adapting the display to the active theme---require additional code. This package fills those gaps.

The main areas of functionality are:

- PDF conversion: extracting text from PDFs via =pdftotext= ([[#h:pdf-conversion][PDF conversion]]).
- Theme integration: automatically toggling midnight mode to match the active Modus theme ([[#h:theme-integration][Theme integration]]).
- Text operations: counting words, copying text with DWIM behavior, and enhanced highlight annotations ([[#h:text-operations][Text operations]]).
- External interaction: opening the PDF in an external viewer at the current page, toggling =writeroom-mode=, deleting the file ([[#h:external-interaction][External interaction]]).
- Ebib integration: jumping from a PDF to the corresponding BibTeX entry ([[#h:ebib-integration][Ebib integration]]).
- Word lookup via double-click ([[#h:word-lookup][Word lookup]]).

The package depends on =pdf-tools= and =johnson=, and optionally integrates with =modus-themes=, =writeroom-mode=, and =ebib-extras.el=.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Text operations
:PROPERTIES:
:CUSTOM_ID: h:text-operations
:END:

#+findex: pdf-tools-extras-count-words
When you need a quick word count for a PDF, the command ~pdf-tools-extras-count-words~ extracts the text with the external =pdftotext= utility, pipes it through =wc -w=, copies the result to the kill ring, and displays it in the echo area. This is handy for checking the length of a paper or report without leaving Emacs.

#+findex: pdf-tools-extras-copy-dwim
The command ~pdf-tools-extras-copy-dwim~ copies PDF text to the kill ring with do-what-I-mean behavior. If a region is active (selected with the mouse or keyboard in the PDF view), it copies only the selected text via ~pdf-view-kill-ring-save~. If no region is active, it extracts the entire document text using ~pdf-tools-extras-convert-pdf~, joins lines that were broken mid-sentence, and copies the result. A message confirms that the full text was copied.

#+findex: pdf-annot-extras-add-highlight-markup-annotation
Highlighting a passage in a PDF and then separately copying the text is a two-step operation in vanilla =pdf-tools=. The command ~pdf-annot-extras-add-highlight-markup-annotation~ combines both steps: it creates a highlight annotation on the active region and simultaneously copies the highlighted text to the kill ring. It accepts optional COLOR and PROPERTY-ALIST arguments, just like the underlying ~pdf-annot-add-markup-annotation~.

** External interaction
:PROPERTIES:
:CUSTOM_ID: h:external-interaction
:END:

#+findex: pdf-tools-extras-open-externally
Sometimes you need to view a PDF in a dedicated external application, for example to use features not available in Emacs. The command ~pdf-tools-extras-open-externally~ opens the current PDF in the system's default viewer. On macOS, if the =opentopage= script is installed in =~/bin=, the PDF opens at the current page. Otherwise it falls back to a plain =open= command.

#+findex: pdf-tools-extras-toggle-writeroom
For distraction-free reading, ~pdf-tools-extras-toggle-writeroom~ toggles =writeroom-mode= with a fixed width of 120 columns and then calls ~pdf-view-fit-height-to-window~ to adjust the PDF display. This provides a clean reading environment without the need to manually configure =writeroom-mode= settings.

#+findex: pdf-tools-extras-delete
The command ~pdf-tools-extras-delete~ permanently deletes the PDF file underlying the current buffer. It asks for confirmation with ~yes-or-no-p~ before proceeding. If confirmed, it deletes the file and kills the buffer. Use this when triaging downloaded papers that you do not wish to keep.

** Ebib integration
:PROPERTIES:
:CUSTOM_ID: h:ebib-integration
:END:

#+findex: pdf-tools-extras-open-in-ebib
If your PDF files are named after BibTeX keys (e.g., =Smith2023.pdf= corresponds to the key =Smith2023=), the command ~pdf-tools-extras-open-in-ebib~ extracts the base name of the current PDF and calls ~ebib-extras-open-key~ to jump to that entry in =ebib=. It signals an error if the current buffer is not in ~pdf-view-mode~. See =ebib-extras.el= for the entry-opening logic.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** PDF conversion
:PROPERTIES:
:CUSTOM_ID: h:pdf-conversion
:END:

#+vindex: pdf-tools-extras-pdftotext-executable
The variable ~pdf-tools-extras-pdftotext-executable~ holds the path to the =pdftotext= executable (default ="pdftotext"=). Customize it if the executable is installed in a non-standard location.

#+findex: pdf-tools-extras-convert-pdf
The function ~pdf-tools-extras-convert-pdf~ converts a PDF file to plain text. It accepts a SOURCE file path and an optional DESTINATION. When DESTINATION is non-nil, the text is written to that file; otherwise, the text is returned as a string. This is the main entry point for PDF text extraction and is used by other packages such as =read-aloud-extras.el=. It has an =;;;###autoload= cookie, so external packages can call it without requiring =pdf-tools-extras= first.

#+findex: pdf-tools-extras--run-pdftotext
Internally, ~pdf-tools-extras--run-pdftotext~ runs the =pdftotext= executable on a SOURCE file, writing to OUTPUT (a file path or ="-"= for stdout). It accepts an optional EXTRA-ARGS list for additional flags such as margin settings. It signals an error if =pdftotext= is not installed.

** Theme integration
:PROPERTIES:
:CUSTOM_ID: h:theme-integration
:END:

#+findex: pdf-tools-extras-apply-theme
Reading a bright-white PDF in a dark Emacs theme is unpleasant, and =pdf-tools= provides ~pdf-view-midnight-minor-mode~ to invert colors. The function ~pdf-tools-extras-apply-theme~ inspects the currently active Modus theme and enables midnight mode if the theme has a dark background, or disables it for a light background. This function is intended to be called from a theme-change hook so that PDFs automatically adapt when you switch themes.

Internally, it calls ~modus-themes-get-current-theme~ and reads the =:background-mode= property from the theme's metadata to determine whether the theme is dark or light.

** Word lookup
:PROPERTIES:
:CUSTOM_ID: h:word-lookup
:END:

#+vindex: pdf-tools-extras-sel-mode-map
#+findex: pdf-tools-extras-lookup-word-at-mouse
The minor mode ~pdf-tools-extras-sel-mode~ binds double-click (=[double-mouse-1]=) to ~pdf-tools-extras-lookup-word-at-mouse~. This command selects the word under the mouse cursor in the PDF, highlights it, and looks it up with ~johnson-lookup~. If no word is found at the click position, it signals a user error.
