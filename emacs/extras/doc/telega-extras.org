This manual describes the features and customization options for the Emacs Lisp file =telega-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=telega-extras.el= extends the =telega= package, an Emacs client for the Telegram messaging service. While =telega= provides a comprehensive Telegram interface, this package adds convenience commands and integrations that streamline daily messaging workflows:

- An intelligent buffer-switching command that cycles through chat, root, and archive buffers ([[#h:switching-to-telega][Switching to telega]]).
- Context-sensitive actions on URLs, buttons, and message input ([[#h:smart-enter][Smart enter]]).
- Capturing Telegram messages into Org mode ([[#h:org-capture-integration][Org capture integration]]).
- Sending files from Dired as Telegram attachments with automatic type detection ([[#h:sending-files-from-dired][Sending files from Dired]]).
- Audio message transcription with optional automatic sharing ([[#h:audio-transcription][Audio transcription]]).
- Quick access to predefined chats and topics via a transient menu ([[#h:chat-and-topic-navigation][Chat and topic navigation]]).

The package depends on =paths= and =telega=.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Auto-sharing audio transcripts
:PROPERTIES:
:CUSTOM_ID: h:auto-sharing-audio-transcripts
:END:

#+vindex: telega-extras-auto-share-audio-transcript
The user option ~telega-extras-auto-share-audio-transcript~ controls whether the transcript of a voice note is automatically posted as a reply after transcription. When set to non-~nil~, and when the transcribed message was sent by you, the transcript is shared as a reply once it becomes available. The default value is ~nil~, meaning transcripts are obtained but not automatically shared.

This option only takes effect when using ~telega-extras-transcribe-audio~ ([[#h:audio-transcription][Audio transcription]]).

** Chat configuration
:PROPERTIES:
:CUSTOM_ID: h:chat-configuration
:END:

#+vindex: telega-extras-chats
The user option ~telega-extras-chats~ stores a plist describing Telegram chats and their topics. Each chat entry includes a name (string), an ID (integer), and optionally a nested plist of topics with their own names and IDs. This data is used by ~telega-extras-open-chat~ and the transient menu ([[#h:chat-and-topic-navigation][Chat and topic navigation]]) to navigate directly to specific chats or topic threads.

The default value is ~nil~. You should populate this option with the chats you access most frequently. The chat and topic IDs can be obtained from the Telegram API or from =telega= itself.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Switching to telega
:PROPERTIES:
:CUSTOM_ID: h:switching-to-telega
:END:

#+findex: telega-extras-switch-to
The command ~telega-extras-switch-to~ provides an intelligent way to navigate the various =telega= buffers with a single key binding. Its behavior depends on the current context:

- If no =telega= buffer exists, it starts =telega=.
- If a =telega= buffer exists but is not the current buffer, it switches to it.
- If the current buffer is a chat buffer and point is not at the end, it moves point to the end (to see the latest messages). If point is already at the end, it kills the chat buffer and returns to the root buffer.
- If the current buffer is the root buffer showing the archive, repeated invocations move point to the top and then reset the filter back to the main view.
- If the current buffer is the root buffer showing the main view, it moves point to the top.

This cycling behavior means a single key binding is sufficient for all common navigation between =telega= buffers.

** Smart enter
:PROPERTIES:
:CUSTOM_ID: h:smart-enter
:END:

#+findex: telega-extras-smart-enter
The command ~telega-extras-smart-enter~ performs a context-dependent action in a =telega= chat buffer. If point is on a URL, it opens the URL with ~browse-url-at-point~. If point is on a button, it activates the button with ~push-button~. Otherwise, it sends the content of the input area as a message via ~telega-chatbuf-input-send~. This provides a unified "do the right thing" key binding for the most common actions in a chat buffer.

** Viewing archive and main chat lists
:PROPERTIES:
:CUSTOM_ID: h:viewing-archive-and-main
:END:

#+findex: telega-extras-view-archive
#+findex: telega-extras-view-main
The command ~telega-extras-view-archive~ resets the active filters in the =telega= root buffer and applies the =archive= filter, showing only archived chats. The command ~telega-extras-view-main~ does the same but applies the =main= filter, restoring the default view. These commands provide direct access to the two primary chat lists without navigating through =telega='s filter interface.

** Org capture integration
:PROPERTIES:
:CUSTOM_ID: h:org-capture-integration
:END:

#+findex: telega-extras-chat-org-capture
The command ~telega-extras-chat-org-capture~ captures the Telegram message at point into an Org mode note. It first stores an Org link to the message with ~org-store-link~ and then invokes ~org-capture~ with the ="n"= template key. This assumes you have an Org capture template bound to ="n"= that is configured to use the stored link. The command is useful for saving important messages or action items from conversations into your Org workflow.

** Sending files from Dired
:PROPERTIES:
:CUSTOM_ID: h:sending-files-from-dired
:END:

#+findex: telega-extras-dired-attach-send
When you have files marked in a Dired buffer that you want to send via Telegram, the command ~telega-extras-dired-attach-send~ handles the entire process. It retrieves the marked files, prompts you to select a destination chat, and attaches each file using the appropriate =telega= attachment function based on the file extension: audio files (=.mp3=, =.flac=) are attached as audio, video files (=.mp4=, =.mkv=) as video, images as photos, and everything else as generic files.

#+findex: telega-extras-chatbuf-attach-most-recent-file
The command ~telega-extras-chatbuf-attach-most-recent-file~ is a convenience for the common case of wanting to send the most recently downloaded file. It finds the newest file in the downloads directory (specified by ~paths-dir-downloads~, using ~files-extras-newest-file~ from =files-extras.el=) and attaches it to the current chat buffer. If the downloads directory is empty, it signals an error.

** Downloading attachments
:PROPERTIES:
:CUSTOM_ID: h:downloading-attachments
:END:

#+findex: telega-extras-download-file
The command ~telega-extras-download-file~ downloads the file attachment from the message at point (or from an optionally provided MESSAGE argument) using ~telega-msg-save~. This provides a quick way to save an attachment without navigating through =telega='s menu system.

** Audio transcription
:PROPERTIES:
:CUSTOM_ID: h:audio-transcription
:END:

#+findex: telega-extras-transcribe-audio
The command ~telega-extras-transcribe-audio~ initiates speech recognition for a voice note message. It checks that the message at point (or the provided MESSAGE argument) is indeed a voice note, then calls the internal ~telega--recognizeSpeech~ function to request transcription from Telegram's servers.

If ~telega-extras-auto-share-audio-transcript~ is non-~nil~ and the message was sent by the current user, the command also sets up a polling mechanism that checks every 5 seconds (for up to 60 seconds) whether the transcript has become available. Once ready, the transcript is automatically posted as a reply to the original voice note ([[#h:auto-sharing-audio-transcripts][Auto-sharing audio transcripts]]).

** Chat and topic navigation
:PROPERTIES:
:CUSTOM_ID: h:chat-and-topic-navigation
:END:

#+findex: telega-extras-menu
The command ~telega-extras-menu~ displays a transient menu for quickly opening predefined chats and topics. The menu entries are derived from the ~telega-extras-chats~ user option ([[#h:chat-configuration][Chat configuration]]). Selecting an item calls ~telega-extras-open-chat~, which starts =telega= if needed, navigates to the specified chat, and optionally opens a topic thread within it.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Tab-bar integration
:PROPERTIES:
:CUSTOM_ID: h:tab-bar-integration
:END:

#+findex: telega-extras-reset-tab-bar
The function ~telega-extras-reset-tab-bar~ schedules a tab-bar reset (via ~tab-bar-extras-quick-reset~ from =tab-bar-extras.el=) one second after it is called. This is intended to be run after =telega= loads, because =telega= may add elements to the tab bar that require a refresh to display correctly.

** Chat and topic lookup
:PROPERTIES:
:CUSTOM_ID: h:chat-and-topic-lookup
:END:

#+findex: telega-extras-open-chat
#+findex: telega-extras-get-id
#+findex: telega-extras-get-last-topic-message
The function ~telega-extras-open-chat~ opens a chat by name and optionally navigates to a topic within it. It looks up the chat and topic IDs from ~telega-extras-chats~ using ~telega-extras-get-id~, then either opens the topic thread (via ~telega-msg-open-thread-or-topic~) or the chat directly (via ~telega-chat-with~).

The function ~telega-extras-get-id~ translates chat and topic names into their numerical IDs by querying the ~telega-extras-chats~ plist. The function ~telega-extras-get-last-topic-message~ retrieves the last message in a given topic thread, which is needed to open the thread view.

** Audio transcript helpers
:PROPERTIES:
:CUSTOM_ID: h:audio-transcript-helpers
:END:

#+findex: telega-extras-get-audio-transcript
#+findex: telega-extras-post-audio-transcript
The function ~telega-extras-get-audio-transcript~ extracts the transcript text from a voice note message if speech recognition has completed. The function ~telega-extras-post-audio-transcript~ formats a transcript and sends it as a reply to the original message. These functions support the transcription workflow described in [[#h:audio-transcription][Audio transcription]].
