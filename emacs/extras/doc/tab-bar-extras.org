This manual describes the features and customization options for the Emacs Lisp file =tab-bar-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=tab-bar-extras.el= extends Emacs' built-in ~tab-bar-mode~ with a rich set of display elements and notification controls for the tab bar's global area (the right-hand side where ~global-mode-string~ is shown). Instead of relying on the mode line for status information, this package moves time, battery, notifications, and other indicators into the tab bar, freeing the mode line for buffer-specific details.

The package integrates with several external packages including =doom-modeline= (for GitHub notifications and debug status), =telega= (for Telegram notifications), =org-pomodoro= (for timer display), =display-wttr= (for weather), and =calendar-extras= (for geolocation).

The main capabilities are:

- *Tab bar elements.* A collection of display elements for time, battery, Emacs profile, notifications, and more ([[#h:tab-bar-elements][Tab bar elements]]).
- *Reset and refresh.* Commands to reset the tab bar display, update geolocation and weather, and restart the clock ([[#h:reset][Resetting the tab bar]]).
- *Notification management.* Commands to toggle the visibility of GitHub and Telega notifications individually or globally ([[#h:notifications][Notification management]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tab-bar-extras-global-mode-string~
:PROPERTIES:
:CUSTOM_ID: h:tab-bar-extras-global-mode-string
:END:

#+vindex: tab-bar-extras-global-mode-string
The user option ~tab-bar-extras-global-mode-string~ defines the list of elements displayed in the tab bar's global area. It should be set to a list containing the desired element constants and ~:eval~ forms from the tab bar elements section ([[#h:tab-bar-elements][Tab bar elements]]). The ~tab-bar-extras-reset~ command ([[#h:reset][Resetting the tab bar]]) applies this value to ~global-mode-string~.

The default value is ~nil~. The type is an alist.

** ~tab-bar-extras-reset-wttr~
:PROPERTIES:
:CUSTOM_ID: h:tab-bar-extras-reset-wttr
:END:

#+vindex: tab-bar-extras-reset-wttr
When set to non-nil, the ~tab-bar-extras-reset~ command ([[#h:reset][Resetting the tab bar]]) will refresh weather information by calling ~display-wttr~, provided that ~display-wttr-mode~ is active. Set this to ~nil~ if you do not use the =display-wttr= package or do not want weather refreshed on each reset.

The default value is ~t~. The type is a boolean.

* Tab bar elements
:PROPERTIES:
:CUSTOM_ID: h:tab-bar-elements
:END:

The package defines a set of constants and evaluated forms that represent individual display elements. You compose these into the ~tab-bar-extras-global-mode-string~ list to build your tab bar layout.

** Basic elements
:PROPERTIES:
:CUSTOM_ID: h:basic-elements
:END:

#+vindex: tab-bar-extras-prefix-element
#+vindex: tab-bar-extras-separator-element
#+vindex: tab-bar-extras-time-element
#+vindex: tab-bar-extras-emacs-profile-element
The constant ~tab-bar-extras-prefix-element~ is a single space, used as the leading separator at the start of the global area.

The constant ~tab-bar-extras-separator-element~ is the string =" | "=, used to separate adjacent elements. Note that conditional elements like the Telega and GitHub notifications include their own leading separator so that it disappears when the element is hidden.

The constant ~tab-bar-extras-time-element~ displays the current time from ~display-time-string~, styled with the ~faces-extras-display-time~ face (from =faces-extras.el=). To change the time format, customize ~display-time-format~.

The constant ~tab-bar-extras-emacs-profile-element~ displays the base name of ~user-emacs-directory~, providing a quick indicator of which Emacs profile is active.

** System status elements
:PROPERTIES:
:CUSTOM_ID: h:system-status-elements
:END:

#+vindex: tab-bar-extras-battery-element
#+vindex: tab-bar-extras-debug-element
The constant ~tab-bar-extras-battery-element~ displays battery status using ~battery-status-function~. It shows a plug icon when charging and a battery icon otherwise, followed by the charge percentage.

The constant ~tab-bar-extras-debug-element~ displays active debugger states (DAP, Edebug, ~debug-on-error~, ~debug-on-quit~) using Doom Modeline helper functions. It only appears when at least one debug mode is active, preceded by a separator.

** Notification elements
:PROPERTIES:
:CUSTOM_ID: h:notification-elements
:END:

#+vindex: tab-bar-extras-telega-element
#+vindex: tab-bar-extras-github-element
#+vindex: tab-bar-extras-pomodoro-element
#+vindex: tab-bar-extras-notification-status-element
The constant ~tab-bar-extras-telega-element~ displays the Telega unread message count from ~telega-mode-line-string~. It is only visible when ~tab-bar-extras-telega-notifications-enabled~ is non-nil, the Telega server is live, and there are unread messages ([[#h:notifications][Notification management]]).

The constant ~tab-bar-extras-github-element~ displays the GitHub notification count from ~doom-modeline--github-notification-number~, using Doom Modeline icons and faces. It requires ~doom-modeline-github~ to be non-nil and is controlled by ~tab-bar-extras-github-notifications-enabled~ ([[#h:notifications][Notification management]]). Counts above 99 are displayed as "99+".

The constant ~tab-bar-extras-pomodoro-element~ ensures the ~org-pomodoro-mode-line~ element is present in ~global-mode-string~, adding it if missing. This keeps the Pomodoro timer visible in the tab bar.

The constant ~tab-bar-extras-notification-status-element~ displays a muted bell icon followed by a separator when ~tab-bar-extras-notifications-enabled~ is ~nil~, serving as a visual reminder that notifications are suppressed.

** Resetting the tab bar
:PROPERTIES:
:CUSTOM_ID: h:reset
:END:

#+findex: tab-bar-extras-reset
#+findex: tab-bar-extras-quick-reset
Over time, the tab bar display can become stale---the clock might freeze, weather data might be outdated, or the global mode string might have been modified by another package. The command ~tab-bar-extras-reset~ performs a full refresh:

1. It restarts the clock with ~display-time~.
2. It reapplies ~tab-bar-extras-global-mode-string~ to ~global-mode-string~.
3. Unless the QUICK argument is non-nil, it updates geolocation via ~calendar-extras-set-geolocation~ (if =calendar-extras= is loaded and ~calendar-extras-use-geolocation~ is set) and sets the weather location accordingly.
4. If ~tab-bar-extras-reset-wttr~ is non-nil and ~display-wttr-mode~ is active, it refreshes the weather display.

The command ~tab-bar-extras-quick-reset~ calls ~tab-bar-extras-reset~ with QUICK set to ~t~, skipping the geolocation and weather steps for a faster refresh.

#+findex: tab-bar-extras-reset-unless-clock
The function ~tab-bar-extras-reset-unless-clock~ calls ~tab-bar-extras-reset~ only when ~org-clock~ is not currently running. This is intended for use in periodic timers: resetting the tab bar would remove the clocked task display, so this function avoids that disruption.

** Notification management
:PROPERTIES:
:CUSTOM_ID: h:notifications
:END:

#+findex: tab-bar-extras-toggle-notifications
#+findex: tab-bar-extras-enable-all-notifications
#+findex: tab-bar-extras-disable-all-notifications
#+findex: tab-bar-extras-toggle-telega-notifications
#+findex: tab-bar-extras-toggle-github-notifications
The command ~tab-bar-extras-toggle-notifications~ toggles all notification elements on or off. It calls the individual toggle functions for GitHub and Telega and updates the master flag ~tab-bar-extras-notifications-enabled~. An optional ACTION argument (~'enable~ or ~'disable~) forces a specific state instead of toggling.

For convenience, ~tab-bar-extras-enable-all-notifications~ and ~tab-bar-extras-disable-all-notifications~ call ~tab-bar-extras-toggle-notifications~ with the corresponding action. These are useful in focus-mode hooks or do-not-disturb workflows.

The functions ~tab-bar-extras-toggle-telega-notifications~ and ~tab-bar-extras-toggle-github-notifications~ control their respective notification elements individually. Each accepts an optional ACTION argument. They update the variables ~tab-bar-extras-telega-notifications-enabled~ and ~tab-bar-extras-github-notifications-enabled~ and display a confirmation message.

** Helper functions
:PROPERTIES:
:CUSTOM_ID: h:helper-functions
:END:

#+findex: tab-bar-extras-set-global-mode-string
#+findex: tab-bar-extras-toggle-individual-notifications
#+findex: tab-bar-extras-get-state
The function ~tab-bar-extras-set-global-mode-string~ sets ~global-mode-string~ to the value of ~tab-bar-extras-global-mode-string~ ([[#h:tab-bar-extras-global-mode-string][User options]]). Messages are suppressed during this operation.

The function ~tab-bar-extras-toggle-individual-notifications~ is the shared implementation behind ~tab-bar-extras-toggle-telega-notifications~ and ~tab-bar-extras-toggle-github-notifications~ ([[#h:notifications][Notification management]]). It updates the given variable and displays a status message.

The function ~tab-bar-extras-get-state~ determines the new boolean state based on an ACTION argument (~'enable~ returns ~t~, ~'disable~ returns ~nil~) or the current value of VAR (toggling it). It is used internally by the notification toggle functions.
