This manual describes the features and customization options for the Emacs Lisp file =org-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=org-extras.el= extends Emacs' built-in Org mode with a substantial
collection of commands and helper functions that address gaps in
day-to-day Org workflows.  Where Org mode provides the foundation for
outlining, task management, clocking, and document authoring,
=org-extras.el= adds convenience commands, DWIM behaviors, and
integration glue that make those workflows smoother and more
keyboard-driven.

The package depends on =el-patch= (for patching several upstream Org
functions), =paths= (for resolving common directory and file paths), and
=transient= (for the navigation menus).  Several commands also rely on
external tools such as =org-roam=, =pandoc=, =pngpaste=, and =gdrive=.

The extensions can be grouped into several thematic areas:

- *Editing and navigation.*  Commands for copying heading names and
  contents, inserting subheadings, counting words, extracting and
  manipulating links, pasting clipboard content as Org markup or images,
  and toggling inline image display ([[#h:editing-and-navigation][Editing and navigation]]).

- *Agenda management.*  Commands for switching to the daily agenda view,
  clocking in from the agenda, marking items done or postponed, toggling
  anniversary display, and DWIM switching between agenda log lines and
  source files ([[#h:agenda-management][Agenda management]]).

- *Clocking and timestamps.*  Commands for inserting clock entries and
  timestamps, jumping to the latest clock entry, inserting clock
  reports, sorting clock tables, deleting headings without logbooks, and
  automatically adding meeting participants when clocking in
  ([[#h:clocking-and-timestamps][Clocking and timestamps]]).

- *Capture hooks.*  A comprehensive hook function that performs
  context-dependent actions when finalizing Org capture templates
  ([[#h:capture-hooks][Capture hooks]]).

- *Folding, cycling, and narrowing.*  Enhanced global cycling that hides
  archives, commands to show properties and logbooks, and narrowing
  commands that respect subtree structure ([[#h:folding-cycling-narrowing][Folding, cycling, and narrowing]]).

- *ID management.*  Automatic addition of Org IDs to headings with
  fine-grained exclusion controls, updating ID locations across the Org
  directory, and a workflow for finding and resolving duplicate IDs
  ([[#h:id-management][ID management]]).

- *Refiling.*  Commands for jumping to the last refiled item, refiling
  and archiving in a single step, refiling to specific positions, and
  regenerating the refile cache ([[#h:refiling][Refiling]]).

- *Links and sorting.*  Sorting lists of Org links, linkifying plain
  text against Org Roam nodes, and copying EWW buffer content with Org
  formatting ([[#h:links-and-sorting][Links and sorting]]).

- *Babel and encryption.*  A confirmation function for selective code
  block evaluation, a tangle command, and a DWIM command that handles
  both decryption and clock-line evaluation ([[#h:babel-and-encryption][Babel and encryption]]).

- *External integration.*  Importing Google Docs via Pandoc, removing
  trailing empty headings in automated note directories
  ([[#h:external-integration][External integration]]).

- *Transient menus.*  Quick-access menus for personal projects, Tlon
  projects, and =config.org= sections ([[#h:transient-menus][Transient menus]]).

- *Patched functions.*  Upstream Org functions patched via =el-patch= to
  improve clock resolution, agenda file handling, and citation insertion
  ([[#h:patched-functions][Patched functions]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Agenda file exclusions
:PROPERTIES:
:CUSTOM_ID: h:agenda-file-exclusions
:END:

#+vindex: org-extras-agenda-files-excluded
The user option ~org-extras-agenda-files-excluded~ holds a list of file
paths to exclude from Org Agenda generation.  Some files lack the
=property= tag that would normally prevent them from appearing in the
agenda, yet they may still be included if they have been modified
recently (for example, via ~vulpea-agenda-files-update~).  Adding them
to this list ensures that their TODOs and time logs do not clutter the
agenda view.

The default value is ~nil~ (no files excluded).  The type is a list of
file paths.

** Automatic ID exclusions
:PROPERTIES:
:CUSTOM_ID: h:automatic-id-exclusions
:END:

#+vindex: org-extras-id-auto-add-excluded-directories
#+vindex: org-extras-id-auto-add-excluded-files
#+vindex: org-extras-id-auto-add-excluded-headings
#+vindex: org-extras-id-auto-add-level-1-dirs
The automatic ID machinery ([[#h:id-management][ID management]]) respects four user
options that control where and how IDs are added.

The user option ~org-extras-id-auto-add-excluded-directories~ is a list
of directory paths.  Files inside these directories will be skipped when
~org-extras-id-auto-add-ids-to-headings-in-file~ runs.  The default
value is ~nil~.

The user option ~org-extras-id-auto-add-excluded-files~ is either a list
of file paths to skip, or ~t~ to disable automatic ID addition globally.
Its default value is a list containing ~paths-file-orb-noter-template~.
When set to a non-nil, non-list value such as ~t~, the function treats
it as "exclude all files".

The user option ~org-extras-id-auto-add-excluded-headings~ is a list of
heading titles (strings) that should never receive an automatic ID.  The
default value is =("Local variables" "COMMENT Local variables" "TODO
Local variables")=.

The user option ~org-extras-id-auto-add-level-1-dirs~ is a list of
directory paths.  For files in these directories, only level-1 headings
receive automatic IDs, rather than headings at all levels.  The default
value includes the =gptel/= and =chatgpt/= subdirectories under the
notes directory.

** BBDB anniversaries heading
:PROPERTIES:
:CUSTOM_ID: h:bbdb-anniversaries-heading
:END:

#+vindex: org-extras-bbdb-anniversaries-heading
The user option ~org-extras-bbdb-anniversaries-heading~ specifies the
Org ID of the heading in =calendar.org= that contains the BBDB
anniversary sexp.  The command ~org-extras-agenda-toggle-anniversaries~
uses this ID to locate and comment or uncomment the anniversary line
([[#h:toggling-anniversaries][Toggling anniversaries]]).

The default value is the string
="2CE5A4E6-842C-4350-ADD0-13D5FC101A20"=.  Set it to ~nil~ to disable
the toggle functionality entirely.

** Clock report parameters
:PROPERTIES:
:CUSTOM_ID: h:clock-report-parameters
:END:

#+vindex: org-extras-clock-report-parameters
The user option ~org-extras-clock-report-parameters~ is a format string
used by ~org-extras-clock-report-insert~ to generate the
=#+BEGIN: clocktable= dynamic block ([[#h:clock-reports][Clock reports]]).  It
contains two =%s= placeholders: the first for the scope (=agenda=,
=file=, or =subtree=), the second for the date range parameters.

The default value is:

#+begin_example
#+BEGIN: clocktable :scope %s :maxlevel 9 :narrow 500! :fileskip0 t %s
#+END:
#+end_example

You might change this to adjust the maximum heading level, column width,
or to add additional clocktable parameters.

** Participant exclusion regexp
:PROPERTIES:
:CUSTOM_ID: h:participant-exclusion-regexp
:END:

#+vindex: org-extras-clock-in-add-participants-exclude
The user option ~org-extras-clock-in-add-participants-exclude~ is a
regexp.  When clocking into a heading whose title matches this regexp,
the automatic participant prompt is suppressed ([[#h:meeting-participants][Meeting participants]]).
This is useful for recurring calendar entries (such as personal focus
blocks) where prompting for participants would be a nuisance.

The default value is ~nil~ (no exclusions).  The type is a regexp
string.

** Babel evaluation languages
:PROPERTIES:
:CUSTOM_ID: h:babel-evaluation-languages
:END:

#+vindex: org-extras-confirm-babel-evaluate-languages
The user option ~org-extras-confirm-babel-evaluate-languages~ is a list
of Org Babel language names (strings) for which code block evaluation
proceeds without confirmation.  The function
~org-extras-confirm-babel-evaluate~ checks this list and returns
non-nil for languages it contains, which tells Org that no confirmation
prompt is needed ([[#h:babel-and-encryption][Babel and encryption]]).

The default value is =("emacs-lisp" "python")=.  Add additional
language names (such as ="shell"= or ="ruby"=) if you trust their
evaluation in your workflow.

** Editing and navigation

*** Setting TODO properties
:PROPERTIES:
:CUSTOM_ID: h:setting-todo-properties
:END:

#+findex: org-extras-set-todo-properties
When triaging tasks, it is common to assign both a priority and an
effort estimate in quick succession.  The command
~org-extras-set-todo-properties~ calls ~org-priority~ followed by
~org-set-effort~, saving a keystroke sequence.

*** Copying URLs and links
:PROPERTIES:
:CUSTOM_ID: h:copying-urls-and-links
:END:

#+findex: org-extras-url-dwim
#+findex: org-extras-link-get-link-at-point
#+findex: org-extras-link-get-url-at-point
#+findex: org-extras-link-get-description-at-point
Org buffers frequently contain both plain URLs and bracketed Org links,
and one often needs to extract a piece of a link for use elsewhere.  The
command ~org-extras-url-dwim~ copies the URL at point to the kill ring,
handling both plain URLs (detected via ~thing-at-point-url-at-point~)
and the target of an Org bracket link.

For more targeted extraction, three additional commands operate on
bracket links: ~org-extras-link-get-link-at-point~ copies the full
bracket link syntax, ~org-extras-link-get-url-at-point~ copies only the
URL portion, and ~org-extras-link-get-description-at-point~ copies only
the description portion.  All three use ~org-extras-link-get-thing-at-point~
internally ([[#h:link-helpers][Link helpers]]).

*** Removing links
:PROPERTIES:
:CUSTOM_ID: h:removing-links
:END:

#+findex: org-extras-remove-link
The command ~org-extras-remove-link~ replaces an Org bracket link at
point with its description text, or with its URL if the link has no
description.  This is handy when you want to keep the display text but
strip the hyperlink markup.

*** Inserting subheadings
:PROPERTIES:
:CUSTOM_ID: h:inserting-subheadings
:END:

#+findex: org-extras-insert-todo-subheading-after-body
#+findex: org-extras-insert-subheading
Org provides ~org-insert-heading~ and ~org-insert-todo-heading~, but
inserting a subheading at exactly the right position often requires
several manual steps.  The command
~org-extras-insert-todo-subheading-after-body~ narrows to the current
subtree, hides its children, moves to the end, and inserts a demoted
TODO heading.  The result is a new TODO subheading at the very end of
the current entry.

The command ~org-extras-insert-subheading~ is similar but inserts a
plain (non-TODO) subheading.  It places the new heading after the
current entry's body but before any existing subheadings, which is
useful when you want to add prose subsections above the task list.

*** Copying heading contents and names
:PROPERTIES:
:CUSTOM_ID: h:copying-heading-contents
:END:

#+findex: org-extras-copy-heading-contents
#+findex: org-extras-copy-heading-name
The command ~org-extras-copy-heading-contents~ copies the body text of
the heading at point (excluding the heading line itself and any child
subtrees) to the kill ring.  It uses ~org-extras-get-heading-contents~
internally ([[#h:heading-content-helpers][Heading content helpers]]).  If the heading has no body
text, a message is displayed instead.

The command ~org-extras-copy-heading-name~ copies the heading title
(without stars, TODO keyword, priority, or tags) to the kill ring.

*** Counting words
:PROPERTIES:
:CUSTOM_ID: h:counting-words
:END:

#+findex: org-extras-count-words
The command ~org-extras-count-words~ counts words in the active region
if one exists, falling back to counting words across the entire current
subtree (including child entries) when no region is active.  In subtree
mode, it skips metadata at the beginning of each entry.  The word count
is copied to the kill ring and a summary message is displayed showing
lines, words, and characters.

*** Jumping to the first heading
:PROPERTIES:
:CUSTOM_ID: h:jumping-to-first-heading
:END:

#+findex: org-extras-jump-to-first-heading
The command ~org-extras-jump-to-first-heading~ widens the buffer and
moves point to the first visible Org heading.  Several other functions
in this package call it internally (for example,
~org-extras-capture-before-finalize-hook-function~).

*** Opening links in EWW
:PROPERTIES:
:CUSTOM_ID: h:opening-links-eww
:END:

#+findex: org-extras-super-return
The command ~org-extras-super-return~ opens the Org link at point using
EWW rather than the default system browser.  It temporarily binds
~browse-url-browser-function~ to ~eww-browse-url~ and clears
~browse-url-handlers~ before calling ~org-open-at-point~.  This is
useful when you want to preview a web page without leaving Emacs.

*** Pasting clipboard content as Org
:PROPERTIES:
:CUSTOM_ID: h:pasting-clipboard-org
:END:

#+findex: org-extras-paste-with-conversion
The command ~org-extras-paste-with-conversion~ converts the contents of
the system clipboard to Org mode format using Pandoc and inserts the
result at point.  If the clipboard contains HTML, it converts from HTML;
otherwise it converts from Markdown.  The command also cleans up stray
backslashes and non-breaking spaces that Pandoc sometimes produces.

Internally it calls ~simple-extras-pandoc-convert~ from =simple-extras.el=.
For the reverse operation (copying Org as formatted HTML), see
~ox-clip-formatted-copy~.  For the Markdown equivalent, see
~markdown-mode-extras-paste-with-conversion~ in =markdown-mode-extras.el=.

*** Pasting images from the clipboard
:PROPERTIES:
:CUSTOM_ID: h:pasting-images
:END:

#+findex: org-extras-paste-image
The command ~org-extras-paste-image~ saves the current clipboard image
to a file and inserts an Org file link at point.  It requires the
=pngpaste= utility (installable via =brew install pngpaste= on macOS).

The image is saved under ~paths-dir-org-images~ with a filename derived
from the current heading's Org ID and an incrementing counter to avoid
collisions.  After saving, the command prompts for an optional caption
and then calls ~org-display-inline-images~ to render the image inline.

*** Toggling inline images
:PROPERTIES:
:CUSTOM_ID: h:toggling-inline-images
:END:

#+findex: org-extras-inline-images
The command ~org-extras-inline-images~ toggles the display of inline
images in the current buffer.  When called interactively without a
prefix argument, it checks whether any inline image overlays already
exist: if they do, it removes them; otherwise it displays all images.

A numeric prefix argument provides explicit control: a negative value
removes images, while a positive value (or ~nil~ from Lisp) displays
them.  The symbol ~toggle~ may also be passed from Lisp for the
toggling behavior.

** Agenda management
:PROPERTIES:
:CUSTOM_ID: h:agenda-management
:END:

*** Switching to today's agenda
:PROPERTIES:
:CUSTOM_ID: h:switching-to-agenda
:END:

#+findex: org-extras-agenda-switch-to-agenda-current-day
The command ~org-extras-agenda-switch-to-agenda-current-day~ opens (or
switches to) the daily agenda in the left window of the frame.  It
calls ~window-extras-split-if-unsplit~ from =window-extras.el= to
ensure the frame is split, then selects window 1 via =winum=.  If the
agenda buffer already exists, it simply switches to it; otherwise it
creates a new agenda view, first enabling anniversary display
([[#h:toggling-anniversaries][Toggling anniversaries]]).

*** Going to an agenda item and clocking in
:PROPERTIES:
:CUSTOM_ID: h:agenda-goto-clock
:END:

#+findex: org-extras-agenda-goto-and-start-clock
The command ~org-extras-agenda-goto-and-start-clock~ navigates from the
Org agenda buffer to the source heading of the item at point and starts
the clock there.  It validates the agenda marker and offers to refresh
the agenda if the marker is stale.  This command must be called from an
~org-agenda-mode~ buffer.

*** Marking done and postponing
:PROPERTIES:
:CUSTOM_ID: h:marking-done-postponing
:END:

#+findex: org-extras-agenda-done-and-next
#+findex: org-extras-agenda-postpone-and-next
Two quick-action commands streamline common agenda operations.  The
command ~org-extras-agenda-done-and-next~ marks the item at point as
DONE and advances to the next line.  The command
~org-extras-agenda-postpone-and-next~ pushes the scheduled date forward
by one day and advances to the next line.  Both are designed for rapid
triage of agenda items.

*** DWIM switching from agenda
:PROPERTIES:
:CUSTOM_ID: h:dwim-switching-agenda
:END:

#+findex: org-extras-agenda-switch-to-dwim
The command ~org-extras-agenda-switch-to-dwim~ adapts its behavior based
on whether the agenda is showing log entries.  When
~org-agenda-show-log~ is active, it calls
~org-clock-convenience-goto-ts~ to jump to the clock timestamp in the
source file.  When log mode is off, it calls ~org-agenda-switch-to~ to
open the heading directly.

*** Toggling anniversaries
:PROPERTIES:
:CUSTOM_ID: h:toggling-anniversaries
:END:

#+findex: org-extras-agenda-toggle-anniversaries
The command ~org-extras-agenda-toggle-anniversaries~ shows or hides BBDB
birthday entries in the agenda.  It works by navigating to the heading
specified by ~org-extras-bbdb-anniversaries-heading~
([[#h:bbdb-anniversaries-heading][BBDB anniversaries heading]]) and toggling the
=%%(org-bbdb-anniversaries-future 1)= sexp line: if the line is present
it is deleted, and if it is absent it is inserted.  Unless the optional
JUST-ENABLE argument is non-nil, the agenda is refreshed afterwards.

This approach of commenting out the diary sexp is the only reliable way
to temporarily suppress anniversaries in the weekly/daily agenda view.

*** Toggling log mode
:PROPERTIES:
:CUSTOM_ID: h:toggling-log-mode
:END:

#+findex: org-extras-agenda-toggle-log-mode
The command ~org-extras-agenda-toggle-log-mode~ toggles
~org-agenda-log-mode~ in clock-check mode, which shows or hides clock
entries in the agenda.

*** Removing agenda highlighting
:PROPERTIES:
:CUSTOM_ID: h:removing-highlighting
:END:

#+findex: org-extras-unhighlight
The command ~org-extras-unhighlight~ is a thin interactive wrapper
around ~org-unhighlight~, which removes search highlighting from the
agenda buffer.

** Clocking and timestamps
:PROPERTIES:
:CUSTOM_ID: h:clocking-and-timestamps
:END:

*** Inserting clock entries
:PROPERTIES:
:CUSTOM_ID: h:inserting-clock-entries
:END:

#+findex: org-extras-new-clock-entry-today
The command ~org-extras-new-clock-entry-today~ inserts a new CLOCK line
under the current heading with today's date.  It prompts for a begin
time and an end time (as strings such as ="09:00"=), then jumps to the
latest existing clock entry via ~org-extras-jump-to-latest-clock-entry~
([[#h:jumping-to-clock-entries][Jumping to clock entries]]), opens a line above it, inserts the
formatted clock range, and evaluates the time range to compute the
duration.

*** Inserting timestamps
:PROPERTIES:
:CUSTOM_ID: h:inserting-timestamps
:END:

#+findex: org-extras-time-stamp-active-current-time
#+findex: org-extras-time-stamp-inactive-current-time
#+findex: org-extras-time-stamp-active-current-date
#+findex: org-extras-time-stamp-inactive-current-date
Four commands insert timestamps with the current date or date-and-time:

- ~org-extras-time-stamp-active-current-time~ inserts an active
  timestamp with date and time, e.g. =<2025-01-15 Wed 14:30>=.
- ~org-extras-time-stamp-inactive-current-time~ inserts an inactive
  timestamp with date and time, e.g. =[2025-01-15 Wed 14:30]=.
- ~org-extras-time-stamp-active-current-date~ inserts an active
  timestamp with date only, e.g. =<2025-01-15 Wed>=.
- ~org-extras-time-stamp-inactive-current-date~ inserts an inactive
  timestamp with date only, e.g. =[2025-01-15 Wed]=.

These are convenient when you need a timestamp at the current moment
without being prompted by the Org date picker.

*** Jumping to clock entries
:PROPERTIES:
:CUSTOM_ID: h:jumping-to-clock-entries
:END:

#+findex: org-extras-jump-to-latest-clock-entry
The command ~org-extras-jump-to-latest-clock-entry~ enables
~visible-mode~, shows the logbook drawer via ~org-extras-show-logbook~
([[#h:showing-drawers][Showing drawers]]), moves back to the current heading, and then
searches forward for the first CLOCK line.  Since clock entries are
stored in reverse chronological order, this effectively jumps to the
most recent clock entry.

*** Clock reports
:PROPERTIES:
:CUSTOM_ID: h:clock-reports
:END:

#+findex: org-extras-clock-report-insert
The command ~org-extras-clock-report-insert~ prompts for a start date,
an end date, and a scope (=agenda=, =file=, or =subtree=), then
inserts a clocktable dynamic block using the format string in
~org-extras-clock-report-parameters~ ([[#h:clock-report-parameters][Clock report parameters]]).
If the start and end dates are the same, a =:block= parameter is used;
otherwise =:tstart= and =:tend= are used.  The clocktable is updated
immediately after insertion.

*** Deleting headings without logbooks
:PROPERTIES:
:CUSTOM_ID: h:deleting-headings-without-logbook
:END:

#+findex: org-extras-delete-headings-without-logbook
The command ~org-extras-delete-headings-without-logbook~ removes every
heading in the current buffer that does not contain a =:LOGBOOK:=
drawer.  It prompts for confirmation before proceeding.  This is useful
for cleaning up files where only clocked entries are of interest.

*** Meeting participants
:PROPERTIES:
:CUSTOM_ID: h:meeting-participants
:END:

#+findex: org-extras-add-participants
#+findex: org-extras-clock-in-add-participants
When clocking into a calendar event (a heading with a =calendar-id=
property), it is useful to record who attended.  The function
~org-extras-clock-in-add-participants~ is installed as =:after= advice
on ~org-clock-in~.  When the heading has a =calendar-id= property and
does not already list participants (checked via
~org-extras-heading-has-participans-p~), it calls
~org-extras-add-participants~.  Headings matching the regexp in
~org-extras-clock-in-add-participants-exclude~ are skipped
([[#h:participant-exclusion-regexp][Participant exclusion regexp]]).

The command ~org-extras-add-participants~ can also be called
interactively.  It presents a ~completing-read-multiple~ prompt
populated with "person" nodes from Org Roam (via
~org-roam-extras-get-people~ in =org-roam-extras.el=), formats the
selected names as Org ID links, and inserts a "Participants: ..." line
after the heading's metadata.

** Font-lock
:PROPERTIES:
:CUSTOM_ID: h:font-lock
:END:

*** Nested source block fontification
:PROPERTIES:
:CUSTOM_ID: h:nested-src-block-fontification
:END:

#+findex: org-extras-fontify-nested-src-blocks
#+findex: org-extras-enable-nested-src-block-fontification
Org's built-in fontification function
~org-fontify-meta-lines-and-blocks-1~ processes each
~#+begin_...#+end_~ pair as a single unit.  When a source block is
nested inside another block (e.g. ~#+begin_details~), the outer block
is fontified first and its entire body is marked with
~font-lock-fontified~, causing the inner ~#+begin_src~ to be skipped.
The function ~org-extras-fontify-nested-src-blocks~ is added to
~font-lock-keywords~ (via ~font-lock-add-keywords~) and runs after the
standard Org keywords.  It finds ~#+begin_src~ lines that were not
fontified with the ~org-block-begin-line~ face and applies the expected
source-block fontification, including native syntax highlighting when
~org-src-fontify-natively~ is non-nil.

To activate this feature in your Org buffers, call
~org-extras-enable-nested-src-block-fontification~.  It appends
~org-extras-fontify-nested-src-blocks~ to the buffer-local
~font-lock-keywords~ and triggers refontification.  A typical setup
adds it to ~org-mode-hook~.

** Folding, cycling, and narrowing
:PROPERTIES:
:CUSTOM_ID: h:folding-cycling-narrowing
:END:

*** Global cycling
:PROPERTIES:
:CUSTOM_ID: h:global-cycling
:END:

#+findex: org-extras-cycle-global
The command ~org-extras-cycle-global~ calls ~org-cycle-global~ and then
hides archived subtrees via ~org-cycle-hide-archived-subtrees~.  Without
this, global cycling would reveal archived content that is normally meant
to stay hidden.  An optional prefix argument ARG works the same way as
in ~org-cycle-global~: =C-u= switches to startup visibility, a numeric
prefix shows all headlines up to that level.

#+findex: org-extras-fold-show-all-headings
The function ~org-extras-fold-show-all-headings~ expands all headings in
the buffer (using ~org-fold-show-all~ with the =headings= argument)
while keeping archived subtrees hidden.  This is not an interactive
command but is designed to be called from hooks or other functions.

*** Showing drawers

#+findex: org-extras-show-properties

*** Narrowing to entries

#+findex: org-extras-narrow-to-entry-no-children
The command ~org-extras-narrow-to-entry-and-children~ narrows the buffer
to the current subtree and then shows the entry and its children while
hiding drawers (via ~org-extras-show-subtree-hide-drawers~).  This gives
a focused view of the heading and all its descendants.

The command ~org-extras-narrow-to-entry-no-children~ narrows the buffer
to the current entry only, excluding any subheadings.  The narrowed
region extends from the beginning of the subtree to the next visible
heading.

** ID management

*** Automatic ID addition
:PROPERTIES:
:CUSTOM_ID: h:automatic-id-addition
:END:

#+findex: org-extras-id-auto-add-ids-to-headings-in-file
The function ~org-extras-id-auto-add-ids-to-headings-in-file~ is
intended to run from a hook (such as ~before-save-hook~).  It adds Org
IDs to all headings in the current file that lack them, subject to
several exclusion conditions:

- The file must be in ~org-mode~ and within ~paths-dir-org~.
- The buffer must not be read-only.
- The buffer-local variable ~org-extras-id-auto-add-exclude-file~ must
  be ~nil~ ([[#h:variables][Variables]]).
- The file must not be inside a directory listed in
  ~org-extras-id-auto-add-excluded-directories~.
- The file must not match ~org-extras-id-auto-add-excluded-files~.
- The current heading must not match
  ~org-extras-id-auto-add-excluded-headings~.
- For files in directories listed in
  ~org-extras-id-auto-add-level-1-dirs~, only level-1 headings receive
  IDs.

See [[#h:automatic-id-exclusions][Automatic ID exclusions]] for details on configuring
these options.

*** Updating ID locations
:PROPERTIES:
:CUSTOM_ID: h:updating-id-locations
:END:

#+findex: org-extras-id-update-id-locations
The command ~org-extras-id-update-id-locations~ scans all =.org= files
under ~org-directory~ (excluding those in =.Trash= directories or with
carriage returns in their names) and updates the ID location database
via ~org-id-update-id-locations~.  If the scan
reports duplicate IDs, the command automatically calls
~org-extras-id-find-duplicate-ids~ ([[#h:finding-duplicate-ids][Finding duplicate IDs]]) so
that you can review and resolve them.

Internally, it temporarily advises ~display-warning~ to intercept
"duplicate IDs found" messages and trigger the duplicate-ID workflow.

*** Finding duplicate IDs
:PROPERTIES:
:CUSTOM_ID: h:finding-duplicate-ids
:END:

#+findex: org-extras-id-find-duplicate-ids
The command ~org-extras-id-find-duplicate-ids~ parses the =*Messages*=
buffer for "Duplicate ID" warnings produced by
~org-id-update-id-locations~ and presents the results in a dedicated
read-only =*Duplicate Org IDs*= buffer.  Each line shows a file path
followed by the duplicate ID.  In this buffer, pressing =SPC= calls
~org-extras-id-process-next-duplicate~ to process the next entry.

*** Processing duplicate IDs
:PROPERTIES:
:CUSTOM_ID: h:processing-duplicate-ids
:END:

#+findex: org-extras-id-process-next-duplicate
The command ~org-extras-id-process-next-duplicate~ is meant to be used
from the =*Duplicate Org IDs*= buffer.  It reads the first line, opens
the associated file, widens the buffer, enables
~simple-extras-visible-mode-enhanced~ to help locate the duplicate,
copies the ID to the kill ring, and removes the processed line from the
buffer.  When all entries have been processed, the buffer is killed
automatically.

** Links and sorting
:PROPERTIES:

*** Sorting links
:PROPERTIES:
:CUSTOM_ID: h:sorting-links
:END:

#+findex: org-extras-sort-links
#+findex: org-extras-sort-keywords
The command ~org-extras-sort-links~ sorts a paragraph of Org links
alphabetically.  It prompts for the separator string used between links
(for example, =", "= or =" | "=), splits the paragraph text on that
separator, sorts the resulting elements using ~org-sort-list~, and
reassembles them.

The command ~org-extras-sort-keywords~ is a convenience wrapper that
calls ~org-extras-sort-links~ with the separator =" . "= (bullet
separator), which is the convention used for keyword lists.

*** Copying EWW content as Org
:PROPERTIES:
:CUSTOM_ID: h:copying-eww-content
:END:

#+findex: org-extras-eww-copy-for-org-mode
The command ~org-extras-eww-copy-for-org-mode~ copies the content of an
EWW buffer (or the active region within it) to the kill ring, converting
HTML formatting to Org syntax.  Links become Org bracket links, bold
text is wrapped in =*asterisks*=, italic text in =/slashes/=, and HTML
bullet characters are replaced with Org list markers (=-=).

This is an enhanced version of ~org-eww-copy-for-org-mode~ from the
=ol-eww= library.
