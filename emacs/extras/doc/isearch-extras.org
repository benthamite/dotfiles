This manual describes the features and customization options for the Emacs Lisp file =isearch-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=isearch-extras.el= extends Emacs' built-in incremental search facility (=isearch=). While isearch is one of the most frequently used Emacs features, its default command set lacks several useful actions: exiting at the other end of the match, copying the match to the kill ring, yanking text literally, and escalating a search to broader scopes.

This package provides two groups of extensions:

- Commands for use during an active isearch session, bound in ~isearch-mode-map~, that add new actions at the point of the match ([[#h:match-actions][Match actions]]) or transfer the search to another tool ([[#h:search-escalation][Search escalation]]).
- An advice function that uses the active region as the initial search string when starting isearch ([[#h:region-as-initial-input][Region as initial input]]).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Match actions
:PROPERTIES:
:CUSTOM_ID: h:match-actions
:END:

#+findex: isearch-extras-exit-other-end
#+findex: isearch-extras-copy-match
#+findex: isearch-extras-yank-kill-literally
Three commands provide additional actions you can take on the current isearch match:

- ~isearch-extras-exit-other-end~ exits the search and moves point to the opposite end of the match. Normally, isearch leaves point at the end of the match when searching forward, or at the beginning when searching backward. This command inverts that behavior, which is useful when you want to edit the text just before a match found by a forward search.

- ~isearch-extras-copy-match~ copies the text of the current match to the kill ring and exits the search. The copied region spans from point to ~isearch-other-end~, so it captures exactly the matched text. This is convenient when you want to use the match as a template for replacement or insertion elsewhere.

- ~isearch-extras-yank-kill-literally~ inserts the most recent kill ring entry into the isearch prompt as literal text, without interpreting regexp metacharacters. The standard isearch yank commands may treat the yanked text as a pattern; this command avoids that by processing each character through ~isearch-text-char-description~.

** Search escalation
:PROPERTIES:
:CUSTOM_ID: h:search-escalation
:END:

#+findex: isearch-extras-consult-line
#+findex: isearch-extras-project-search
Two commands let you seamlessly escalate an in-buffer isearch to a broader search scope:

- ~isearch-extras-consult-line~ transfers the current search string to ~consult-line~ from the =consult= package. It saves the search string to the isearch ring, exits the isearch session, and immediately starts ~consult-line~ with the same query (regexp-quoted if the search was not already in regexp mode). This is useful when you realize mid-search that you want to see all matches at once with preview.

- ~isearch-extras-project-search~ transfers the current search string to ~project-find-regexp~. Like ~isearch-extras-consult-line~, it saves the string, exits isearch, and immediately starts a project-wide search. This lets you escalate from "searching in this buffer" to "searching in all project files" without retyping the query.

Both commands preserve the isearch ring so the query remains available for future searches.

** Region as initial input
:PROPERTIES:
:CUSTOM_ID: h:region-as-initial-input
:END:

#+findex: isearch-extras-use-selection
The function ~isearch-extras-use-selection~ is designed to be added as =:around= advice on ~isearch-forward~ and ~isearch-backward~. When a region is active at the time isearch is invoked, this function takes the selected text, adds it to the isearch ring, deactivates the mark, and then starts the search with an immediate repeat in the appropriate direction. The effect is that the selected text becomes the search string and point moves to the next (or previous) occurrence.

When no region is active, the function delegates to the original isearch command with no changes.

This provides a quick workflow: select a word or phrase, press =C-s=, and immediately jump to the next occurrence without any additional input.
