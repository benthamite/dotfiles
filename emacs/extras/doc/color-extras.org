This manual describes the features and customization options for the Emacs Lisp file =color-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=color-extras.el= extends Emacs' built-in ~color~ library with
practical tools for working with color values in everyday editing.
Theme authors, CSS developers, and anyone who manipulates color codes
in source files will find it useful.

The package provides three main capabilities:

- *Conversion* between hexadecimal (HEX) and Hue-Saturation-Luminance
  (HSL) color representations, either for the color at point or for
  every color in a buffer ([[#h:conversion][Conversion commands]]).
- *Contrast analysis* against white, black, and an optional reference
  color, following the WCAG contrast ratio standard
  ([[#h:contrast-analysis][Contrast analysis]]).
- *Embark integration* that lets you act on any HEX or HSL color
  string with a single keystroke ([[#h:embark-integration][Embark integration]]).

Internally, the package relies on Emacs' built-in ~color-rgb-to-hsl~
and ~color-hsl-to-rgb~ for the mathematical conversions and on the
=ct= package for contrast ratio computation.

** Regular expression patterns
:PROPERTIES:
:CUSTOM_ID: h:regexp-patterns
:END:

#+vindex: color-extras-hex-pattern
The constant ~color-extras-hex-pattern~ is a regular expression that
matches a six-digit hexadecimal RGB color string, with or without a
leading =#=.  The first capture group contains the six hex digits.
This pattern is used throughout the package to detect HEX colors at
point and in buffer-wide searches ([[#h:conversion][Conversion commands]]).

#+vindex: color-extras-hsl-scaled-pattern
The constant ~color-extras-hsl-scaled-pattern~ is a regular expression
that matches an HSL color written as three comma-separated numbers
(e.g., =210, 50%, 40%=).  Hue ranges from 0 to 360; saturation and
luminance range from 0 to 100 and may include trailing =%=  signs.
The three capture groups correspond to hue, saturation, and luminance.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Conversion
:PROPERTIES:
:CUSTOM_ID: h:conversion
:END:

#+findex: color-extras-convert
When you need to switch a color between HEX and HSL notation, place
point on the color value and call ~color-extras-convert~.  If the
color at point is HEX (e.g., =#ff0000=), the command converts it to
HSL (e.g., =0, 100%, 50%=); if it is HSL, the command produces HEX.
The converted value is saved to the kill ring and reported in the echo
area, so you can immediately yank it wherever you need it.  If no
recognizable color is found at point, the command reports this in the
echo area.  You can also pass a COLOR argument from Lisp code.

#+findex: color-extras-convert-all
To convert every color in a file at once, use
~color-extras-convert-all~.  It prompts for the target format---either
="hex"= or ="hsl"=---and replaces all colors matching the /opposite/
format throughout the buffer.  For example, choosing ="hex"= finds all
HSL strings and rewrites them as HEX values.  An optional FILE
argument lets you operate on a file other than the current buffer.

** Contrast analysis
:PROPERTIES:
:CUSTOM_ID: h:contrast-analysis
:END:

#+findex: color-extras-contrast
Verifying that foreground and background colors meet accessibility
guidelines is a frequent need.  The command ~color-extras-contrast~
computes the WCAG contrast ratio between a primary color and both
white (=#ffffff=) and black (=#000000=), displaying the results in the
echo area.  If you supply an optional second color, the command also
reports the contrast ratio between the two.

The primary color defaults to the color at point; if none is found,
the command prompts for one.  The second color is always prompted for
and may be left blank.  This command requires the =ct= package for its
~ct-contrast-ratio~ function.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Parsing and detection
:PROPERTIES:
:CUSTOM_ID: h:parsing-detection
:END:

#+findex: color-extras-parse-color
#+findex: color-extras-looking-at-rgb
#+findex: color-extras-looking-at-hsl-scaled
#+findex: color-extras-looking-at-color
The function ~color-extras-parse-color~ accepts a color string and
returns either a HEX string (without the leading =#=) or an HSL list
of three numbers, depending on which pattern matches.  Pass a non-nil
STRING argument to get the HSL result as a formatted string instead of
a list.

Three companion functions detect colors at point:
~color-extras-looking-at-rgb~ returns the six-digit HEX string (or
~nil~), ~color-extras-looking-at-hsl-scaled~ returns the HSL color as
a list or formatted string, and ~color-extras-looking-at-color~ tries
HEX first and falls back to HSL.  These are the building blocks used
by the interactive commands ([[#h:conversion][Conversion]]).

** HSL formatting
:PROPERTIES:
:CUSTOM_ID: h:hsl-formatting
:END:

#+findex: color-extras-hsl-split
#+findex: color-extras-hsl-to-string
#+findex: color-extras-format-hsl
The function ~color-extras-hsl-split~ extracts hue, saturation, and
luminance from the current regexp match data (assuming a match against
~color-extras-hsl-scaled-pattern~) and returns them as a list of three
numbers.  When the optional STRING argument is non-nil, match data is
read from that string (for use after ~string-match~); otherwise, match
data is read from the current buffer (for use after
~thing-at-point-looking-at~ or ~re-search-forward~).  ~color-extras-hsl-to-string~ takes such a list and formats
it as ="H, S%, L%"=, rounding each component to the nearest integer.
~color-extras-format-hsl~ is a convenience wrapper that returns either
the list or the string form depending on its STRING argument.

** Color space conversions
:PROPERTIES:
:CUSTOM_ID: h:color-space-conversions
:END:

#+findex: color-extras-hex-to-hsl
#+findex: color-extras-hsl-to-hex
#+findex: color-extras-hex-to-rgb
#+findex: color-extras-rgb-to-hex
These four functions form the conversion core of the package.

~color-extras-hex-to-rgb~ converts a HEX string (supporting 3-, 4-,
6-, and 8-digit forms) to a list of three RGB floats in the range
0.0--1.0.  ~color-extras-rgb-to-hex~ does the reverse, accepting
three integers in the range 0--255 and returning a =#RRGGBB= string.

~color-extras-hex-to-hsl~ converts a HEX string to an HSL list.  When
the optional RESCALE argument is non-nil, hue is scaled to 0--360 and
saturation and luminance to 0--100; otherwise all three values are in
the 0.0--1.0 range.  ~color-extras-hsl-to-hex~ performs the inverse
transformation, with the same RESCALE convention.

** Rescaling helpers
:PROPERTIES:
:CUSTOM_ID: h:rescaling-helpers
:END:

#+findex: color-extras-hsl-rescale
#+findex: color-extras-hsl-maybe-rescale
The function ~color-extras-hsl-rescale~ converts HSL values between
the normalized (0.0--1.0) and scaled (0--360 / 0--100) ranges.  The
OPERATION argument should be ~*~ (to go from normalized to scaled) or
~/~ (to go from scaled to normalized).
~color-extras-hsl-maybe-rescale~ is a conditional wrapper that only
rescales when its RESCALE argument is non-nil.

* Embark integration
:PROPERTIES:
:CUSTOM_ID: h:embark-integration
:END:

#+findex: color-extras-embark-color-finder
When the =embark= package is loaded, =color-extras.el= registers a
target finder, ~color-extras-embark-color-finder~, that recognizes HEX
and HSL color strings at point and tags them with the ~color~
category.

A dedicated keymap, ~color-extras-embark-map~, is associated with this
category.  It provides two actions:

- =c= :: ~color-extras-convert~ --- convert between HEX and HSL
  ([[#h:conversion][Conversion]]).
- =w= :: ~embark-copy-as-kill~ --- copy the color string to the kill
  ring.

This means you can invoke =embark-act= with point on any color value
and immediately convert it or copy it.
