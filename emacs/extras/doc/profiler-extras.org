This manual describes the features and customization options for the Emacs Lisp file =profiler-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=profiler-extras.el= extends the built-in Emacs profiler (=profiler.el=) with convenience commands that reduce the number of steps needed for common profiling workflows. The standard profiler requires separate invocations to start, stop, and generate a report. This package provides a single toggle command that manages the entire cycle, as well as a command for quickly expanding all sub-entries in a profiler report.

The package requires only the built-in =profiler= library.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Toggling the profiler
:PROPERTIES:
:CUSTOM_ID: h:toggling-the-profiler
:END:

#+findex: profiler-extras-profiler-toggle
Profiling in Emacs normally involves calling ~profiler-start~, reproducing the behavior you want to measure, then calling ~profiler-stop~ and ~profiler-report~ separately. The command ~profiler-extras-profiler-toggle~ collapses this into a single toggle. On the first invocation it starts the profiler for both CPU and memory (=cpu+mem=). On the second invocation it generates the profiler report and stops the profiler. An internal variable tracks the toggle state.

A typical workflow is to bind this command to a convenient key, press it once to begin profiling, perform the operations you want to measure, and press it again to see the results.

** Expanding report entries
:PROPERTIES:
:CUSTOM_ID: h:expanding-report-entries
:END:

#+findex: profiler-extras-profiler-report-toggle-entry-global
When examining a profiler report, you often want to see the full call tree beneath a particular entry. The built-in ~profiler-report-toggle-entry~ expands one level at a time, which can be tedious for deep call stacks. The command ~profiler-extras-profiler-report-toggle-entry-global~ calls ~profiler-report-toggle-entry~ with a prefix argument of 4, which recursively expands all sub-entries below the entry at point. This lets you quickly inspect the complete call hierarchy without repeated toggling.
