Project-local skills for macOS maintenance live under this directory so they
load only while working on macOS security and machine-posture tasks.

*** audit-mac-app
:PROPERTIES:
:ID:       22CD790D-004C-42DE-8AA3-AF9CB0088948
:END:

Static-first security audit of macOS applications before running them or
granting sensitive permissions. Covers trust verification, permission analysis,
app type detection, Electron source extraction and security checks, network
endpoint scanning, and spaces-safe binary analysis.

*** security-audit
:PROPERTIES:
:ID:       E7A2C626-9A58-48B3-B8F8-124BE3A61945
:END:

Audits the development environment security posture across secrets hygiene,
supply-chain dependencies, macOS machine posture, and Claude Code configuration
risk. It is audit-only unless remediation is explicitly requested and reports
findings by severity with concrete locations and remediation steps.

** Personal updates
:PROPERTIES:
:ID:       ADAD483E-B9D6-498E-974B-0A4D3775C2E0
:END:

=bin/personal-updates= manages Homebrew updates with three scheduled jobs:

| Job | Runs | What it does |
|-----+------+--------------|
| =fast-brew= | 03:00 daily | Upgrades fast-track packages immediately. Defaults: =claude-code@latest= and =codex=. |
| =scan= | 08:15 daily | Checks what Homebrew packages are outdated and records when each update was first seen. |
| =delayed= | 09:00 daily | Upgrades packages that have been outdated for at least 7 days. |

The important idea is simple: critical tools can update immediately, while
ordinary Homebrew updates wait until they have been visible for a week.

The state file is:

#+begin_src text
~/.local/state/personal-updates/state.json
#+end_src

The policy files are:

| File | Meaning |
|------+---------|
| =macos/personal-updates/fast-brew.txt= | Extra packages to update immediately. |
| =macos/personal-updates/hold.txt= | Packages to skip in the delayed updater. |
| =macos/personal-updates/excluded-casks.txt= | Casks to record but never upgrade automatically. |

Leave those files empty unless a package needs special handling.

Manual checks:

#+begin_src sh
bin/personal-updates fast-brew --dry-run
bin/personal-updates delayed --dry-run
#+end_src

Install or refresh the scheduled jobs with:

#+begin_src sh
mkdir -p ~/Library/LaunchAgents ~/.local/state/personal-updates
ln -sfn "$PWD/macos/LaunchAgents/com.stafforini.personal-updates.fast-brew.plist" \
  ~/Library/LaunchAgents/com.stafforini.personal-updates.fast-brew.plist
ln -sfn "$PWD/macos/LaunchAgents/com.stafforini.personal-updates.scan.plist" \
  ~/Library/LaunchAgents/com.stafforini.personal-updates.scan.plist
ln -sfn "$PWD/macos/LaunchAgents/com.stafforini.personal-updates.delayed.plist" \
  ~/Library/LaunchAgents/com.stafforini.personal-updates.delayed.plist
launchctl bootstrap "gui/$(id -u)" ~/Library/LaunchAgents/com.stafforini.personal-updates.fast-brew.plist
launchctl bootstrap "gui/$(id -u)" ~/Library/LaunchAgents/com.stafforini.personal-updates.scan.plist
launchctl bootstrap "gui/$(id -u)" ~/Library/LaunchAgents/com.stafforini.personal-updates.delayed.plist
#+end_src

There is also a manual command for macOS system updates:

#+begin_src sh
bin/personal-updates fast
#+end_src

That command runs =softwareupdate --install --recommended=. It is not scheduled
because macOS updates may require privileges or a restart.
