This manual describes the features and customization options for the Emacs Lisp file =org-ref-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=org-ref-extras.el= extends =org-ref=, the comprehensive citation and bibliography management package for Org mode. Rather than adding new commands, this package focuses exclusively on patching existing =org-ref= functions to fix behaviors that interfere with a particular BibTeX formatting workflow.

The package uses =el-patch= to make two targeted modifications:

- *Disable automatic filling after sorting.* The standard ~org-ref-sort-bibtex-entry~ command calls ~bibtex-fill-entry~ after sorting, which wraps long field values. This patch removes that call, preserving the original line structure of field values ([[#h:sort-entry-patch][Sorting without filling]]).
- *Remove redundant Year field from ISBN imports.* The ~isbn-to-bibtex~ command fetches BibTeX data from =www.ebook.de=, which often includes both a =date= and a =Year= field. The patch strips the redundant =Year= line before inserting the entry ([[#h:isbn-patch][ISBN import cleanup]]).

The package depends on =org-ref= (including =org-ref-isbn=) and =el-patch=.

** Sorting without filling
:PROPERTIES:
:CUSTOM_ID: h:sort-entry-patch
:END:

#+findex: org-ref-sort-bibtex-entry
The command ~org-ref-sort-bibtex-entry~ reorders the fields within a BibTeX entry according to the precedence defined in ~org-ref-bibtex-sort-order~. In the upstream version, after sorting the fields the command calls ~bibtex-fill-entry~, which reformats every field value by wrapping long lines to fit within a fixed column width. This automatic filling can be undesirable when you want to preserve the exact formatting of field values, especially for entries with long URLs or titles that are more readable on a single line.

The =el-patch= in =org-ref-extras.el= removes the ~bibtex-fill-entry~ call while keeping the rest of the sorting and cleaning logic intact. The entry is still cleaned by ~bibtex-clean-entry~ after sorting, which validates the entry structure without altering line breaks within field values.

** ISBN import cleanup
:PROPERTIES:
:CUSTOM_ID: h:isbn-patch
:END:

#+findex: isbn-to-bibtex
The command ~isbn-to-bibtex~ (from =org-ref-isbn=) fetches a BibTeX entry for a given ISBN from the =www.ebook.de= service and inserts it into a bibliography file. The fetched data frequently includes a =Year= field alongside a =date= field, creating redundancy that can confuse BibTeX processors or produce duplicate information in formatted citations.

The =el-patch= in =org-ref-extras.el= adds a post-processing step that removes any line matching =Year =...= from the fetched entry string before it is inserted into the bibliography and cleaned. This ensures that only the =date= field (which is the preferred field in modern BibLaTeX usage) survives the import process. The rest of the import workflow --- including ISBN prompting from the region or kill ring, entry cleaning via ~org-ref-isbn-clean-bibtex-entry~ and ~org-ref-clean-bibtex-entry~, and saving the file --- remains unchanged.
