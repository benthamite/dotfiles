This manual describes the features and customization options for the Emacs Lisp file =outline-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=outline-extras.el= extends the built-in =outline= library with improved heading promotion and demotion commands. The built-in ~outline-promote~ and ~outline-demote~ commands, when called with a prefix argument, operate on the entire subtree but leave point at an unintuitive position afterward. The commands in this package wrap those operations so that point reliably returns to the heading line you just modified, making repeated promotions and demotions smoother.

The package requires only =outline= itself and works in any buffer where =outline-minor-mode= or a derived mode (such as =org-mode=) is active.

** Promoting and demoting headings
:PROPERTIES:
:CUSTOM_ID: h:promoting-and-demoting-headings
:END:

#+findex: outline-extras-promote-heading
#+findex: outline-extras-demote-heading
When restructuring a document you often need to promote or demote a heading together with its subtree. The built-in commands ~outline-promote~ and ~outline-demote~ accept a prefix argument of ~C-u~ to operate on the entire subtree, but after the operation point may land on the line below the heading rather than on the heading itself. This is disorienting when you want to perform several adjustments in sequence.

The command ~outline-extras-promote-heading~ promotes the current heading and its subtree by one level, then moves point back to the heading line. Similarly, ~outline-extras-demote-heading~ demotes the heading and its subtree by one level and repositions point on the heading. Both commands achieve this by calling the underlying =outline-promote= or =outline-demote= with a prefix argument of 4 and then invoking ~outline-previous-heading~ to correct the cursor position.

These commands are most useful when bound to convenient keys so that you can rapidly restructure an outline without losing your place.
