This manual describes the features and customization options for the Emacs Lisp file =org-roam-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=org-roam-extras.el= extends the =org-roam= package, a personal
knowledge management system for Emacs based on Zettelkasten principles.
It depends on =org-extras= for heading manipulation utilities and
=paths= for directory constants.

The package adds several capabilities on top of the standard =org-roam=
interface.  It provides a configurable node inclusion function that
filters out headings tagged with "noid" or "ARCHIVE", certain
conventional heading names (like "Local variables" or "Further reading"),
and sub-headings in specific directories such as journal entries.  This
keeps the Org Roam database focused on substantive notes.

For note creation, the package offers a workflow that prompts for a note
type (generic, person, or project-specific), determines the appropriate
directory and tags, creates a slugified file, and sets up the initial
structure.  A complementary command converts existing headings into
standalone notes.

The package enhances node finding with a specialized command that
queries the Org Roam database for prioritized nodes, optionally filtered
by tag or directory.  This surfaces actionable items sorted by priority,
skipping nodes that already have scheduled dates or deadlines.

Backlink tracking is supported through functions that count backlinks
and update the modeline (integrating with =doom-modeline-extras.el=).
An option to automatically display the backlink buffer when opening a
file with backlinks is also available.

A set of commands for replacing internal ~[[*Title]]~ links with proper
~[[id:UUID]]~ links automates the conversion of legacy link formats
within entire directories of Org files.

Finally, a statistics command summarizes your open TODO items and their
total effort estimates from the Org Roam database ([[#h:statistics][Statistics]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Exclusion lists
:PROPERTIES:
:CUSTOM_ID: h:exclusion-lists
:END:

#+vindex: org-roam-extras-excluded-dirs
#+vindex: org-roam-extras-excluded-files
The user option ~org-roam-extras-excluded-dirs~ specifies a list of
directory paths to exclude from Org Roam's file scanning.  Files within
these directories will not be indexed.  The default value includes
inactive, archive, and collaborator-specific directories.  Add
directories here when they contain Org files that should not appear in
Org Roam queries.

The user option ~org-roam-extras-excluded-files~ specifies a list of
file names or patterns to exclude from Org Roam.  Entries can be literal
file names or regular expression patterns.  The default includes
template files, old data files, and Dropbox conflict copies.  Both
options feed into ~org-roam-file-exclude-regexp~.

** Backlink buffer display
:PROPERTIES:
:CUSTOM_ID: h:backlink-buffer-option
:END:

#+vindex: org-roam-extras-auto-show-backlink-buffer
The user option ~org-roam-extras-auto-show-backlink-buffer~ controls
whether the Org Roam backlink buffer is automatically displayed when
opening a file that has backlinks.  The default value is ~nil~, meaning
the backlink buffer is not shown automatically.  Set it to ~t~ if you
want the backlink buffer to appear whenever you visit a note with
incoming links.

When enabled, ~org-roam-extras-show-backlink-buffer~ ([[#h:backlink-display][Backlink display]])
should be added to ~find-file-hook~.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Creating notes
:PROPERTIES:
:CUSTOM_ID: h:creating-notes
:END:

#+findex: org-roam-extras-new-note
#+findex: org-roam-extras-convert-heading-to-note
The command ~org-roam-extras-new-note~ creates a new Org Roam note
through a guided workflow.  It first prompts the user to select a note
type: "generic", "person", "Borges", "tango", or "tlon-notes".  The
type determines the target directory (e.g., ~paths-dir-notes~ for
generic, ~paths-dir-people~ for person) and the initial tags.

After selecting the type, it prompts for the note name, creates a file
with a slugified version of that name (via ~simple-extras-slugify~),
inserts the =#+title:= line and an initial heading, sets the tags,
creates an Org ID, narrows to the new entry, and places point at the end
of the buffer.

The command ~org-roam-extras-convert-heading-to-note~ extracts the
current Org heading and its subtree from the source buffer, prompts for
a note name (defaulting to the heading title), and creates a new file in
the current directory.  The heading content is moved into the new file,
which gets a =#+title:= line and an adjusted top-level heading.  The
original heading is removed from the source buffer.

** Finding nodes
:PROPERTIES:
:CUSTOM_ID: h:finding-nodes
:END:

#+findex: org-roam-extras-node-find
#+findex: org-roam-extras-node-find-special
#+findex: org-roam-extras-node-select-tag
The command ~org-roam-extras-node-find~ acts as an enhanced wrapper
around ~org-roam-node-find~.  Before invoking the finder, it widens the
buffer and shows all headings (via ~org-extras-fold-show-all-headings~).
After jumping to the selected node, it recenters the buffer.

The command ~org-roam-extras-node-find-special~ provides a filtered view
of Org Roam nodes.  It queries the database for nodes that have a
priority, have either no TODO state or the state "TODO", and do not have
a SCHEDULED or DEADLINE timestamp.  The results are presented sorted by
priority (ascending, so =#A= appears first).

The command accepts an optional FILTER-SPEC argument:

- ~nil~: no additional filter.
- ~t~ or a prefix argument: prompt the user to filter by tag or
  directory.
- ~(:tag TAG)~: filter by a specific tag.
- ~(:dir DIRECTORY)~: filter by a specific directory.

Selection uses ~consult--read~ for a completing-read interface with
previews.  Jumping to the selected node uses ~org-id-goto~.

The command ~org-roam-extras-node-select-tag~ prompts for a tag from all
tags present in the Org Roam database.  It is used internally by
~org-roam-extras-node-find-special~ when filtering by tag.

** Navigating to nodes
:PROPERTIES:
:CUSTOM_ID: h:navigating-nodes
:END:

#+findex: org-roam-extras-id-goto
The command ~org-roam-extras-id-goto~ jumps to the location of an Org
ID.  It widens the buffer both before and after the jump, which handles
the case where the target is in a currently narrowed buffer.  The jump
is performed twice (via ~org-roam-id-open~) to ensure the correct
position is reached even when narrowing interferes.

** Replacing star links with ID links
:PROPERTIES:
:CUSTOM_ID: h:replacing-star-links
:END:

#+findex: org-roam-extras-replace-star-links-with-id
The command ~org-roam-extras-replace-star-links-with-id~ walks through
every =.org= file in a given directory and converts links of the form
~[[*Title][Description]]~ into ~[[id:UUID][Description]]~, where UUID is
the Org ID of the node whose title matches.

The command is fully automatic: it visits each file, performs the
substitutions (using ~org-element-map~ for robust parsing), and saves
any modified buffers.  When called with a prefix argument (NOCASE), the
title matching is case-insensitive.  An optional DIRS argument restricts
the ID lookup to nodes within specific directories.

Internally, the command uses
~org-roam-extras--replace-star-links-with-id-in-file~ to process each
file and ~org-roam-extras-get-id-of-title~ to resolve titles to IDs
([[#h:link-utilities][Link utilities]]).

** Statistics
:PROPERTIES:
:CUSTOM_ID: h:statistics
:END:

#+findex: org-roam-extras-count-todos-and-efforts
The command ~org-roam-extras-count-todos-and-efforts~ queries the Org
Roam database for all nodes with a TODO keyword and tallies those that
are still open (i.e., not in a done state such as "DONE", "CANCELLED",
or "DELEGATED").  It sums their effort estimates and reports the total
count and effort in the echo area, formatted as hours and minutes.

This gives you a quick overview of your outstanding workload across the
entire Org Roam database without needing to visit individual files.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Node inclusion
:PROPERTIES:
:CUSTOM_ID: h:node-inclusion
:END:

#+findex: org-roam-extras-node-include-function
The function ~org-roam-extras-node-include-function~ is intended for
~org-roam-node-include~.  It returns ~t~ for headings that should
receive an Org ID and be included in the Org Roam database, and ~nil~
for those that should be excluded.

Exclusion criteria include:

- Headings tagged with "noid" or "ARCHIVE".
- Headings with specific conventional names such as "Local variables",
  "Evaluation", "History", "Further reading", "External links", and
  "Related entries".
- Sub-headings (level > 1) in specific directories such as the journal,
  Trash, and gptel directories.

** Backlink tracking
:PROPERTIES:
:CUSTOM_ID: h:backlink-tracking
:END:

#+findex: org-roam-extras-backlink-count
#+findex: org-roam-extras-update-backlink-count
#+findex: org-roam-extras-update-modeline
#+vindex: org-roam-extras-current-backlink-count
The function ~org-roam-extras-backlink-count~ queries the Org Roam
database for the number of backlinks pointing to the node at point.  It
returns ~nil~ if the buffer is not in ~org-mode~ or if no node ID is
found.

The function ~org-roam-extras-update-backlink-count~ calls
~org-roam-extras-backlink-count~ and stores the result in the
buffer-local variable ~org-roam-extras-current-backlink-count~.

The function ~org-roam-extras-update-modeline~ updates the backlink
count and forces a modeline refresh by calling
~doom-modeline-update-buffer-file-name~.  It is intended for hooks like
~buffer-list-update-hook~ to keep the modeline backlink count current.

** Backlink display
:PROPERTIES:
:CUSTOM_ID: h:backlink-display
:END:

#+findex: org-roam-extras-show-backlink-buffer
The function ~org-roam-extras-show-backlink-buffer~ displays the Org
Roam buffer when visiting a file with backlinks, provided
~org-roam-extras-auto-show-backlink-buffer~ is non-~nil~ ([[#h:backlink-buffer-option][Backlink buffer display]]).  It checks whether the Org Roam buffer is already
visible and only displays it if it is not.

** People directory
:PROPERTIES:
:CUSTOM_ID: h:people-directory
:END:

#+findex: org-roam-extras-get-people
The function ~org-roam-extras-get-people~ queries the Org Roam database
for level-1 headings in files within the people directory
(~paths-dir-people~).  It returns a list of =(title id)= pairs, useful
for building completion interfaces or reports about contacts.

** Note creation helpers
:PROPERTIES:
:CUSTOM_ID: h:note-helpers
:END:

#+findex: org-roam-extras-get-note-properties
#+findex: org-roam-extras-create-file-for-note
The function ~org-roam-extras-get-note-properties~ prompts the user to
select a note type and returns a list of =(DIR TAGS)= appropriate for
that type.  It is used by ~org-roam-extras-new-note~
([[#h:creating-notes][Creating notes]]).

The function ~org-roam-extras-create-file-for-note~ creates a file with
a slugified name in the specified directory (defaulting to
~paths-dir-notes~).  It signals an error if the file already exists.

** Link utilities
:PROPERTIES:
:CUSTOM_ID: h:link-utilities
:END:

#+findex: org-roam-extras-get-id-of-title
The function ~org-roam-extras-get-id-of-title~ returns the Org ID of the
node whose title matches exactly.  When NOCASE is non-~nil~, the
comparison is case-insensitive.  When DIRS is provided (as a list of
directory paths or a single path), only nodes within those directories
are considered.  An error is signaled if no node or multiple nodes match
the title.

** Capture support
:PROPERTIES:
:CUSTOM_ID: h:capture-support
:END:

#+findex: org-roam-extras-remove-file-level-properties
The function ~org-roam-extras-remove-file-level-properties~ is intended
for ~org-capture-before-finalize-hook~.  When the capture template key
is ="r"=, it removes the file-level "ID" property and ensures the first
heading receives an Org ID instead.

** Recent files
:PROPERTIES:
:CUSTOM_ID: h:recent-files
:END:

#+findex: org-roam-extras-recent
The function ~org-roam-extras-recent~ returns a list of Org Roam files
modified within the last DAYS.  An optional LIMIT argument causes the
function to return the list only if its length is below the limit,
returning ~nil~ otherwise.  The function excludes files matching
~org-roam-file-exclude-regexp~.
