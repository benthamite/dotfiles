This manual describes the features and customization options for the Emacs Lisp file =window-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=window-extras.el= extends Emacs' built-in ~window.el~ with commands for switching between windows, moving buffers across windows, and intelligently splitting frames. Emacs' default window management commands are powerful but sometimes cumbersome for common workflows like toggling between two windows or moving a buffer to an adjacent window without losing context.

The main capabilities are:

- *Switching to the last window.* A quick toggle between the current and most recently used window, including the minibuffer ([[#h:switch-to-last-window][Switching to the last window]]).
- *Intelligent splitting.* Automatically splitting frames into two or three windows based on frame width ([[#h:split-if-unsplit][Splitting]]).
- *Buffer movement.* Moving the current buffer to an adjacent window or swapping buffers between windows ([[#h:buffer-movement][Buffer movement]]).
- *Minibuffer and side window access.* Jumping to the active minibuffer or selecting the leftmost/rightmost window ([[#h:special-windows][Special window access]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~window-extras-frame-split-width-threshold~
:PROPERTIES:
:CUSTOM_ID: h:window-extras-frame-split-width-threshold
:END:

#+vindex: window-extras-frame-split-width-threshold
The user option ~window-extras-frame-split-width-threshold~ specifies the minimum frame width (in columns) at which ~window-extras-split-if-unsplit~ ([[#h:split-if-unsplit][Splitting]]) will create a three-window layout instead of a two-window layout. On wide displays, three side-by-side windows can be more productive than two very wide ones.

The default value is ~350~. The type is an integer. You might lower this value if you use a smaller font or increase it for larger fonts.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Switching to the last window
:PROPERTIES:
:CUSTOM_ID: h:switch-to-last-window
:END:

#+findex: window-extras-switch-to-last-window
A common workflow involves editing in one window, briefly switching to another for reference, and then wanting to return. The command ~window-extras-switch-to-last-window~ selects the most recently used window in the current frame, providing a quick toggle between two windows without relying on directional movement or window numbers.

If the minibuffer was the last selected window and is currently active, the command will switch to it. This makes it easy to return to an ongoing minibuffer session after checking something in a file buffer. If no other window exists, the command signals an error.

Internally, the command uses ~window-extras-get-last-window~ to determine the target window ([[#h:utility-functions][Utility functions]]).

** Splitting
:PROPERTIES:
:CUSTOM_ID: h:split-if-unsplit
:END:

#+findex: window-extras-split-if-unsplit
The command ~window-extras-split-if-unsplit~ ensures the frame has at least two windows. If the frame contains only one window, it splits horizontally with ~split-window-right~. If the resulting frame width exceeds ~window-extras-frame-split-width-threshold~ ([[#h:window-extras-frame-split-width-threshold][User options]]) and there are still only two windows, it splits again to create a third and balances all window sizes.

This command is called internally by the buffer movement commands ([[#h:buffer-movement][Buffer movement]]) to ensure there is always a target window available.

** Buffer movement
:PROPERTIES:
:CUSTOM_ID: h:buffer-movement
:END:

#+findex: window-extras-buffer-swap
#+findex: window-extras-buffer-move-right
#+findex: window-extras-buffer-move-left
The command ~window-extras-buffer-swap~ exchanges the buffer in the current window with the buffer in the most recently used window. If there is only one window, it first splits the frame via ~window-extras-split-if-unsplit~ ([[#h:split-if-unsplit][Splitting]]). This is useful when you realize two buffers would be more productive in each other's position.

The command ~window-extras-buffer-move-right~ moves the current buffer to the window immediately to the right, replacing the current window's buffer with the alternate buffer (from =files-extras.el=). If no window exists to the right, it creates one by splitting. Focus follows the buffer to the right window.

The command ~window-extras-buffer-move-left~ does the same leftward. When there is only one window, it splits the frame and leaves focus on the left window (which now shows the alternate buffer, with the original buffer having moved right). This difference from ~window-extras-buffer-move-right~ ensures that the user's focus matches the expected position.

All three commands use the internal function ~window-extras--move-or-swap~ for the actual buffer placement and ~window-extras--remove-buffer-from-window-history~ to clean up the source window's buffer history, preventing the moved buffer from reappearing unexpectedly.

** Special window access
:PROPERTIES:
:CUSTOM_ID: h:special-windows
:END:

#+findex: window-extras-switch-to-minibuffer-window
#+findex: window-extras-select-side-window
The command ~window-extras-switch-to-minibuffer-window~ switches focus to the active minibuffer window. This is useful when a minibuffer prompt is waiting for input but you have navigated away from it. If no minibuffer is active, the command does nothing.

The command ~window-extras-select-side-window~ prompts for a side (="left"= or ="right"=) and selects the window furthest in that direction, ignoring the minibuffer. This provides a quick way to jump to the edge of a multi-window layout without counting windows.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Utility functions
:PROPERTIES:
:CUSTOM_ID: h:utility-functions
:END:

#+findex: window-extras-get-last-window
#+findex: window-extras-buffer-move
#+findex: window-extras-frame-is-maximized-p
The function ~window-extras-get-last-window~ returns the most recently selected window in the current frame. If the minibuffer is active and the current buffer is not a minibuffer, it returns the minibuffer window. Otherwise, it uses ~get-mru-window~ to find the most recently used non-current window. This function underpins ~window-extras-switch-to-last-window~ ([[#h:switch-to-last-window][Switching to the last window]]) and the buffer movement commands.

The function ~window-extras-buffer-move~ moves the current buffer to a specified TARGET-WINDOW, replacing the current window's buffer with the alternate buffer from ~files-extras-get-alternate-buffer~. It handles window creation and history cleanup. It is used by ~window-extras-buffer-move-right~ and ~window-extras-buffer-move-left~ ([[#h:buffer-movement][Buffer movement]]).

The predicate ~window-extras-frame-is-maximized-p~ returns non-nil when the current frame's pixel dimensions match the display's pixel dimensions, indicating that the frame occupies the full screen.
