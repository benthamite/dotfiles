This manual describes the features and customization options for the Emacs Lisp file =citar-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=citar-extras.el= extends =citar=, the modern bibliography management interface for Emacs built on top of the ~completing-read~ API. Where =citar= focuses on selecting and acting on citation keys, =citar-extras.el= bridges the gap between the =citar= selection interface and other bibliography tools, particularly =ebib=.

The package provides three categories of functionality:

- *Ebib integration.* Commands to open a bibliographic entry selected via the =citar= completion UI directly in =ebib=, or to navigate to its raw BibTeX definition in the source =.bib= file ([[#h:ebib-integration][Ebib integration]]).
- *Cache management.* Commands to refresh the =citar= bibliography cache for one or all configured bibliography files, which is useful after external edits to =.bib= files ([[#h:cache-management][Cache management]]).
- *Visual indicators.* Custom indicator variables using =nerd-icons= that display file, link, note, and citation status icons in the =citar= selection UI ([[#h:visual-indicators][Visual indicators]]).

The package depends on =citar= and =paths=.

** Ebib integration
:PROPERTIES:
:CUSTOM_ID: h:ebib-integration
:END:

#+findex: citar-extras-open-in-ebib
#+findex: citar-extras-goto-bibtex-entry
#+findex: citar-extras-move-entry-to-tlon
The =citar= interface provides a convenient way to search for and select bibliographic entries, but its built-in actions may not cover every workflow. The commands in this section let you use =citar= as the selection front-end and then hand off to =ebib= or the raw BibTeX file for further work.

The command ~citar-extras-open-in-ebib~ prompts for a citation key using the =citar= completion interface and opens the corresponding entry in =ebib=. If =ebib= is not already running, it starts it first. This is useful when you want to inspect or edit the full metadata of an entry in the rich =ebib= interface after locating it through =citar=.

The command ~citar-extras-goto-bibtex-entry~ navigates directly to the BibTeX source definition of a selected entry. It determines which =.bib= file contains the entry (using ~ebib-extras-get-file-of-key~ from =bibtex-extras.el=), opens that file, and places point at the beginning of the entry. When called interactively without an argument, it prompts for a citation key. This command is helpful when you need to manually inspect or edit the raw BibTeX markup.

The command ~citar-extras-move-entry-to-tlon~ selects a citation key via =citar=, navigates to its BibTeX definition (using ~citar-extras-goto-bibtex-entry~), and then calls ~tlon-move-entry-to-fluid~ to move the entry to the Tlon project bibliography. This supports a workflow where entries are triaged from a personal bibliography into a shared project collection.

** Cache management
:PROPERTIES:
:CUSTOM_ID: h:cache-management
:END:

#+findex: citar-extras-refresh-bibliography
#+findex: citar-extras-refresh-all-bibliographies
The =citar= package maintains an internal cache of parsed bibliography data for fast completion. When bibliography files are modified outside Emacs or by other tools, the cache may become stale. The commands in this section let you explicitly refresh it.

The command ~citar-extras-refresh-bibliography~ refreshes the =citar= cache for a single bibliography file. Interactively, it prompts for the file path. When called with a prefix argument (FORCE non-nil), the cache is rebuilt unconditionally, even if =citar= believes it is up to date.

The command ~citar-extras-refresh-all-bibliographies~ refreshes the cache for every bibliography file returned by ~citar--bibliographies~. It accepts an optional FORCE prefix argument that is passed through to ~citar-extras-refresh-bibliography~ for each file ([[#h:cache-management][Cache management]]).

* Visual indicators
:PROPERTIES:
:CUSTOM_ID: h:visual-indicators
:END:

#+vindex: citar-extras-indicator-files-icons
#+vindex: citar-extras-indicator-links-icons
#+vindex: citar-extras-indicator-notes-icons
#+vindex: citar-extras-indicator-cited-icons
The =citar= package supports displaying small indicator icons next to each candidate in the completion interface, showing at a glance whether an entry has associated files, links, notes, or is cited in the current document. =citar-extras.el= defines four indicator variables that use icons from the =nerd-icons= package, replacing the default plain-text indicators with visually distinctive glyphs.

The variable ~citar-extras-indicator-files-icons~ displays a green file icon (=nf-fa-file_o=) for entries that have associated files, as determined by ~citar-has-files~.

The variable ~citar-extras-indicator-links-icons~ displays an orange link icon (=nf-fa-link=) for entries that have associated links, as determined by ~citar-has-links~.

The variable ~citar-extras-indicator-notes-icons~ displays a blue note icon (=nf-cod-note=) for entries that have associated notes, as determined by ~citar-has-notes~.

The variable ~citar-extras-indicator-cited-icons~ displays a green circle icon (=nf-fa-circle_o=) for entries that are cited in the current document, as determined by ~citar-is-cited~.

To use these indicators, set ~citar-indicators~ in your configuration:

#+begin_src emacs-lisp
(setq citar-indicators
      (list citar-extras-indicator-files-icons
            citar-extras-indicator-links-icons
            citar-extras-indicator-notes-icons
            citar-extras-indicator-cited-icons))
#+end_src
