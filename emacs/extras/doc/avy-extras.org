This manual describes the features and customization options for the Emacs Lisp file =avy-extras.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=avy-extras.el= extends =avy=, the package that lets you jump to visible text in Emacs buffers using short character sequences. While =avy= provides general-purpose jumping commands, it does not include line-scoped word jumping, end-of-line targeting, or mode-specific "jump and act" commands.

This package provides three groups of extensions:

- Word-jumping commands restricted to the current visual line, with variants for the whole line, behind point, or ahead of point ([[#h:word-jumping-in-line][Word jumping in line]]).
- End-of-line jumping commands for lines above or below the cursor ([[#h:end-of-line-jumping][End-of-line jumping]]).
- Mode-specific commands that combine ~avy-goto-line~ with an action in Dired, Ebib, mu4e, Telega, and Elfeed ([[#h:mode-specific-jumping][Mode-specific jumping]]).
- Character-based jumping commands inspired by Vim's =f= motion, with repeatable search via timeout ([[#h:character-jumping][Character jumping]]).

The package depends on =el-patch= and =use-package-extras=.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~avy-extras-goto-char-timeout~
:PROPERTIES:
:CUSTOM_ID: h:avy-extras-goto-char-timeout
:END:

#+vindex: avy-extras-goto-char-timeout
The user option ~avy-extras-goto-char-timeout~ controls how long the character jumping commands ([[#h:character-jumping][Character jumping]]) wait for a repeated keypress before exiting. If the same key is not pressed within this number of seconds, the search stops and the next keypress is treated as a normal command.

The default value is ~1~ (one second). A higher value gives you more time to decide whether to continue jumping; a lower value makes the command feel snappier but requires faster key presses.

The type is a number (integer or float).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Word jumping in line
:PROPERTIES:
:CUSTOM_ID: h:word-jumping-in-line
:END:

#+findex: avy-extras-goto-word-in-line
#+findex: avy-extras-goto-word-in-line-behind
#+findex: avy-extras-goto-word-in-line-ahead
When editing prose or code, you often want to jump to a nearby word on the same line without being overwhelmed by candidates from the entire visible window. Three commands restrict ~avy-goto-word-0~ to the current visual line:

- ~avy-extras-goto-word-in-line~ offers candidates across the entire visual line, from its beginning to its end.
- ~avy-extras-goto-word-in-line-behind~ offers candidates only between the beginning of the visual line and point. This is useful when you need to jump backward to fix a typo earlier on the same line.
- ~avy-extras-goto-word-in-line-ahead~ offers candidates only between point and the end of the visual line.

All three commands use ~avy-goto-word-0~ internally, so they respect your =avy= style and key settings.

** End-of-line jumping
:PROPERTIES:
:CUSTOM_ID: h:end-of-line-jumping
:END:

#+findex: avy-extras-goto-end-of-line-above
#+findex: avy-extras-goto-end-of-line-below
Sometimes you want to append text at the end of a nearby line. These commands combine ~avy-goto-line-above~ or ~avy-goto-line-below~ with a subsequent ~end-of-line~ call:

- ~avy-extras-goto-end-of-line-above~ jumps to a visible line above the cursor and moves point to its end.
- ~avy-extras-goto-end-of-line-below~ jumps to a visible line below the cursor and moves point to its end.

Both accept optional OFFSET and BOTTOM-UP arguments, which control the distance between the closest avy candidate and the cursor and the display order of candidates, respectively.

** Mode-specific jumping
:PROPERTIES:
:CUSTOM_ID: h:mode-specific-jumping
:END:

#+findex: avy-extras-dired-find-file
#+findex: avy-extras-ebib-view-entry
#+findex: avy-extras-headers-view-message
#+findex: avy-extras-telega-view-message
#+findex: avy-extras-elfeed-search-show-entry
Several Emacs modes display lists of items (files, bibliography entries, email messages, chat messages, feed entries) where you select an item by moving to its line and pressing a key. These commands combine line selection and action into a single step:

- ~avy-extras-dired-find-file~ jumps to a line in a Dired buffer and visits the file or directory on that line (via ~dired-find-alternate-file~, which reuses the current buffer).
- ~avy-extras-ebib-view-entry~ jumps to a line in an Ebib index buffer and opens the entry for editing (via ~ebib-edit-entry~).
- ~avy-extras-headers-view-message~ jumps to a line in a mu4e headers buffer and views the message (via ~mu4e-headers-view-message~).
- ~avy-extras-telega-view-message~ jumps to a line in a Telega chat list and opens the chat (via ~push-button~).
- ~avy-extras-elfeed-search-show-entry~ jumps to a line in an Elfeed search buffer and displays the entry (via ~elfeed-search-show-entry~).

** Character jumping
:PROPERTIES:
:CUSTOM_ID: h:character-jumping
:END:

#+findex: avy-extras-goto-char-forward
#+findex: avy-extras-goto-char-backward
These commands provide Vim-style =f= and =F= motions: they prompt for a character and immediately jump to its first occurrence in the specified direction. Unlike =avy-goto-char=, which displays overlay hints for all visible occurrences, these commands jump directly to the nearest match and then allow you to repeat the jump by pressing the same key again.

- ~avy-extras-goto-char-forward~ prompts for a character and jumps forward to its first occurrence. Pressing the same character again within the timeout period ([[#h:avy-extras-goto-char-timeout][~avy-extras-goto-char-timeout~]]) advances to the next occurrence. Pressing a different key or waiting for the timeout exits the search and processes the key normally.
- ~avy-extras-goto-char-backward~ works the same way but searches backward.

The last character used is stored in the variable ~avy-extras-last-char~.

** Region marking action
:PROPERTIES:
:CUSTOM_ID: h:region-marking
:END:

#+findex: avy-extras-action-mark-to-char
The function ~avy-extras-action-mark-to-char~ is an avy action function that activates the mark and moves point to the selected position, effectively creating a region from the original point to the avy target. This is useful as a custom avy action for selecting text between two positions.

This function is adapted from Karthink's "Avy can do anything" guide.
